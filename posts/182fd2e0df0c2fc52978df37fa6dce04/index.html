nginx基础及编译安装

一. nginx基础

（一） 什么是nginx

        nginx 是由俄罗斯人开发的开源软件，它同 Apache，Tomcat 一样是一种服务器软件。nginx 可以搭建 Web 服务器，同时也可以做正向/反向代理服务器，同时还可以用作邮件代理服务器以及实现负载均衡。nginx 在 Web 服务器市场份额排位第二（第一是 Apache），它弥补了 Apache 高并发处理能力的不足。

 

（二）为什么选择nginx

1. 作为 Web 服务器而言，Nginx 相比于 Apache 性能更好，占用的内存更少，更加稳定轻量。

2. 功能强大。Web 服务器，代理服务器，邮件服务器，负载均衡等。

3. 可扩展性高。nginx 采用高度模块化，低耦合的设计，它是由多个不同功能、不同层次、不同类型的模块组成的，而且这种低耦合模块化的设计使得 nginx 具有数量庞大的第三方库。

4. 跨平台。支持 UNIX，Linux，也有 Windows 移植版本。

5. 配置简单。配置文件修改方便，还支持热部署，热升级。

 

二. nginx编译安装

（一）nginx下载

官网：http://www.nginx.org

nginx 的几种版本认识

1. mainline 版本。主线版本，是 nginx 目前主力研发的版本，版本号中间数字为奇数，更新快，一般一个月就会发布一个新版本，里面含有最新功能和 bug 修复，缺点就是稳定性会差一些。

2. stable 版本。最新的稳定版本，版本号中间数字为偶数，稳定性好，生产环境建议选择这个版本。

3. legacy 版本。老的稳定版本。

这里我们选择 nginx 14.2 版本。在命令行输入 wget http://nginx.org/download/nginx-1.14.2.tar.gz 就可以将 nginx 安装包下载到当前目录下，再通过 tar -zxvf nginx-14.2.2.tar.gz 将压缩包解压出来。

 

（二）安装前提

1. 要求 Linux 内核版本为 2.6 或以上，因为早期 Linux 并不支持 epoll（可以使用 uname -r 查看当前系统内核版本）。

2. 编译安装，需要安装 gcc 和 g++（sudo apt-get install gcc g++）。

3. 安装 pcre 库（解析正则表达式）（sudo apt-get install libpcre3-dev）。

4. 安装 zlib 库（压缩和解压缩）（sudo apt-get install libz-dev）。

5. 安装 openssl 库（ssl 功能相关库，用于网站通讯加密）（sudo apt-get intall openssl libssl-dev）。

 

（三）编译安装

1. 首先进入 nginx 源码包所在的目录，在命令行下输入 ./configure --help 可以查看支持的安装编译选项。

--prefix=PATH 指定安装路径。

--sbin-path=PATH 指定安装后的二进制文件存放路径。

--conf-path=PATH 指定安装后的配置文件存放路径。

        我们执行 ./configure 默认安装，在当前目录下就会看到一个 Makefile 文件和一个 objs 目录，objs 目录下有一个 ngx_modules.c 文件，里面可以看到安装时哪些模块被包含进去了。

2. 执行 make 进行编译。

3. 执行 make install 进行安装（默认安装到 /usr/local/nginx 目录）。

 

三. 启动nginx

        安装完成后，进入 nginx 安装目录（默认安装是 /usr/local/nginx），可以看到目录结构下有 sbin 目录，进入 sbin 目录后，执行 ./nginx 即可启动。如果 Linux 安装了图形界面和浏览器，在浏览器输入 127.0.0.1，可以看到 Welcome to nginx! 。如果没有安装图形界面和浏览器，可以通过其他机器访问启动了 nginx 服务的 IP 地址。如果看到下面的显示信息说明已经安装并且成功启动了。
