AJAX版本、响应数据、超时设置、同步异步

关于 AJAX

  1. 浏览器与服务器之间通信的基础是HTTP协议
  2. 用户通过网址或表单向服务去提交请求，服务器向浏览器发送响应
    AJAX： Asyncchronous JavaScript and XML 异步的JavaScript和XML
    它是什么？ Javascript脚本发起的HTTP通信
    AJAX历史:
    1999 IE5.0 允许JS脚本向服务器单独发送HTTP请求的新功能
    2004 Gmail 推出异步邮件更新服务
    2005 Google Map异步更新地图服务
    2005 AJAX被大厂公认命名
    2006 W3C发布AJAX国际标准
    JavaScript异步通信：请求服务器返回JSON/XML文档，前端从XML文档中提取数据，在不刷新整个网页的基础上，渲染到网页相应的位置

AJAX创建XMLHTTPREQUEST对象

原生XMLHttpRequest对象与ActiveX对象
作用：JS脚本HTTP请求的发起必须通过XMLHttpRequest对象，但是通过AJAX进行浏览器与服务器通信接口，不局限于XML，可以发送任何格式的数据
XMLHttpRequest本身是一个js引擎内置的构造函数, 所有的XMLHttpRequest对象都需要被实例化

var xhr = new XMLHttpRequest();

兼容性:(IE5/IE6使用ActiveXObject对象)

if(window.XMLHttpRequest) {
	xhr = new XMLHttpRequest();
} else {
	xhr = new ActiveXObject('Microsoft.XMLHTTP');
}

2、AJAX事件
onreadstatechange事件： 挂载到XMLHttpRequest对象上的事件
readyState状态：通过对XMLHttpRequest对象发送HTTP请求的各个阶段状态码（0-4）
0：请求未初始化
1：服务器连接已建立
2：请求已接收
3：请求处理中
4：请求已完成
没发送之前是0，发送以后是1，onreadystatechange事件里面监控的是2，3，4这几个状态，readystate仅仅是针对请求的状态码，获取资源是否成功取决于status的状态
status状态： 服务器响应的状态码（200🆗， 404未找到页面）
当ReadyState 变化时，将触发onreadstatechange事件执行其回调函数
3、AJAX服务器响应
responseText:获取字符串数据
responseXML:获取XML数据
4、AJAX-POST请求方式注意点
post请求方式下，send参数中的格式为a=1&b=3
post请求方式必须设置这个请求头信息

AJAX的版本

XMLHttpRequest标准分为level1和level2

XMLHttpRequest level缺点
1、无法发送跨域请求
2、不能传递非纯文本的数据
3、无法获取传输进度
XMLHttpRequest Level II改进
1、可以发送跨域请求
2、支持获取二进制数据（非纯文本数据）
3、支持上传文件（图片、文档、视频）
4、formData对象
5、可以获取传输时间
6、可以设置超时时间
兼容性问题
1、IE8/9Opara Mini不支持xhr对象，不支持level2
2、IE10/11不支持响应类型为JSON
3、部分浏览器不支持超时设置
4、部分浏览器不支持blob（文件对象的二进制数据）

AJAX状态码与状态提示

xhr.status/xhr.statusText:服务器回应的HTTP状态码/服务器发送的状态提示
200 OK 访问正常
301 Moved Permanently 永久移动
302 move temporarily 暂时移动
304, Not Modified 未修改
307 Temporary Redirect 暂时重定向
401 Unauthotized 未授权
403 Forbidden 禁止访问
404 Not Found 未发现指定网址
500 Internal Server Error 服务器发生错误

AJAX响应事件

xhr.onloadstart :绑定HTTP请求发出的监听事件
xhr.onload: 绑定请求成功完成的监听事件
xhr.onerror:绑定请求失败的监听函数
xhr.onabort:绑定请求终止的监听事件（调用了abort()函数）
xhr.onloadend:绑定请求完成（不管成功与失败）的监听函数
事件流程: loadstart ---> readState === 4 --> load/error/abort --> loadend

AJAX超时设置

xhr.timeout: 多少毫秒后，如果请求仍没有结果，就会自动终止，如果值为0，表示没有时间限制
xhr.ontimeout:绑定请求超时一个监听函数，如果发生timeout时间，就会触发这个函数

xhr.timeout = 10000;
xhr.ontimeout = function()
{
xhr.abort();//终止请求
xhr = null;
}

但是由于timeout的兼容性不好，所以，AJAX的超时设置，利用定时器来模拟

AJAX的同步异步

async: 异步默认的async为true，AJAX异步请求时，不影响页面的加载，用户操作以及AJAX程序后的程序执行
同步：async: false,AJAX发送同步请求，浏览器就必须等到请求完成并响应成功后，解决同步异步配置的问题
场景：AJAX请求后，在成功的回调函数中给某一全局变量赋值，如果是异步请求，是不能赋值成功的，因为AJAX操作需要时间，而外部程序执行比AJAX快，所以不会成功，但是异步可以，因为程序阻塞

AJAX 的 dataType

dataType:AJAX请求返回的数据类型
JSON TEXT XML
xhr.responseXML返回xml类型
