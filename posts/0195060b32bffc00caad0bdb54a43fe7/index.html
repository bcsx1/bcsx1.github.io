如何在后台运行 Linux Shell 命令 / 脚本 ?

在 Linux 终端上执行命令的通常只需运行它，然后等待它优雅地退出即可。一旦命令退出，您可以继续执行其他命令，这就是所谓的前台中的运行命令。顾名思义，您可以在终端上看到命令的输出。

然而，有时在前台运行命令会带来一系列挑战。该命令可能需要很长时间才能退出，可能会浪费您宝贵的时间，它可能完全依赖于 shell 会话，让你陷入困境。

在这种情况下，在后台运行命令是最好的选择。当您在前台并发地执行其他命令时，可以将一个(多个) 命令发送到后台。这提高了在终端上工作的效率，节省了您的时间。

在本指南中，我们关注如何在后台运行 Linux shell 命令或脚本。

在后台使用“&”符号运行 shell 命令

要在后台运行命令或脚本，请在末尾用&号结束它，如下所示

$ command &

注意: 以&号结束命令并不会将该命令与您分离。它只是将它发送到您正在使用的当前 shell 的后台，该命令仍然将输出打印到 STDOUT 或 STDERR，这将阻止您在终端上执行其他命令。

一种更好的方法是将命令重定向到 /dev/null，然后在末尾附加 & 符号，如下所示

$ command &>/dev/null &

要确认命令被发送到后台，运行 jobs 命令

如果需要终止后台进程，执行 kill 命令，后面跟着进程的 PID，选项 -9 将立即终止进程。

$ kill -9 138314

使用 nohup 命令在后台运行 shell 命令或脚本

在后台运行命令的另一种方法是使用 nohup 命令。nohup 命令是 no hang up 的缩写，它是一个命令，即使在退出 shell 后也能保持进程运行。

它通过阻止进程接收 SIGHUP (信号挂起) 信号来做到这一点，这个信号通常是在进程退出终端时发送给进程的。

要在后台发送命令或脚本并保持其运行，请使用以下语法：

$ nohup command &>/dev/null &

$ nohup shell-script.sh &>/dev/null &

例如：

$ nohup ping google.com &>/dev/null &

同样，您可以使用 jobs 命令来验证该命令是否在后台运行

$ jobs

末尾总结

我们介绍了在后台运行命令或 shell 脚本的两种方法。我们已经了解了如何使用&符号和 nohup 命令在后台运行命令。
