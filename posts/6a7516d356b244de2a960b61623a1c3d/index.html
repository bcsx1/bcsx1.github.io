js中的观察者模式 和发布订阅模式

​ 前言:作为一个职场小白,这个面试题是有机会问到的,我也没有去了解过这个方面,我知道有这个东西的时候是,问新同事的面试题什么才知道有这个东西!其实我们使用vue,里面也有发布订阅模式,下面我使用了nodejs的例子阐述下,这两种模式

观察者模式:

//在观察者模式中,只有两个角色 目标<-->观察者 者两个

//举例代码
// 被观察者 小宝宝 心情好和不好
//function Subject(name){this.name = name this.observers = []}   也可以使用构造函数写法
class Subject {  
    constructor(name) {
        this.name = name
        this.observers = []  //被观察者要放在 观察者中
        this.status = '开心'
    }
    // 被观察者要提供一个接受观察者的方法
    watchFn (observer) {   //Subject.prototype.watchFn(observer){...} //相当于构造函数写法
        this.observers.push(observer) //存放所有的观察者
    }
    //更改小宝宝状态的方法
    setState (newState) {
        this.state = newState
        this.observers.forEach(o => o.update(newState))  //发布订阅模式
    }
}
class Observer {
    constructor(name) {
        this.name = name
    }
    update (newState) {
        console.log('我通知了' + this.name + ',' + '小宝宝' + newState);
    }
}
let obf = new Observer('爸爸')
let bb = new Subject('小宝宝')
bb.watchFn(obf)  //添加 监听对象
setTimeout(() => {
    bb.setState('哭了')
}, 2000);
setTimeout(() => {
    bb.setState('笑了')
}, 4000);

分析:

  1. 可以看到,首先设置两个class ,一个是小宝宝的class ,先定义了构造器有name(名字),和state(状态),还定义了一个空的数组observers(观察者)

  2. 宝宝class提供了两个函数 一个是观察者函数,还有一个改变宝宝状态的函数=>{

    ​ 观察者函数:作用是存放所有的观察者 存着每一偶个new出来观察者对象(可以使用该对象的原型方法update)

    ​ 改变宝宝状态函数:

    ​ 1 .把小宝宝的状态改变 this.state=new state
    ​ 2.遍历所有的存储观察者,调用观察者类的函数=>

    ​ update(newState){

    ​ …处理的过程

    ​ }

    }

  3. 设置的观察者class=>就是设置name值,还有一个当宝宝更新状态触发的处理函数update

发布订阅模式

//在发布订阅模式中 发布-->中间代理 <--订阅 发布和订阅是没有关系的

// 发布-->中间代理 <--订阅  发布和订阅是没有关系的
const fs = require('fs')
const path = require('path')
function resolveFile (dir) {

    return path.join(__dirname, dir)
}
function Events () {
    this.callback = [],
        this.results = []
}
Events.prototype.on = function (callback) {  //订阅
    this.callback.push(callback)
}
Events.prototype.emit = function (data) {  // 发布
    this.results.push(data)
    this.callback.forEach(c => c(this.results)) //这个就是触动on的this.callback里面存的都是callback函数 发布就要触发订阅
}
let e = new Events()   //new 一个event实例 
e.on(function (arr) {   //订阅一个回调函数
    if (arr.length === 3) {
        console.log(arr)
    }
})
fs.readFile(resolveFile('name.text'), 'utf8', function (error, data) {//这个触发函数,又有点像是中间代理
    e.emit(data + 1)
    // console.log(data + 1);
})
fs.readFile(resolveFile('name.text'), 'utf8', function (error, data) {
    e.emit(data + 2)
    // console.log(data + 1);
})
fs.readFile(resolveFile('name.text'), 'utf8', function (error, data) {
    e.emit(data + 3)
    // console.log(data + 1);
})

分析:

  1. 发布订阅模式,有3个角色,发布者,订阅者,中间代理(发布与订阅,中间没有任何关系)

  2. 本例子使用的是nodejs查询文件内容

  3. 使用nodesjs核心模块path,fs=>{

    ​ function resolveFile(dir){ //获取绝对路径,或者基准路径

    ​ return path.join(__dirname, dir)

    ​ }

    ​ }

  4. 定义一个构造函数(Event),函数里面声明两个数组:回调函数的数组(callback),传递信息的数组(results)

  5. 在Event构造函数中的原型中挂载on(发布),emit(订阅)函数=>{

     									on函数是把触发的回调函数函数存到,callback数组中去(用户必须先订阅才能触发emit函数)

    ​ emit函数接受一个传递信息为参数,把信息加到results,然后遍历回调函数,调用每个遍历 的回调函数,回调函数传入信息作为入参

    ​ }

  6. new Event对象

  7. e.on(传递的信息){

    ​ …执行的函数

    }

  8. 这里我是使用nodejs,fs.readFile函数 这个函数是异步函数,所以不是按照顺序 data1,data2,data3执行

  9. 在fs.readFile的第3个参数是自带的回调函数

    这里我们使用e.emit(传入信息)=>{

    ​ 把信息加到results,然后遍历回调函数,调用每个遍历的回调函数,callback回调函数传入信息作为 入参.

    ​ callback函数就是我们定义的e.on(信息)函数

    ​ }

总结

  1. 我们可以看出,观察者模式跟发布订阅模式是有一部分相同,又不同的

  2. 观察者模式,只有两个角色,观察者和被观察者,当被观察者发生变化,也就会触发通知观察者函数

  3. 发布订阅模式,有3个角色,发布者,订阅者,中间代理,首先会是先产生订阅者发布监视观察者,当触发了观察者,就会触发中间代理来通知订阅者
