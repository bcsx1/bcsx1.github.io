import javax.servlet.RequestDispatcher;
import javax.servlet.ServletException;
import javax.servlet.annotation.WebServlet;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import java.io.IOException;

@WebServlet("/StudentServlet")
/*这里是一个很简单的注释，由是 Java Servlet 3.0 规范中引入的注解*/


public class StudentServlet extends HttpServlet {
    protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {

        request.setCharacterEncoding("UTF-8");//这里是定义编码的规则


        //这里就是一系列的定义变量，从下面的页面拿值过来
        String name = request.getParameter("name");
        String studentId = request.getParameter("studentId");
        int age = Integer.parseInt(request.getParameter("age"));

        Student student = new Student();
        student.setName(name);
        student.setStudentId(studentId);
        student.setAge(age);

        //这里是我一直出错用来测试的代码
       /* System.out.println("Name: " + name);
        System.out.println("Student ID: " + studentId);
        System.out.println("Age: " + age);*/

        request.setAttribute("student", student);


        response.setContentType("text/html; charset=UTF-8"); // 也是设置内容类型和字符编码
        response.setCharacterEncoding("UTF-8");



        //获取数据成功最后显示的界面
        RequestDispatcher dispatcher = request.getRequestDispatcher("viewStudent.jsp");
        dispatcher.forward(request, response);
    }
}

在Java Servlet中，request.setCharacterEncoding("UTF-8") 方法是用来设置请求的字符编码，它告诉服务器将请求的数据按照指定的字符编码进行解析。因此，这个设置应该在读取请求参数之前进行，以确保服务器正确地解析请求中的参数。

当你使用 request.getParameter("name") 等方法来获取请求参数时，服务器需要知道如何解码这些参数的字节流。如果在读取参数之前设置了编码，服务器就会按照指定的编码进行解码，从而正确地获取参数的值。

如果你把 request.setCharacterEncoding("UTF-8") 放在读取参数之后，可能会导致参数的解析不正确，因为服务器已经按照默认的编码（通常是ISO-8859-1）对请求进行了解析，此时再设置编码已经晚了。

所以，为了确保正确的解析请求参数，通常建议在读取参数之前就设置请求的字符编码。这是一种良好的实践，确保在获取参数时使用正确的编码，从而避免乱码等问题。

HTTP 协议定义了多种请求方法，其中常见的有 GET 和 POST。这两种请求方法在使用上有一些区别，主要涉及到参数的传递方式、安全性等方面。

  1. 参数传递方式：

    • GET： 参数以查询字符串的形式附加在 URL 后面，例如 http://example.com/page?name=value。在 URL 中，参数之间使用 & 符号分隔。
    • POST： 参数传递在请求体中，而不是在 URL 中，因此在请求 URL 上不可见。

  2. 安全性：

    • GET： 由于参数在 URL 上可见，因此不适合传递敏感信息，如密码。GET 请求的参数会被保存在浏览器的历史记录、服务器的日志文件等地方。
    • POST： 参数在请求体中，不会保存在浏览器历史记录中，相对更安全。适合传递敏感信息。

  3. 数据量：

    • GET： 由于参数在 URL 中，传输的数据量有限，一般用于获取资源。URL 有长度限制，对于大量数据的传递不适用。
    • POST： 可以传递大量数据，适合用于提交表单等操作。

关于我的问题，为什么 GET 请求不会出现乱码，主要是因为在一般情况下，GET 请求的参数直接附加在 URL 上，而 URL 使用的编码通常是 UTF-8。所以，对于简单的 GET 请求，浏览器默认会使用 UTF-8 编码进行参数的传递，不容易出现乱码问题。

然而，如果在 URL 中手动指定了其他编码，或者服务器端在解析请求时使用了不同的编码，就可能导致乱码问题。因此，保持 URL 使用默认的 UTF-8 编码通常可以避免这类问题。
