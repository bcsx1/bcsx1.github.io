ASP.NET CORE的ActionFilter过滤器详解

ActionFilter是ASP.NET CORE提供的一种方法过滤器或拦截器
一般用来在方法执行前或方法执行后来进行某些操作（AOP）
功能有点类似于Spring的Interceptor和AspectJ提供的AOP功能，
但实现的方法有好几种，下面我来介绍一下

1.实现IAcionFilter接口，重写其中方法

IActionFilter需要实现两个方法OnActionExecuting，OnActionExecuted
它们分别代表方法执行前，和方法执行后。

public class TestFilter : IActionFilter
{
	//方法执行后调用
    public void OnActionExecuted(ActionExecutedContext context)
    {
        throw new NotImplementedException();
    }
	//方法执行前调用
    public void OnActionExecuting(ActionExecutingContext context)
    {
        throw new NotImplementedException();
    }
}

这里测试一下

public class TheFirstFilter : IActionFilter
{
    private readonly ILogger<TheFirstFilter> logger;

    public TheFirstFilter(ILogger<TheFirstFilter> logger)
    {
        this.logger = logger;
    }

    public void OnActionExecuted(ActionExecutedContext context)
    {
        logger.LogInformation("Do something After Action");
    }

    public void OnActionExecuting(ActionExecutingContext context)
    {
        logger.LogInformation("Do something Before Action");
    }
}

在Services里配置过滤器

 services.AddMvc(option => {
                option.Filters.Add(typeof(TheFirstFilter));
 });

创建方法

[HttpGet]
public string ActionFilterTestOne()
{
    logger.LogInformation("The Action is Executing Now");

    return "ActionFilterTestOne";
}

执行

一.ActionExecutingContext的使用

这是方法执行前的参数，它和ActionExecutedContext一样继承了FilterContext，但还是有些不同的

1.无法在对应的方法执行前的函数中获取返回值

虽然ActionExecutingContext提供了result属性，当你获取时，会造成空指针异常
改造Filter如下。

public class TheFirstFilter : IActionFilter
{
    private readonly ILogger<TheFirstFilter> logger;

    public TheFirstFilter(ILogger<TheFirstFilter> logger)
    {
        this.logger = logger;
    }

    public void OnActionExecuted(ActionExecutedContext context)
    {
        logger.LogInformation("Do something After Action");
    }

    public void OnActionExecuting(ActionExecutingContext context)
    {
        //尝试在方法执行前获取方法的返回值
        ObjectResult result = (ObjectResult)context.Result;

        logger.LogInformation(result.Value.ToString());
    }
}

再次调接口时报错

2.可以很方便的获取方法参数值和类型

ActionExecutingContext提供了一个ActionArguments属性，它直接就是参数的名和值的Map类型，非常方便使用
测试：

public void OnActionExecuting(ActionExecutingContext context)
 {
     //获取方法的参数名和值
     IDictionary<string, object> argsMap = context.ActionArguments;

     foreach(KeyValuePair<string,object> pair in argsMap)
     {
         logger.LogInformation("Arg:" + pair.Key + " | Value:" + pair.Value);
     } 
 }

在调用的接口中添加一些参数

[HttpGet]
public string ActionFilterTestOne(string paramOne,string paramTwo,string paramThree)
{
    logger.LogInformation("The Action is Executing Now");

    return "ActionFilterTestOne";
}


（PS:还有更吊的方法，可以用来获取参数的Type信息，用来完成一些更复杂的操作）

 public void OnActionExecuting(ActionExecutingContext context)
 {
     //获取方法的参数名和值
     IDictionary<string, object> argsMap = context.ActionArguments;

     foreach(KeyValuePair<string,object> pair in argsMap)
     {
     	 //可以通过ActionExecutingContext和ActionExecutedContext都有的ActionDescriptor.Parameters中获取参数的类型信息
         logger.LogInformation
         ("Arg:" + pair.Key + " | Value:" + pair.Value + " | Type:" + context.ActionDescriptor.Parameters.Where(item=>item.Name==pair.Key).First().ParameterType);
     } 
 }

3.中断方法的执行，设置方法短路

前面说到，ActionExecutingContext无法获取Result属性，
但是可以设置Result属性，然后通过空return进行方法短路，不再执行调用的方法体，直接结束请求
这一点在做JWT的验证器Filter中是被大量实践的，
当然也可以用来验证Session
测试：

public class TokenFilter : IActionFilter
{
   
    public void OnActionExecuting(ActionExecutingContext context)
    {
        string P1 = context.ActionArguments["paramOne"].ToString();

        if (!P1.Equals("12345"))
        {
            context.Result = new JsonResult("方法的执行被短路");
			//设置返回值，并调用空返回，则直接中断方法
            return;
        }
    
    }

    public void OnActionExecuted(ActionExecutedContext context){}
}

测试

二.ActionExecutedContext的使用

ActionExecutedContext对应的方法执行后，
可以获取执行的返回值
但无法获取函数的参数（ActionExecutedContext没有ActionArguments属性）

1.获取返回值，记录一些数据

如果有一些记录表需要写入的话，缓存需要设置的话，都可以使用，
直接获取Result属性即可

 public class TokenFilter : IActionFilter
 {
     private readonly ILogger<TokenFilter> logger;

     public TokenFilter(ILogger<TokenFilter> logger)
     {
         this.logger = logger;
     }

     public void OnActionExecuting(ActionExecutingContext context)
     {
        .....
     }

     public void OnActionExecuted(ActionExecutedContext context)
     {
         ObjectResult res = (ObjectResult)context.Result;

         logger.LogInformation("记录方法的返回值为"+res.Value.ToString());
         
     }
 }

调用方法

[HttpGet]
public string ActionFilterTestOne(string paramOne,string paramTwo,string paramThree)
{
    logger.LogInformation("The Action is Executing Now");

    return "ActionFilterTestOne";
}

缺陷：

1.没有环绕AOP的方法，只能单纯的在前后添加操作，前后操作不能连续
（PS:如果想要使用环绕的AOP，需要使用IAsyncActionFilter）

2.继承ActionFilterAttribute

提示：

(PS:ActionFilterAttribute有一个同名的类，无法在ASP.NET CORE中使用
Microsoft.AspNetCore.Mvc.Filters.ActionFilterAttribute（正确）
System.Web.Http.Filters.ActionFilterAttribute（错误）)

提供的执行方法

ActionFilterAttribute提供了注解的方式配置Filter，
可以在类和方法上进行设置,
它实现了所有的Filter相关接口，有6种方法提供使用

[AttributeUsage(AttributeTargets.Class | AttributeTargets.Method, AllowMultiple = true, Inherited = true)]
//ActionFilterAttribute实现了多个接口，可以完成
public abstract class ActionFilterAttribute : Attribute, IActionFilter, IFilterMetadata, IAsyncActionFilter, IAsyncResultFilter, IOrderedFilter, IResultFilter
{
    protected ActionFilterAttribute();

    public int Order { get; set; }
	//方法前调用
    public virtual void OnActionExecuted(ActionExecutedContext context);
    //方法后调用
    public virtual void OnActionExecuting(ActionExecutingContext context);
    //方法环绕调用
    [DebuggerStepThrough]
    public virtual Task OnActionExecutionAsync(ActionExecutingContext context, ActionExecutionDelegate next);
    
    //下面则是结果Filter
    public virtual void OnResultExecuted(ResultExecutedContext context);
    public virtual void OnResultExecuting(ResultExecutingContext context);
    [DebuggerStepThrough]
    public virtual Task OnResultExecutionAsync(ResultExecutingContext context, ResultExecutionDelegate next);
}

3.环绕AOP的IAsyncActionFilter异步过滤器

ASP.NET CORE使用
IAsyncActionFilter来完成环绕的AOP
它有两种执行方式

1.同步方式

例：

public class RoundFilter : IAsyncActionFilter
{
   private readonly ILogger<RoundFilter> logger;

   public RoundFilter(ILogger<RoundFilter> logger)
   {
       this.logger = logger;
   }
   //如果你不在方法前加上async关键字则运行依旧是同步的
   //方法自带ActionExcutingContext参数，你也可以从next的Invoke方法中得到
   //ActionExecutedContext来获取返回值
   //由于是同步方法，你必须返回一个Task,如果你选择添加async则不需要返回Task
   public Task OnActionExecutionAsync(ActionExecutingContext context, ActionExecutionDelegate next)
   {
       logger.LogInformation("环绕------------方法执行前");
	   //如果你不调用next方法，则不会执行切入的方法
       next.Invoke();
	   
	   //推荐将Action做为方法执行后直接返回，
       Action action = () => { logger.LogInformation("环绕----------方法执行后"); };

       return Task.Run(action);
      
   }
}

配置：

services.AddMvc(option => {
     option.Filters.Add(typeof(RoundFilter));//也按正常配置即可
 });

接口：

[HttpGet]
//[TestAttributeFilter]
public string ActionFilterTestOne(string paramOne,string paramTwo,string paramThree)
{
    logger.LogInformation("The Action is Executing Now");

    return "ActionFilterTestOne";
}

2.异步方式

异步的方式则更加简单
例：

public class RoundFilter : IAsyncActionFilter
 {
     private readonly ILogger<RoundFilter> logger;

     public RoundFilter(ILogger<RoundFilter> logger)
     {
         this.logger = logger;
     }
	 //
     public async  Task OnActionExecutionAsync(ActionExecutingContext context, ActionExecutionDelegate next)
     {
       
         logger.LogInformation("环绕方法执行前获取参数");

         IDictionary<string, object> ArgsMap = context.ActionArguments;

         foreach (KeyValuePair<string, object> pair in ArgsMap)
             logger.LogInformation("参数名为" + pair.Key + " |参数值为" + pair.Value);
		 //可以运行next方法获取执行方法的ActionExecutedContext对象，从而获取返回值等信息
         ActionExecutedContext actionExecutedContext = await next();

         ObjectResult result = (ObjectResult)actionExecutedContext.Result;

         logger.LogInformation("方法执行后的返回值为" + result.Value.ToString());

     }
 }
