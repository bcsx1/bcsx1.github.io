关于arcgis server创建站点失败的问题解决方案

arcgis server创建站点可能遇到问题解决方法 （虚拟机- win7专业版 arcgis10.4）

    • 1.如下图（Failed to create the site.Failed to configure the server machine'XXXXXX',Server machine'XXXXXX' is not a local server machine.）
    • 2.如下图（Failed to create the site. Failed to create the service ‘System/CachingTools.GPServer’）

（虚拟机- win7专业版 arcgis10.4）

最近在做图形部署，按照部署的操作手册进行配置，安装完arcgis server后，创建站点时，遇到如下创建失败的错误，现将本人解决方法共享如下。

1.如下图（Failed to create the site.Failed to configure the server machine’XXXXXX’,Server machine’XXXXXX’ is not a local server machine.）


解决方法：
百度后，最终将解决方案定位在server的一个配置文件上，文件位置：server安装文件下的这个文件（如图）

进入该文件后，修改以下几个位置的名称为主机名（我是修改为主机名，因为我是在本地做的测试）

修改后保存。
然后重启一下arcgis server的服务，刷新站点创建页面，重新创建，问题解决。

说明：这里网上还有两个其他说法

       （1）除了修改我上面所说的文件外，还要修改这个路径下的一个xml文件。


对此，我的arcgis是安装的10.4的全套，安装后未见有这个文件，网上说如果没有，删除站点文件夹，重启电脑和服务。我尝试了，依然没有。就没有管它，后来也没见到有影响。
（2）在修改config-substitutions文件时，主机名里不能有“-”（中划线）。本人按照他的意思把计算机名改了，然后重新创建站点，发现问题并没有解决，然后我又把计算机名改回来了。
重启了下电脑，重新服务，再创建站点，竟然成功了。
也就是说，上面的（1）（2）应该不是必须的，从我这里来看的话。

2.如下图（Failed to create the site. Failed to create the service ‘System/CachingTools.GPServer’）

网上说是防火墙导致的，参见 https://zhidao.baidu.com/question/561765522739697524.html

本人试着改了一下，问题便解决。按下图路径进入防火墙配置界面，列表里是没有上图说的几个.exe文件的，所以点击“允许允许另一程序”按钮，按照上图说的路径，依次找到这几个exe，添加进去，并保证签名的方框里打上勾。

重新创建站点即可。
