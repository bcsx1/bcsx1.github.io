Python异常处理

什么是异常？

异常就是一个事件，该事件会在程序执行过程中发生，影响了程序的正常执行。一般，在 Python 无法正常处理程序时会发生异常。

异常是Python对象，表示一个错误。当Python脚本发生异常时，我们需要处理它，否则程序会停止终止。

异常处理

捕捉异常可以使用try/except语句。

尝试/排除语句检测语句块的错误，从而使动作异常信息并处理。

如果你不想在发生时发生，那么快结束你的异常程序。

语法：简单的try....except...else的语法：

尝试：
<语句> #运行别代码
除了<名字>：
<语句> #如果在尝试群体引发了'name'异常
除了<名字>，<数据>：
<语句> #如果引发了'name'异常，获得附加的数据
别的：
<语句> #如果没有发生异常

try的工作原理是，当开始一个try语句后，python就在当前程序的上下文中作标记，这样当异常时就可以回到这里，try子句先执行，接下来会发生什么依赖于执行时出现是否出现异常。

  • 如果当尝试后的语句执行时发生异常，蟒蛇就跳回去尝试并执行第一个匹配该异常的异常句，异常处理完毕，控制流就通过整个尝试语句（可以在处理异常时又引发新）的异常）。
  • 如果在尝试后的语句里发生了异常，但没有匹配的除句外，异常将被提交到上层的尝试，或者到最上层（这样将结束程序，并打印默认的错误信息）。
  • 如果在尝试语句执行时没有发生异常，python 将执行其他语句后的语句（如果有其他语句），然后通过整个尝试语句控制流。

实例

   以下是简单的列表访问异常实例

#!/usr/bin/env python
# -*- coding: utf-8 -*-
# Author： WilliamFeng
# datetime： 2021/11/11 11:18 
# File ：异常处理.py
# ide： PyCharm
# Motto：ABC(Always Be Coding)
names = ['{name:bob}','william']
try:
    names[2]
    print(names[2])
except IndexError as e:
    print("列表操作错误", e)
except ImportError as e:
    print("导入包错误",e)

except Exception as e:  # 判断所有的错误，一般不使用，调试代码的时候可以用到
    print("未知错误",e)
else:
    print("一切运行正常")  # 标记代码运行状况
finally:
    print("无论正常异常都会打印本段文字！") # 标记代码运行状况

无论正常异常都会打印本段文字！

触发异常

我们可以使用raise语句自己触发异常

raise 语法格式如下：

引发 [异常 [, args [, 回溯]]]
 [异常[, args [,回溯]]]]

语句中 Exception 是异常的类型（例如，NameError）参数标准异常中任一种，args 是自已提供的异常参数。

一个参数是任选的（在实践中很少使用），如果存在，就是查异常对象。

实例

Python 的内核提供的异常，几乎都是实例化的类，这是一个类的实例的参数。

定义一个异常非常简单，如下所示：
