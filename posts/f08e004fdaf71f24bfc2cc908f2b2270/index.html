2.常见的设置参数

参数包含含义
缓存设置包括key_buffer_size、query_cache_size、innodb_buffer_pool_size等参数用于设置缓存的大小和使用情况。
连接设置包括max_connections、wait_timeout、connect_timeout等参数用于设置连接的数量和超时时间
日志设置包括log_error、slow_query_log、general_log等参数用于设置错误日志、慢查询日志和一般查询日志等。
安全设置包括skip_networking、skip_external_locking、secure_auth等参数用于设置网络连接、外部锁定和安全认证等。
存储设置包括innodb_file_per_table、innodb_file_format、innodb_log_file_size等参数用于设置存储引擎的文件格式、日志大小等。

使用这些参数需要在MySQL配置文件中进行设置，一般为my.cnf文件。可以通过以下方式进行设置：

  1. 找到my.cnf文件所在的位置，一般在/etc/mysql/或/usr/local/mysql/etc/目录下。
  2. 打开my.cnf文件，找到需要设置的参数，修改其值。
  3. 保存my.cnf文件，重启MySQL服务，使设置生效。
    例如，设置key_buffer_size为256M，可以在my.cnf文件中添加以下内容：
[mysqld]
key_buffer_size = 256M

保存文件后，重启MySQL服务即可生效。

3.基础语句

3.1.查询

3.1.1.基础查询

-- 查询整个学生表数据
select * from stdent
-- 查询姓名为小白的数据
SELECT * FROM student WHERE name="小白";

3.1.2.查询在指定区间内数据

-- 从student表中筛选age在20和99之间的，包括起始
SELECT * FROM student WHERE age BETWEEN 20 AND 99;
-- 从student表筛选年龄满足（）内任意一项的数据
SELECT * FROM student WHERE age IN (10,20,30);
-- 从student表中筛选年龄不满足（）内任意一项的数据
SELECT * FROM student WHERE age NOT in (10,20,30);

3.1.3.查询指定条件的数据

-- 从student中筛选成绩不为空的
SELECT * FROM student WHERE score is NOT NULL;
-- and or多条件查询
--从student中筛选性别为女，年龄小于30，并且姓名长度小于3位的数据
SELECT * from student WHERE sex='girl' or (age<30 AND LENGTH(`name`)<>3);

3.1.4.查询指定行数数据

-- 在student表中查询前两行数据
SELECT * FROM student LIMIT 2;

-- 从student表中查询所有，然后从查询结果第二行开始，往后两行的数据
SELECT * FROM student LIMIT 2,2;

-- 在student表查询所有，然后从查询结果偏移两行后，返回1行数据
SELECT * FROM student LIMIT 1 OFFSET 2;

3.1.5.查询数据排序

-- 在student表中查询所有数据，并且按age升序返回，默认asc
SELECT * FROM student ORDER BY age;

-- student表中查询所有数据，并且按age降序返回
SELECT * FROM student ORDER BY age DESC;

3.1.6.去重查询

-- 从student表中筛选所有年龄项，重复的直接去除
SELECT DISTINCT age FROM student;

3.1.7.通配符查询

-- 通配符%，匹配多个单词，通配符搜索只能用于文本字符串
-- 在数据中筛选name列第一个为小，后面为任意数量字符的字符的所有数据
SELECT * FROM student WHERE `name` LIKE "小%";

--通配符_，匹配一个单词
-- 在数据中筛选名字第一个为小，后面带任意一个字符的所有数据
SELECT * FROM student WHERE `name` LIKE "小_";

3.1.8.字段拼接查询

-- 从math表中将score和age列数据相加并起别名为scoreAge，数据罗列出来
SELECT (score+age) as scoreAge FROM math;
-- 从student表中，将name和sex列数据用_连接起来并起别名nameAge，数据罗列出来
SELECT CONCAT(name,"_",age) as nameAge  FROM student ;

3.1.9.子查询（也叫嵌套查询）
子查询就是在一个sql语句中嵌套另一个sql语句，主体内部的sql语句被称为子查询，主体本身sql称为主查询，子查询可以作为主查询的一部分

-- 在student表中首先子查询sex为girl的age列数据，然后主查询满足age列数据的所有name列数据
SELECT name from student WHERE age IN (SELECT age FROM student WHERE sex="girl")

3.1.10.联结查询
联结查询（Join）是指在多个表中根据某些条件将它们连接起来，形成一个新的虚拟表。联结查询是SQL语言中最常用的查询方式之一，它可以将多个表中的数据进行关联，从而得到更加全面和准确的查询结果。

MySQL常见的联结类型含义
内联结
（INNER JOIN)		只返回两个表中匹配的行。
左联结
（LEFT JOIN）		返回左表中的所有行以及右表中匹配的行。
右联结
（RIGHT JOIN）		返回右表中的所有行以及左表中匹配的行。
全外联结
（FULL OUTER JOIN）		返回左表和右表中的所有行，如果没有匹配的行则填充NULL。
自联结
（SELF JOIN）		将一张表看作两张表来进行联结，用于处理树形结构等。
交叉联结
（CROSS JOIN）		返回两个表中的所有行的笛卡尔积，即没有任何条件限制的联结
--将English和chinese内联结起来，只返回满足on后条件的数据，并且两个表的满足条件的列都会返回；
SELECT * FROM english INNER JOIN chinese ON english.score<chinese.score;
--将English和chinese左联结起来，只返回满足on后条件的数据，并且返回English表所有的列，返回chinese满足条件的列
SELECT * FROM english INNER JOIN chinese ON english.score<chinese.score;

demo:在student表中，不知道小白是什么性别，想要在student表中筛选与小白一样性别的人

-- 内联方法，必须起别名，一张表分两张查询，但是会多一对重复列
SELECT * FROM student as s1 INNER JOIN student as s2 WHERE s1.sex=s2.sex AND s2.`name`="小白";
-- 子查询方法，首先查询小白性别，根据子查询结果筛选数据
SELECT * FROM student WHERE sex=(SELECT sex FROM student WHERE `name`="小白");

4.2存储过程

SQL中的存储过程是一组预编译的SQL语句，它们被存储在数据库中，并可以通过一个名称来调用执行。存储过程可以接受参数，可以包含条件语句、循环语句、异常处理等，并且存储过程中可以调用其他的存储过程，可以大大简化复杂的SQL操作，提高数据库的性能和安全性。

4.2.1存储过程的定义
-- 将sql语句语句分割符改为$
DELIMITER $ 
-- 定义一个存储过程，名称为addCh，形参为a,b,类型分别为文本，整型
CREATE PROCEDURE addCh(a VARCHAR(200),b INT)
-- 定义过程体
BEGIN
INSERT INTO chinese VALUES(a,b);
END$
-- 还原sql语句分割符号为默认;
DELIMITER;
4.2.2存储过程的使用
-- 调用存储过程
call addCh("小明",18)
4.2.3存储过程的查询与修改
-- 显示数据库中所有的存储过程
SHOW PROCEDURE STATUS;
4.2.4存储过程的删除
-- 删除存储过程addCh
DROP PROCEDURE addCh;
4.3.事务

事务就是多个sql语句组成的代码块
注意：如果事务不提交，运行后，代码仍然能会执行，并且修改的数据会一直存储在内存当中，所以创建事务一定要结束，否则会可能导致内存泄露。
增（insert into)，删（delete），改（update)这三种操作为事务处理，select不是事务处理，所以不可以回退。create和drop也不是事务处理，所以也不可回退。
常规写法如下：

SET AUTOCOMMIT=0;-- 改变事务提交模式为手动
START TRANSACTION;-- 开启事务
UPDATE chinese SET score=score+1 WHERE `name`="小美";
UPDATE english SET score=score+1 WHERE `name`="小中";
COMMIT;-- 结束事务
4.3.1.常用术语
事务用语含义
transation事务
rollback回退
commit提交
savepoint保留点，用于回退
rollback to XX回退到保留点
4.3.2.事务特征
特征含义
原子性整个事务中，只要有一个语句执行错误，就会整体回滚，mysql中只有innoDB引擎支持事务，并且会自动回滚，如果想要指定回滚的地方，定义保留点即可
一致性事务开始前，结束后，数据库完整状态不被破坏。确保输入规范
隔离性多个事务操作同一张表时，可以使用 事务隔离防止数据出错
4.3.3.事务提交与隐式/显式事务
术语解释
事务提交模式mysql事务提交模式有两种，一种为自动，一种为手动
隐式事务无明显的开始结束标记， mysql默认所有语句均为隐式事务，也就是每个增删改语句都是一个事务
显式事务指定开始和结束标记，开始用strat/begin transation，结束用commit/rollback
4.3.4.事务中常用代码
代码块含义使用位置
select @@autocommit查询当前事务提交模式任意
set autocommit=0禁用事务自动提交功能自定义显式事务开始前
set autocommit=1开启事务自动提交功能自定义显式事务结束后
`start/begin transation·开启一个事务用于定义显式事务的开始
commit/rollback提交/回滚事务用于结束一个事务
4.3.5常见的数据错乱
名称解释
脏读事务A，B，A读取了B更新的表数据，但是B没有commit提交，如果B回滚，那A中读取数据为内存中的数据，数据无效。
不可重复读事务A，B，A读取了表数据，但是在二次次读取前，B事务把表数据更新并提交了，结果导致A再次读取数据与第一次不一致
幻读事务A，B，A都去了数据，但是在二次读取前，B事务删除或者新增了数据，导致A两次读取数据不一致
4.3.6事务隔离（解决数据错乱)
隔离级别隔离名称含义
0read uncommited
(读未提交)		允许当前事务读取被其他事务提交的变更
1read commited
读已经提交数据		只允许当前事务读取被其他事务已经提交的数据
2repeatable read
(可重复读)		在当前事务持续期间，其他任何事务禁止操作此字段
3serializable串行化
