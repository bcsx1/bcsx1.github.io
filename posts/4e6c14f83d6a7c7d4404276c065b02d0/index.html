（个人理解，一个字节为8bit，可存放最高八位的二进制数，如11111111，且第一位为符号位，即11111111为-127,01111111为127，且由于补码的存在，正数的补码是他自身，负数的补码是符号位以外取反+1，所以，正数的最大补码（正数的补码是本身）：01111111，即127.负数的最大补码10000000，十进制：-1+(-127)=-128 二进制原码：10000001+11111111 二进制反码：11111110+10000000二进制补码：11111111+10000001=10000000[只保留8位]）

五、整数类型

1.int为四个字节，JAVA的整型常量默认为int，声明long型常量后需要加入l或者L

                int a=1;可以运行而int a=1L;将会报错，需要改成long a=1L;

2.JAVA中变量常声明为int

3.bit是计算机最小储存单位，byte计算机中最基础的储存单元，1byte=1bit

六、浮点类型

1.float单精度浮点型，占用4个字节，double双精度浮点型，占用8个字节

2.浮点数在机器中储存形式为 浮点数=符号位+指数位+尾数位（尾数位可被省略）

3.JAVA的浮点型常量默认为double，需要声明为float时，需在后面加上f或者F

                float n1 = 1.1；

        会报错，默认1.1是八个字节，需要在1.1后加F，即

                float n1 = 1.1f；

        无报错

对于以下的代码

                double = 1.1f；

        正常，无报错

4.十进制小数的形式；5.12           512.0f           .512

科学计数法：5.12e2 为 5.12*10^2         5.12E-2 为 5.12/10^2

5.通常情况下，应该使用double型，因为它比float更精确

6.浮点型的使用陷阱

                double num11 = 2.7；

                double num12 = 8.1 /  3 ；

        输出结果 num11 = 2.7； num12 = 2.699999999999999997；

        原因分析，计算机理解为8.10000000000 / 3 计算机以精度的形式储存

所以，在对运算结果是小数的进行相等判断时，要小心。应该与两个差值的绝对值，在某个精度范围内进行判断
