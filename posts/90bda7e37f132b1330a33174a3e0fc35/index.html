2、作用域

作用域分全局作用域和函数作用域，作用域有上下级关系就看函数是在哪个作用域下创建的。

作用域最大的用处就是隔离变量，不同作用域下同名变量不会有冲突。
作用域链：如果在当前作用域没查到值，就会向上级作用域查询直到全局作用域，这个查找的过程形成了作用域链。

 var a = 10;     // 全局作用域
 function first(){    // first作用域
 	var b = 20;
 	fucntion last(){   // last作用域
 		var c = 30;
 	}
 }
3、原型原型链

原型分为显示原型(prototype)和隐式原型（proto），构造函数的prototype和实例的__proto__都是指向原型对象。
constructor和prototype是成对的，构造函数prototype和原型对象constructor互相指向对方。

俗话说（proto）的路径就叫原型链，构造函数A的prototype指向原型对象，既然对象说明构造函数A的原型对象是构造函数Object的实例，也说明了构造函数A的原型对象的__proto__指向Obeject原型，Obeject原型的__proto__是指向null，是原型链的终点

原型继承就是实例可以使用构造函数的prototype中的方法。

function Show() {  //构造函数
  this.name = "run";
}
Show.prototype.sayName = function () {  //原型对象添加方法
  console.log(this.name)
}
const person = new Show()   //实例
person.sayName()   //使用构造函数prototype的中的方法

instanceof 可以判断B的prototype是否在A的原型链上。

4、闭包（何为闭包、优点与缺点）

闭包就是能够读取其他函数内部变量的函数，利用闭包可以突破作用域链。
闭包的作用是 延伸量的作用范围
优点是 使外部能访问到局部的东西， 避免全局变量导致的全局变量污染。
缺点是 不会销毁变量，容易造成内存泄漏的问题（栈溢出）。
如何解决闭包造成的内存泄漏：使用结束后手动清除或者置空变量或者方法。
应用场景：

//3秒钟之后,打印三次Doris,立即执行函数也成为小闭包
const name = 'Doris'
for (var i = 0; i < 3; i++) {
  (function (i) {
    setTimeout(function () {
      console.log(name);
    }, 3000)
  })(i);
}

6、this指向

指向什么取决与函数在哪里调用
1）构造函数中的this指向 当前的实例
2）全局环境下的this指向 window
3）对象中的this指向 调用这些方法的对象
4）箭头函数没有this 它会绑定最近的非箭头函数作用域中的this
5）函数内的this指向 严格模式下this指向undefined，非严格模式下this指向window

7、call、apply、bind的区别

call、apply、bind的作用都是改变函数运行时this的指向。
1）call 和 apply 改变 this 指向时，函数会立即执行，bind 不会，而是返回一个改变了上下文 this 后的函数，便于稍后调用。
2）call 、apply和bind 第一个参数都是绑定给this的值，第一个参数为null、undefined的时候，默认指向window。第二个参数 call方法为参数列表，apply和bind方法为参数数组。

8、Promise 可阅读文章

Promise 是异步编程的一种解决方案。promise有三种状态： pending(等待态)，fulfiled(成功态)，rejected(失败态)；状态一旦改变，就不会再变。创造promise实例后，它会立即执行。

有什么用？
1、解决了回调地狱问题。
2、代码可读性提高。
3、它的状态只会改变一次且不可逆
对于async/await，它用同步方式执行异步代码。

10、深浅拷贝的区别
  • 深拷贝层层拷贝，浅拷贝只拷贝第一层，深层只是引用。
  • 在深拷贝中，新对象中的更改不会影响原始对象，而在浅拷贝中，新对象中的更改，原始对象也会跟着改。
  • 在深拷贝中，原始对象不与新对象共享相同的属性，而在浅拷贝中，他们具有相同的属性。
11、var、let、const的区别

var 存在变量提升，可重复声明，声明变量可改
let 是块级作用域，没有变量提升，不可重复声明，声明变量可改
const 是块级作用域，没有变量提升，不可重复声明，声明的变量不可改且必须初始化。

12、 localStorage 、sessionStorage 、cookies 有什么区别?

1）localStorage:以键值对的方式存储 储存时间没有限制 永久生效 除非自己删除记录
2）sessionStorage：当页面关闭后被清理与其他相比不能同源窗口共享 是会话级别的存储方式
3）cookies 数据不能超过4k，同时因为每次http请求都会携带cookie，所有cookie只适合保存很小的数据。（比如记账本，容量有限）
4）session数据存在服务器内存中，但是通过cookie传递id，会有少量的传送流量。（比如会员卡，只需要id就可以查询）

sessionStorage 和session 两者完全不是依赖关系或者是相同关系。
session存储于服务器，而sessionStorage存储于本地。

13、什么是防抖？什么是节流？

防抖：当事件被触发后，延迟n秒后再执行回调，如果在这n秒内事件又被触发，则重新计时。（例如：王者回城）
节流：可以减少一段时间内事件的触发频率，周期操作五秒，在五秒内连续点击射击，操作无效（例如：王者使用技能的倒计时）

14、inNaN 与 Number.isNaN的区别
  1. isNaN：除了判断NaN为true，还会把不能转成数字的判断为true，例如’xxx’
  2. Number.isNaN：只有判断NaN时为true，其余情况都为false
15、valueOf 与 toString

toSring 和 valueOf 两者都是对象的方法。前者返回对象的字符串，后者返回对象的原始值，如果没有原始值则返回对象本身。

  1. 转换字符串优先调用 toString ，转换数字优先调用 valueOf
  2. 正常情况下，优先调用 toString
  3. 运算操作符情况下优先调用 valueOf
16、讲一讲JavaScript的装箱和拆箱？

装箱：把基本数据类型转化为对应的引用数据类型的操作

const s1 = 'helloWorld'
const index = s1.indexOf('o')
console.log(index)  // 4

拆箱：将引用数据类型转化为对应的基本数据类型的操作（通过valueOf和toString方法）

var objNum = new Number(123);  
var objStr = new String("123");
 
console.log( typeof objNum ); //object
console.log( typeof objStr ); //object 

console.log( typeof objNum.valueOf() ); //number
console.log( typeof objStr.valueOf() ); //string
console.log( typeof objNum.toString() ); // string 
console.log( typeof objStr.toString() ); // string
17、事件循环

事件循环：一句话概括就是入栈到出栈的循环。
循环过程：

  1. 所有同步任务都在主线程上依次执行，形成一个执行栈，异步任务则放入一个任务队列
  2. 当执行栈中任务执行完，再去检查微任务队列里的微任务是否为空，有就执行，如果执行微任务过程中又遇到微任务，就添加到微任务队列末尾继续执行，把微任务全部执行完
  3. 微任务执行完后，再到任务队列检查宏任务是否为空，有就取出最先进入队列的宏任务压入执行栈中执行其同步代码
  4. 然后回到第2步执行该宏任务中的微任务，如此反复，直到宏任务也执行完，如此循环

JS的执行机制：先同步后异步，先微任务(VIP客户)后宏任务(普通客户)。
相当于银行排队，要等VIP客户运行完才到普通客户，如果来了新VIP客户可以插队到普通客户最前面。
