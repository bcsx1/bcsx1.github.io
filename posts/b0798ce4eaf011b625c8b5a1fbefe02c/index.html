Mybatis-Dao 层 Mapper 接口

Mapper 接口开发方法只需要程序员编写 Mapper 接口（相当于 Dao 接口），由 Mybatis 框架根据接口定义创建接口的动态代理对象. 使用 session.getMapper(接口.class);获得代理对象Mapper 接口开发需要遵循以下规范：

1、 Mapper.xml 文件中的 namespace 与 mapper 接口的类路径相同。

<mapper namespace="com.qn.mybatisPro.mapper.AdminMapper">
<!--对应接口地址-->

2、 Mapper 接口方法名和 Mapper.xml 中定义的每个 statement 的 id 相同

public interface AdminMapper {
  /*
        sql写在xml文件中,我们不能直接访问,
        mybatis让我们在业务代码可以定义接口,在接口中定义操作的方法.
        方法定义是由要求的:
          方法名 与 xml中某个标签id相同.  参数类型也是一致,返回值类型也是一致
      */
  void saveAdmin(Admin admin);
}

3、 Mapper 接口方法的输入参数类型和 mapper.xml 中定义的每个 sql 的parameterType 的类型相同

4、 Mapper 接口方法的输出参数类型和 mapper.xml 中定义的每个 sql 的resultType 的类型相同

随着配置信息的加多我们将相关的数据库信息新建一个config.properties用来储存数据库信息

driverClassName=com.mysql.cj.jdbc.Driver
url=jdbc:mysql://127.0.0.1:3306/mybatis_db?characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
userName=root
passWord=root

同时在mybatis-config.xml中添加配置

<!-- 引入存放值的文件--> <properties resource="config.properties"></properties>

将之前的配置进行修改

  <environments default="development">
        <!--配置基本数据库信息-->
        <environment id="development">
            <!--配置事务管理类型,使用JDBC事务管理-->
            <transactionManager type="JDBC"></transactionManager>
            <!--数据源-->
            <!--type="UNPOOLED"不需要使用数据库链接池-->
            <dataSource type="UNPOOLED">
                <property name="driver" value="${driverClassName}" />
                <property name="url" value="${url}"/>
                <property name="username" value="${userName}" />
                <property name="password" value="${passWord}"/>
            </dataSource>
        </environment>
    </environments>

Mybatis 日志

Mybatis 内置的日志工厂提供日志功能，具体的日志实现有以下几种方式：

SLF4J | LOG4J | LOG4J2 JDK_LOGGING

COMMONS_LOGGING STDOUT_LOGGING NO_LOGGING

具体选择哪个日志实现由 MyBatis 的内置日志工厂确定。它会使用最先找到的

  <settings>
        <setting name="logImpl" value="LOG4J"/>
    </settings>

注意添加的位置放置在environments之前,不然无法正常加载

参数传递

简单的参数形式不需要使用 parameterType 参数定义,例如:User
selectUsers(int id);

多个参数使用@Param(“id”)绑定User selectUsers(@Param(“id”)int id,@Param(“name”)String name)

  <!--  为类定义别名-->
    <typeAliases>
        <typeAlias type="com.qn.mybatisPro.model.Admin" alias="Admin"></typeAlias>
    </typeAliases>

在AdminMapper.xml中就可以简写类的地址与简称了

  <insert id="saveAdmin" parameterType="Admin">/*映射地址就可以用简称了*/
        insert into admin(account,password,sex)values(#{account},#{password},#{sex})
    </insert>

简化代码

为了简化代码量我们将读取mybatis核心配置部分的代码封装到一个类中,并创建获取sqlsession的方法

package util;

import org.apache.ibatis.io.Resources;
import org.apache.ibatis.session.SqlSession;
import org.apache.ibatis.session.SqlSessionFactory;
import org.apache.ibatis.session.SqlSessionFactoryBuilder;
import java.io.IOException;
import java.io.Reader;

public class MybatisUtil {
    static SqlSessionFactory sqlSessionFactory = null;
    //静态代码块,在类加载时执行,只执行一次
     static {
         Reader reader = null;
         try {
             reader = Resources.getResourceAsReader("mybatis-config.xml");
         } catch (IOException e) {
             e.printStackTrace();
         }
          sqlSessionFactory = new SqlSessionFactoryBuilder().build(reader);
     }
     //获得SqlSession
     public static SqlSession getSqlSession(){
         return sqlSessionFactory.openSession();
     }

}
