前言

多线程是java语言中最重要的一个知识点，重要到什么程度呢，可以说这是大家以后找工作面试时必定问的知识点。各种锁，各种线程并发与并行，常常让大家感觉脑瓜子疼。下面让我带领大家学习一些多线程的基础关键知识，让大家明白什么是多线程，并且会运用多线程，开启多线程的启蒙课。（注：本文只是多线程知识的基础入门学习，多线程的知识内容要写的话完全可以写成一本书）

一，线程是什么

说到程序，想必每个人都听说过，但是如果我问你：程序是什么？你该如何回答呢？想必一时半会表达不出来吧。所谓的程序用我们程序员的话来说就是一段静态的代码，静态对象。而正在运行的程序，或者说程序的一次执行过程，这种动态的过程就叫做进程。接下来让我们把进程进一步的划分，就是线程了。
线程可以说是进程的进一步细化，指的是程序内部的一条执行路径，一个进程中的多个线程共享相同的内存单元/内存地址空间—>简单来说就是他们从同一堆中分配对象，可以访问相同的变量和对象。

那么java程序最少就几个线程呢？两个，分别是：

  1. JVM GC垃圾回收机制（守护线程/后台线程）
  2. main 线程/main 线程组

二，创建新的线程方式

其实我们经常用的main()方法，他就是一个线程，准确的来说他是主线程

方式一：

  1. 写一个类继承Thread类，并且重写Thread类的run()方法
public class MyThread extends Thread{
    @Override
    public void run () {
        System.out.println ("一个新的线程启动了");
    }
}
  1. 创建一个线程对象，并且这个对象调用start()方法
public class MyThread extends Thread{
    @Override
    public void run() {
        System.out.println("一个新的线程启动了");
    }

    public static void main(String[] args) {
        MyThread myThread = new MyThread();
        myThread.start();  
    }
}

为什么调用start()方法，会运行run()方法

看到这里，你可能会纳闷：为什么调用的是start()方法，但是run()方法却被运行了。我门可以看一下start()方法的源码说明，可以知道start()就是一个开关，当我们打开它，JVM会自动调用run()方法。
那么我们可以不以直接调用run()方法呢？答案肯定是不可以，如果我们直接调用他，实际运行这个方法的就是主线程main了。我们可以写个代码验证一下

public class MyThread extends Thread{
    @Override
    public void run() {
        System.out.println(Thread.currentThread()+"线程启动了");
    }

    public static void main(String[] args) {
        MyThread myThread = new MyThread();
        myThread.run();   //直接调用run()方法
    } }

**currentThread()**是Thread类的一个静态方法，返回运行当前代码的线程

可以看到其实是主线程执行的方法

3，多线程运行分析

让我们先创建多条线程让他们运行起来，看看运行状态是怎么样的

public class MyThread implements Runnable {
    int num = 100;
    @Override
    public void run() {
        while (num > 0) {
                System.out.println(Thread.currentThread().getName() + "票号" + num + "卖出去了");
                num--;
        }
    }
    public static void main(String[] args) {
        MyThread myThread = new MyThread();
        Thread thread1 = new Thread(myThread, "线程一");
        Thread thread2 = new Thread(myThread, "线程二");
        Thread thread3 = new Thread(myThread, "线程三");
        thread1.start();
        thread2.start();
        thread3.start();

    }
}

控制台输出代码部分如下

分析：程序是由main方法开始执行的，首先运行for循环(1)，根据控制台输出名字可知这是主线程执行的代码。出了循环体以后，并不是按照代码顺序执行，而是三条线程同时开启，并且“抢占式运行”，即谁获得cpu的执行权，谁就运行，但是可能运行着运行着又被别的线程抢走这个执行权了。

4，关键—》同步方法

多条线程同时卖票问题

生活中大家都应该遇到这种情景:比如说买票，大家可以同时在各类线上app，线下窗口买票，无论我们从哪里买，他票的总数其实是一定的，这其实就是一个多线程问题，多个渠道同时卖票，有效的提高了人民买票的效率。下面我们写个代码来更加清晰的了解他

public class MyThread implements Runnable {
    int num = 100;
    @Override
    public void run() {
  for  {
    System.out.println(Thread.currentThread().getName() +"票号"+ num+"卖出去了");
    num--;
}
    }

    public static void main(String[] args) {
        MyThread myThread = new MyThread();
        Thread thread1 = new Thread(myThread, "线程一");
        Thread thread2 = new Thread(myThread, "线程二");
        Thread thread3 = new Thread(myThread, "线程三");

        thread1.start();
        thread2.start();
        thread3.start();
    }
}

这里我们传入Thread构造方法的对象是同一个，那么这三个线程就共用同一个票数num，但是细心的朋友可能会发现一个问题

居然票号为100的票卖了3次，为什么会这样呢，我们从多线程的关键：抢占式运行来分析，首先程序运行，线程一首先抢到了cpu的执行权，它开始执行run()方法，执行第一次循环，但是他执行到一半，就是刚刚执行完System.out.println(Thread.currentThread().getName() +“票号”+ num+“卖出去了”);
这段代码，还没来得及让票数减一，线程二抢到了cpu的执行权，此时线程二得到的num还是没有自减的，仍然是100，然后正当他要执行num–的时候，线程三又把cpu执行权抢占过来了，任然输出100…

synchronized同步监视器

首先我们需要明白synchronized是干嘛用的，用synchronized修饰一段代码，等于给这段代码上了一把锁，上锁以后，别的线程无法抢占cpu资源，当执行完这段代码，锁才会打开，这样再回到刚刚的卖票问题，解决方式就明了了：只需要加上一把锁，
代码如下

public class MyThread implements Runnable {
    int num = 100;
    @Override
    public void run() {
        while (true) {           
            synchronized (this) {        
                if (num > 0) {
                    System.out.println(Thread.currentThread().getName() + "票号" + num + "卖出去了");
                    num--;
                }
            }
        }
    }
    public static void main(String[] args) {
        MyThread myThread = new MyThread();
        Thread thread1 = new Thread(myThread, "线程一");
        Thread thread2 = new Thread(myThread, "线程二");
        Thread thread3 = new Thread(myThread, "线程三");
        thread1.start();
        thread2.start();
        thread3.start();
    }
}

注意：this称为锁

锁对象的选择

任何类的实例对象都可以充当锁，但是锁也有一些限制，比如：

  1. 锁必须保证唯一性，即这些线程的锁是同一个
  2. 通常一个类内会设置一个独立的变量作为锁对象使用
  3. 保证类不得用于锁对象的使用
  4. 当前类也可以充当锁（万物皆对象，即类也是对象，Class类的对象实例）

方法锁

通过上面的例子我们知道，锁可以锁住代码块，理所当然，他也能锁住方法，叫做方法锁，来吧，让我们换一种方式来实现上面的例子，

public class MyThread implements Runnable {
    int num = 100;
    @Override
    public void run() {
        while (true) {       //run()方法调用buy()
            this.buy();
        }
    }
    public synchronized void buy () {         //使用synchronize把这个方法锁住，锁对象为调用该方法的对象，即myThread
        if (num > 0) {
            System.out.println(Thread.currentThread().getName() + "票号" + num + "卖出去了");
            num--;
                }
    }

    public static void main(String[] args) {
        MyThread myThread = new MyThread();
        Thread thread1 = new Thread(myThread, "线程一");
        Thread thread2 = new Thread(myThread, "线程二");
        Thread thread3 = new Thread(myThread, "线程三");
        thread1.start();
        thread2.start();
        thread3.start();
    }
}

总结

使用synchronized(同步监视器)，俗称锁，任何类的实例对象都可以充当锁
同步的方式，解决了线程的安全问题—好处
操作同步代码时，只能有一个线程参与，其他线程等待。相当于是一个单线程的过程，效率低-劣势

五，线程生命周期

任何事物都有生命周期，当然线程也不例外，他的生命周期可以分为五个阶段

  1. 新建—》当一个Thread类或其子类的对象被生命创建是，线程处于新建状态
  2. 就绪—》处于新建状态的线程被start()后，将进入线程队列等待CPU时间片，此时他已经具备了运行的条件，只是还没有分配到CPU资源
  3. 运行—》当就绪的线程被调度并且获得CPU资源时，便进入运行状态，run()方法定义了线程的操作和功能
  4. 阻塞—》在某种特殊情况下，被人为挂起或执行输入输出操作时，让出CPU并临时中止自己的执行，进入阻塞状态
  5. 死亡—》线程完成了他的全部工作或线程被提前强制性的终止或出现异常导致结束

六，多线程操作相关方法

构造方法

Thread();

Thread(Runnable target);
以 Runnable 接口实现类为当前线程的执行目标，new 关键字 + 构造方法实例化线程对象

Thread(Runnable target, String threadName);
以 Runnable 接口实现类为当前线程的执行目标，同时明确当前线程的名字，
new 关键字 + 构造方法实例化线程对象

成员方法

String getName();
void setNane(String name);
针对线程获取和设置名称

get/setPriority():获取/设置当前线程优先级
public final static int MAX_PRIORITY = 10; 全局常量
NORM_PRIORITY=5
MIN_PRIORITY=1

优先级范围为1-10，如果cpu非常繁忙时，优先级越高的线程获得更多的时间片，但是cpu空闲时，设置优先级几乎没有任何作用。高优先级抢占低优先级的cup使用权是从概率上来讲的

void setDaemon(boolean flag);
boolean isDaemon();
设置和判断当前线程是否为守护线程
flag 为 true 表示守护线程

public static Thread currentThread();
在哪一个线程代码中执行，获取当前线程代码对应的线程对象。

public static void sleep(int ms)
当前线程阻塞millis毫秒 （1s=1k毫秒），时间结束了也不一定会执行当前线程
自带捕捉异常，需要try catch捕获异常，虽然线程阻塞，但是不会释放当前锁 。

void wait();
阻塞线程，通过锁对象调用，在哪一个代码中执行，当前线程进入进入无限等待状态

void wait(long timeout);通过锁对象调用，在哪一个线程代码中执行，当前线程进入计时等待状态，可以在指定休眠之后，或者被notify和notifyAll方法唤醒

notify();
通过锁对象调用，随机唤醒一个通过当前锁对象睡眠的线程对象。

notifyAll();
通过锁对象调用，唤醒所有通过当前锁对象睡眠的线程对象。

**yield()😗*释放当前cpu的执行权，但是可能还是这个线程执行，又被他抢到执行权

join()：线程a中调用线程b的join（）方法，线程a会变成阻塞状态，必须先执行完线程b，阻塞状态才会取消
