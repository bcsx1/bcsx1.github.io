springMVC获取请求参数

服务器获取请求参数的类型有以下四种：
1、基本类型
2、POJO(没有业务逻辑的简单对象)类型
3、数组类型
4、集合类型

一、基本类型

业务方法的参数名与请求参数的name一致，参数值会自动匹配。
业务方法如下：

	@RequestMapping("/testControll9")
    @ResponseBody /* 告诉springmvc函数要返回的是一个普通数据，而非要跳转的页面名,这里无返回值，
     即不做处理,如果不添加该注解，服务器会找不到要返回的页面 */
    public void controllTest9(String name, int age) {
        System.out.println("name:" + name + " age:" + age);
    }

在浏览器输入以下地址访问该业务函数：
http://localhost:8089/user/testControll9?name=shen&age=19（tips:地址中带有user目录，是因为我在业务模块类头上添加了@RequestMapping("/user")，类头没有添加该mapping的，地址中不需要添加user目录）
此时springmvc会自动将地址中的name和age与业务方法参数中的name和age进行匹配赋值。

二、POJO类型

业务方法中的POJO参数的属性名与请求参数的name一致，参数会自动匹配映射。

使用方法如下：
1、首先定义一个POJO类 UserInfo

public class UserInfo {
    private String name;
    private int age;

    public void setName(String name) {
        this.name = name;
    }

    public void setAge(int age) {
        this.age = age;
    }

    public String getName() {
        return name;
    }

    public int getAge() {
        return age;
    }

    @Override
    public String toString() {
        return "UserInfo{" +
                "name='" + name + '\'' +
                ", age=" + age +
                '}';
    }
}

2、业务方法如下：

    @RequestMapping("/testControll10")
    @ResponseBody
    public void controllTest10(UserInfo user) {
        System.out.println(user);
    }

在浏览器输入以下地址访问该业务函数：
http://localhost:8089/user/testControll9?name=lisi&age=10
此时springmvc会自动将地址中的name和age与业务方法参数中的UserInfo的属性名进行匹配赋值。

三、数组类型

业务方法中数组名称与请求参数name一致，参数会自动匹配映射
业务方法如下：

	@RequestMapping("/testControll11")
    @ResponseBody
    public void controllTest11(String[] addr) {
        System.out.println(Arrays.asList(addr));
    }

在浏览器输入以下地址访问该业务函数：
http://localhost:8089/user/testControll11?addr=beijing&addr=shanghai&addr=shenzhen
此时springmvc会自动将地址中的addr与业务方法参数中的addr进行匹配赋值。

四、集合类型

获取集合类型请求数据
第一种情况：例如需要获取List<UserInfo>请求参数
需要先封装一个VO(Value Object)值对象,该对象内包含一个私有变量List<UserInfo>，以及该变量的get/set/toString方法如下：

public class VO {
    private List<UserInfo> userInfoList;

    public List<UserInfo> getUserInfoList() {
        return userInfoList;
    }

    public void setUserInfoList(List<UserInfo> userInfoList) {
        this.userInfoList = userInfoList;
    }

    @Override
    public String toString() {
        return "VO{" +
                "userInfoList=" + userInfoList +
                '}';
    }
}

然后在voForm.jsp表单中以如下方式提交数据

<%@ page contentType="text/html;charset=UTF-8" language="java" isELIgnored="false" %>
<html>
<head>
    <title>Title</title>
</head>
<body>
    <form action="${pageContext.request.contextPath}/user/testControll12" method="post">
    	<%-- name值为VO对象中封装的集合中对象的属性名 --%>
        第一个UserInfo对象的name<input type="text" name="userInfoList[0].name"/><br/>
        第一个UserInfo对象的age<input type="text" name="userInfoList[0].age"/><br/>
        第二个UserInfo对象的name<input type="text" name="userInfoList[1].name"/><br/>
        第二个UserInfo对象的age<input type="text" name="userInfoList[1].age"/><br/>
        <input type="submit">
    </form>
</body>
</html>

业务方法如下：

    @RequestMapping("/testControll12")
    @ResponseBody
    public void controllTest12(VO vo) {
        System.out.println(vo);
    }

在浏览器输入以下地址访问jsp表单页面：
http://localhost:8089/jsp/voForm.jsp

在表单中填入数据，点击提交，发送请求给业务处理方法。
此时springmvc会自动将表单中的userInfoList与业务方法参数中的VO的属性名userInfoList进行匹配赋值。

获取集合类型请求数据
第二种情况：当使用ajax提交请求时，指定contentType为json，且在业务方法参数位置使用@RequestBody可以直接接受集合数据，无需封装对象。

1、由于需要使用ajax,所以要先导入jquery-3.5.1.js包到工程下,工程目录如下

2、创建ajax.jsp负责发送请求，代码如下

<%@ page contentType="text/html;charset=UTF-8" language="java" isELIgnored="false" %>
<html>
<head>
    <title>Title</title>
    <script src="/jquery/jquery-3.5.1.js"></script>
    <script>
        var userInfoList = new Array();
        userInfoList.push({name:"zhangsan",age:18});
        userInfoList.push({name:"lisi",age:10});

        $.ajax({
            type:"POST",
            url:"http://localhost:8089/user/testControll13",
            data:JSON.stringify(userInfoList),
            contentType:"application/json;charset=utf-8"
        });
    </script>
</head>
<body>
</body>
</html>

3、业务方法如下

@RequestMapping("/testControll13")
    @ResponseBody
    public void controllTest13(@RequestBody List<UserInfo> userInfo) {
        System.out.println(userInfo);
    }

在浏览器输入以下地址访问该业务函数：
http://localhost:8089/jsp/ajax.jsp访问jsp页面，jsp会在加载后去访问http://localhost:8089/user/testControll13并将数据userInfoList以json格式一起发送给服务器
此时springmvc会自动将收到的json数据解析并与业务方法参数中的userInfo参数进行匹配赋值。
