2.元Annotation

在 java.lang.annotation 包下

用于修饰其他的注解

@Retention

指定被修饰注解的生效时期（编译、类加载、运行），默认为类加载时期

@Documented
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.ANNOTATION_TYPE)
public @interface Retention {
    /**
     * Returns the retention policy.
     * @return the retention policy
     */
    RetentionPolicy value();
}

public enum RetentionPolicy {
    /**
     * Annotations are to be discarded by the compiler.
     */
    SOURCE,

    /**
     * Annotations are to be recorded in the class file by the compiler
     * but need not be retained by the VM at run time.  This is the default
     * behavior.
     */
    CLASS,

    /**
     * Annotations are to be recorded in the class file by the compiler and
     * retained by the VM at run time, so they may be read reflectively.
     *
     * @see java.lang.reflect.AnnotatedElement
     */
    RUNTIME
}

设为 SOURCE 和 CLASS 时，程序需要继承AbstractProcessor类并重写其process方法

设为 RUNTIME 则旨在顺利配合发射机制

@Target

指定被修饰注解的作用对象

@Documented
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.ANNOTATION_TYPE)
public @interface Target {
    /**
     * Returns an array of the kinds of elements an annotation type
     * can be applied to.
     * @return an array of the kinds of elements an annotation type
     * can be applied to
     */
    ElementType[] value();
}

public enum ElementType {
    /** Class, interface (including annotation type), or enum declaration */
    TYPE,

    /** Field declaration (includes enum constants) */
    FIELD,

    /** Method declaration */
    METHOD,

    /** Formal parameter declaration */
    PARAMETER,

    /** Constructor declaration */
    CONSTRUCTOR,

    /** Local variable declaration */
    LOCAL_VARIABLE,

    /** Annotation type declaration */
    ANNOTATION_TYPE,

    /** Package declaration */
    PACKAGE,

    /**
     * Type parameter declaration
     *
     * @since 1.8
     */
    TYPE_PARAMETER,

    /**
     * Use of a type
     *
     * @since 1.8
     */
    TYPE_USE
}

Q：如何将注解上信息注入到方法中？

通过反射机制

1.反射出该类的方法

2.通过方法得到注解上具体的信息

3.将注解上的信息注入到方法上

 		//反射出该类的方法
        Class aClass = Demo2.class;
        Method method = aClass.getMethod("add", String.class, int.class);

        //通过该方法得到注解上的具体信息
        MyAnnotation annotation = method.getAnnotation(MyAnnotation.class);
        String username = annotation.username();
        int age = annotation.age();

        //将注解上的信息注入到方法上
        Object o = aClass.newInstance();
        method.invoke(o, username, age);

作用

1.让编译器检查代码

2.将数据注入方法、成员变量、类中

继承/实现泛型类/接口

/*
    把泛型定义在接口上
 */
public interface Inter<T> {
    public abstract void show(T t);

}

情况一：

/**
 * 子类明确泛型类的类型参数变量:
 */
public class InterImpl implements Inter<String> {
    @Override
    public void show(String s) {
        System.out.println(s);
    }
}

public static void main(String[] args) {
	Inter<String> i = new InterImpl();
	i.show("hello");
}

情况二：

/**
 * 子类不明确泛型类的类型参数变量:
 *      实现类也要定义出<T>类型的
 *
 */
public class InterImpl<T> implements Inter<T> {

    @Override
    public void show(T t) {
        System.out.println(t);

    }
}

public static void main(String[] args) {
	Inter<String> ii = new InterImpl<>();
    ii.show("100");
}

泛型擦除

泛型是提供给javac编译器使用的，它用于限定集合的输入类型，让编译器在源代码级别上，挡住向集合中插入的非法数据

但编译器编译完带有泛型的java程序后，为使运行效率不受到影响，生成的class文件中将不再带有泛型信息，这个过程称之为“擦除”

由上图可知，无论是ArrayList<String>还是ArrayList<Integer>，编译后都将其视为ArrayList类型

NIO

始于JDK1.4，目的在于提高速度

IO是面向流的处理，I/O 系统一次一个字节地处理数据

NIO是面向块(缓冲区)的处理，I/O 系统以块的形式处理数据

组成

自己的理解：数据的载体是buffer，buffer的载体是channel

Ⅰ.buffer

缓冲区，类似货车

核心方法

写数据：put()

读数据：get()

核心变量
1.Capacity（容量）

缓冲区所能容纳数据元素的最大数量

2.Limit（上界）

缓冲区现有数据量

3.Position（位置）

下一次读写的位置

4.Mark（标记）

上一次读写的位置

模式切换

调用 flip() 可切换为读模式

调用 clear() 后可回到写模式

缓冲区类型
直接缓冲区

默认情况

直接缓冲区不需要经过copy阶段，内存直接映射到文件

非直接缓冲区

非直接缓冲区需要经过一个copy阶段（内核空间与用户空间）

Ⅱ.Channel

管道，类似公路

Ⅲ.Selector

选择器

对比

传统IO的流是单向的，而NIO的Channel是双向的
