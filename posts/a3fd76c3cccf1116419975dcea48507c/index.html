一、实验题目

编写程序，实现FOIL（First Order Inductive Learner）算法，对如下给定的知识图谱和目标谓词进行规则学习，并得到新的以目标谓词为关系的事实（用一阶逻辑表示）。

(1)、给定的知识图谱所有三元组的一阶逻辑如下（测例1）：

Father(Jack,Dell)
Father(Dell,Stephen)
Grandfather(Jack,Stephen)
Father(Dell,Seth)

目标谓词（祖父关系）

Grandfather(x,y)

(2)、给定的知识图谱所有三元组的一阶逻辑如下（测例2）：

zero(0),succ(0,1),succ(1,2),succ(2,3),succ(3,4),succ(4,5),succ(5,6),succ(6,7),succ(7,8),succ(8,9)
pred(1,0), pred(2,1), pred(3,2), pred(4,3), pred(5,4), pred(6,5), pred(7,6), pred(8,7), pred(9,8)

目标谓词（0-9个数字之间的前序关系）

pred(x,y)

（答案是succ(y,x)->pred(x,y)）

(3)、（附加题）给定的知识图谱所有三元组的一阶逻辑如下（测例3）：

edge(a,b),edge(b,c),edge(c,d),edge(b,a), connected(a,b), connected(b,c), connected(c,d), connected(b,a), connected(a,c),connected(a,d), connected(a,a), connected(b,d), connected(b,b)

目标谓词（该图中的连接关系）

connected(x,y)

（答案是 edge(x,y)-> connected(x,y ); edge(x,z), connected(z,y )-> connected(x,y )）

要求的输入输出格式如下：

输入格式：
第1行：知识图谱的三元组数目n
第2~(n+1)行：三元组的一阶逻辑表示，例如L(David,Mike)
第n+2行：目标谓词，即"Grandfather(x,y)"
输出格式：
第1行：以目标谓词和前提约束谓词组成的规则推理，例如：Mother(z, y)∧ Couple(x,z)→Father(x, y)
第2行及以后：每一行输出一条推理所得事实，用一阶逻辑表示，例如H (David, Ann)

二、实验内容

1、算法原理（改进后）

（1）用re包读取并将逻辑语言分割成列表形式，分割成事实、目标、关系、成员四个列表。
（2）将事实中与目标谓词相同的作为正例，为了完成附加题，反例改用自己输入。
（3）建立约束谓词集，用于存储所有可能的约束谓词，但要排除成员相同和目标项。
（4）采用多层循环来寻找条件，最外层以剩余正例不为0作为循环依据，找到所有的充分条件（但不一定必要）。
（5）通过find_pre()函数获取充分条件时，对所有的约束谓词进行遍历，通过find_num()获取每个前提下的正反例数，然后进行以下操作：

正例数反例数操作
0任意值Continue
Num!=00去除这些正例，返回充分条件
Num!=0Num!=0通过gain获取最适条件，递归调用

（6）在计算成立正反例数时，通过对正反例赋值，查看前提是否在事实中，如果在则加一。
（7）将获取的所有充分条件合在一起就是充要条件，然后格式化输出。
（8）通过不断地对推理前提赋值实例化，如果前提在事实中，则相应结论一定成立，输出推理结论。

3、创新点&优化

我总共写了三个版本，并把对应的代码文件和输入文件附到code文件夹下，下面是他们的优缺点：

版本FOIL.pyFOIL_plus.pyFOIL_plus_plus.py
获取前提循环次数0(采用分步计算)1次2次
包含测试样例2个2个3个
优点结构简单，思路清晰循环调用，通用性强满足多个条件推理
1、反例生成问题：

  在前两个程序里，我都是采用自动生成的反例集，这样不仅能够减少自己的推理麻烦，而且更能够体现“智能”性，让机器自己去推理。但这有一些弊端，那就是会出现一些特殊情况，比如条件覆盖father->parent，就会把未知事实作为反例。
  在实现附加题时，我被迫采用了自己输入反例的方法，因为它的反例更是与事实关系不大，简单的更换谓词并不能满足需求，这样也更加合理，但自己推出反例确实费脑。

2、逻辑语言实例化：

  我的事实和例子是通过三元列表存储的，通过采用字典索引，可以实现不用真实替代或者使用新的空间生成实例化体而进行赋值比较，直接封装成函数使用更加简单。

3、通过不断删除正例集元素实现寻找多个充分条件

  我在每个大循环判断后，删除成立的正例集，让剩余的正例集取寻找新的充分条件去覆盖，这样不仅能够解决附加题，还能够解决三个或者更多的充分条件问题。

def FOIL_Gain(m1, m2):
    if m1==0:
        return -9999
    answer = m1*(math.log2(m1/(m1+m2))-(math.log2(len(pos)/(len(pos)+len(neg)))))
    return answer
