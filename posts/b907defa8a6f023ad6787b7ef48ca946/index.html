miniconda的使用以及在指定位置创建conda的虚拟环境

Miniconda简介

Miniconda是一款小巧的python环境管理工具，安装包大约只有70M多点，其安装程序中包含conda软件包管理器和Python。一旦安装了Miniconda，就可以使用conda命令安装任何其他软件工具包并创建环境等。本文是在Windows 10系统下进行下载和安装。

下载

可以进入miniconda的官网下载，也可通过以下链接直接进入下载页面。下载版本最好选择python3的64位版本，即Miniconda3 Windows 64-bit ，（电脑是32位，则下载32位版本）。

 安装miniconda，根据下面的网址选择版本即可

Miniconda — conda documentation

安装配置

下载完成后，可进入开始菜单栏中点击Anaconda Promot，通过conda命令进行Miniconda的安装和配置环境变量。以下通过conda命令进行安装和配置：

1.配置清华源

此处需要配置一个清华的镜像服务器，以解决conda下载文件速度慢的问题。

在Anaconda Promot中依次输入以下两条命令：

1. conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free/
2. conda config --set show_channel_urls yes

2.创建python 3.7 虚拟环境

我要创建python3.7虚拟环境，则需要在Anaconda Promot中输入命令：

conda create -n py37 python=3.7

虚拟环境名称：py37（该名称可根据自己喜好设定，建议设置容易记住的名称）

版本：python=3.7（3.7代表我创建的环境为3.7版本，此版本号可以自行设定）

3. 激活Python 3.7环境

如果需要触发刚刚创建的环境，则输入：

conda activate py37

4、退出或切换python 3.7环境

退出：

conda deactivate

切换到其他环境（此处切换到默认的base环境，即下载的python 3.7 环境 ）：

activate base

虚拟环境

5、使用命令查看当前拥有的虚拟环境

conda info --envs

6、在指定目录下创建新的虚拟环境，输入命令：

conda create --prefix=C:/ProgramData/Anaconda3/envs/pytorch python=3.8

其中C:/ProgramData/Anaconda3/envs 是创建的目录所在位置；/pytorch是所创建的环境的名称 python=3.8是创建的python的版本。

此时创作出来的环境没有名称，激活需要使用全路径，就是这一步设置的路径来激活

7.重命名或者复制原来的环境

 conda create --name newName（新环境名） --clone oldName（旧环境名）
 conda remove --name oldName（旧环境名） --all 

8、修改默认虚拟环境路径方法
首先，找到用户目录下的.condarc文件（C:\Users\username）。

打开.condarc文件之后，添加或修改.condarc 中的 env_dirs 设置环境路径，按顺序第⼀个路径作为默认存储路径，搜索环境按先后顺序在各⽬录中查找。直接在.condarc添加：

envs_dirs:
  - D:\SoftD\08envs
pkgs_dirs:
  - D:\SoftD\08envs
