理解js数组的sort函数排序

 每次用js的sort函数排序都容易犯错误，当时搞懂了一段时间就又忘了。 现在来把它搂一搂。 

 sort函数的用法： 

arr = [3,2,1]; 

arr.sort((a,b)=>{

...

}); 

 其实这个函数确实设计得有点奇怪,他让你传入一个自己的函数来确实排序算法，所以关键是自己怎么来写这个比较函数，

关于这个返回值 我很容易就想到 true/false 来表示 换不换位置。 

易犯错点一： 

错误把返回值写成 true/false

实际的返回值应该是数值。 

易犯错点二： 总是容易搞混这个第一参数，第二参数到底代表啥，总认为他传入是按顺序来的 a = arr[0],b = arr[1] 这样依次类推，

实际上是 a =arr [1],b = arr[0] ....  依次类推。 

还要就是关于他的实现原理： 有人说实现是用冒泡来实现的，也有人说不是，当然用冒泡之类的排序肯定可以实现，但是实际上不是，加打印看下就行，比如：

arr = [3,2,1]; 

arr.sort((a,b)=>{

  console.log("a = " + a + " b = " + b); 

   if(b>a){

        return -1;         //相当于从小到大排列

  }

return 1; 

}); 

从打印可以看出是 3 和 2 比，2和1比，只比再次，而冒泡就要比3次了。 关于他的实现原理我感觉更像是插值法，每个元素都找好自己的位置插进去，最后就是想要的排序了。 

有时候用起来感觉复杂和容易犯错，其实是思维定性导致的，比如返回true/false。 

最后的总结：

写sort的比较函数其实很简单

(a,b)=>{

   ...

}

实际只要想明白 a = arr[1] , b = arr[0] , 然后这两个数应该怎么排， 如果是想交换位置变成 arr[1]arr[0] 这样排就返回 -1，其他的啥都不用管，就是这样简单。
