当客户端计算机尝试连接到 SQL Server 实例时，SQL Server 客户端可能会更改协议

09/25/2020

本文内容

本文介绍在客户端计算机尝试连接到 SQL Server 实例时，SQL Server 客户端可能会更改协议。

原始产品版本：   SQL Server

原始 KB 数：   328383

摘要

(MDAC) 版本2.6 或更高版本的 Microsoft 数据访问组件的客户端计算机可以尝试使用多个协议或进程间通信 (IPC) 机制来建立与 SQL Server 的连接。

详细信息

对客户端网络库进行了改进，Dbnetlib.dll MDAC 版本2.6 及更高版本。 使用 MDAC 版本2.6 及更高版本时，如果有多个协议可用，并且第一个协议的连接尝试失败，则客户端应用程序会立即尝试使用其他协议之一。

默认情况下，客户端将 TCP 和命名管道作为可用协议。 您可以使用 SQL Server 客户端实用工具来处理协议排序。 客户端应用程序按客户端计算机上指定的顺序使用协议。 协议顺序存储在值 ProtocolOrder 下的以下注册表项位置：

HKLM\Software\Microsoft\MSSQLServer\Client\SuperSocketNetLib

如果使用的是 SQL Server 2005，则协议顺序存储在以下注册表子项下的 ProtocolOrder 注册表项中：

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\MSSQLServer\Client\SNI

例如，如果客户端计算机具有可用的 TCP 和命名管道，并且顺序为：

TCP

命名管道

当客户端计算机尝试建立与服务器的 TCP 连接，并且连接尝试返回一个非零的返回代码时，客户端将使用列表中的下一个协议透明地尝试一个连接，即 "管道"。 在这种情况下，客户端无法建立 TCP 连接;但是，客户端会成功建立命名管道连接。

备注

客户端不会收到指示第一次协议失败的错误。

如果客户端应用程序使用第二个协议，并且它还返回错误，则会向客户端返回错误。

如果使用以下方法之一创建别名，客户端应用程序将使用别名信息建立与服务器的连接，而不使用任何其他协议。

使用 SQL Server 客户端网络实用工具

使用 SQL Server 配置管理器

创建 ODBC 数据源名称时 (DSN)

如果要控制客户端应用程序对每次连接尝试使用的协议，而不允许客户端尝试多个协议，可以执行以下操作之一：

使用 SQL 客户端网络实用工具或 SQL Server 配置管理器通过指定您喜欢的协议来创建别名。

在连接字符串中指定协议。 例如：

DSN=DSNName;SERVER=servername;DATABASE=YourDataBaseName;Network=DBMSSOCN;Address=IP_Address,1433;UID=YourUID;PWD=YourPassword;

在此示例中，将 "网络协议" 指定为 DBMSSOCN ，这意味着您要使用 tcp/ip 协议。 如果在连接字符串中指定协议，则 Dbnetlib 只使用指定的协议，而不会尝试其他任何协议。 同样，若要仅启用命名管道协议，请使用类似如下的连接字符串：

DSN=DSNName;SERVER=servername;DATABASE=YourDataBaseName;Network=DBNMPNTW;Address=\\.\pipe\sql\query;UID=YourUID;PWD=YourPassword;

使用客户端网络实用工具删除其他协议。
