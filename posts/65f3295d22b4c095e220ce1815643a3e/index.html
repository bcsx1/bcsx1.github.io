数据类型

1.字符数据类型


（1）char（n）。固定长度的非Unicode字符数据类型。n用于定义字符数据的长度，n的取值范围为1~8000。存储大小为nB。
（2）varchar（n）。可变长度的非Unicode字符数据类型。n表示字符串可达到的最大长度。n的取值范围为1~8000。存储大小为所输入字符的实际长度+2B。
（3）text。长度可变的非Unicode字符数据类型，字符串最大长度为2^31-1（2147483647）B，用于存储大量的字符数据。
（4）nchar（n）。固定长度的Unicode字符数据类型。n用于定义字符数据的长度，n的取值范围为1~4000。存储大小为nB的2倍。当排序规则代码页使用双字节字符时，存储大小仍然为nB。
（5）nvarchar（n）。可变长度的Unicode字符数据类型。n表示字符串可达到的最大长度。n的取值范围为1~4000。存储大小（以B为单位）是所输入字符实际长度的2倍+2B。

（6）ntext,固定长度的Unicode字符数据类型,字符串最大长度2^30-1(1073741823)

2.精确数值数据类型

  (1)   bit ：取0,1或null，存储空间为1B。
（2）tinyint，取0~255之间的整数，存器空间为1B。
（3）smallint，取一215~25-1之间的整数，存储空间为2B。
（4）int，取一251~21-1之间的整数，存储空间为4B。
（5）bigint，取一258~28-1之间的整数，存储空间为8B。
（6）numeric（p，s）或decimal（p，s），取一10^38+1~10^38-1之间的数值，存储空间最多为17B，可存储小数点右边或左边的变长位数。p（precision）为总位数；s（scale）是小数点右边的位数。

(7)   money，取值为-2^63-2^63-1之间的数据，存储空间为8B。

 3.近似数值数据类型一间的数值


（a）float，取一1.79E+308~1.79E-303周w有，存储空间为8B。
（2）real，取-3.40E+38~3.40E+38之间的数值，存储空间为4B。


4.二进制数据类型


（1）binary（n),长度为nB的固定长度二进制数据，其中n的取值范围为1-80，存储大小为nB。
（2）varbinary（n）。可变长度二进制数据。n的取值范围为1~8000。存储大小员输入数据的实际长度+2B。
（3）image。用于存储照片、图片等，实际存储的是长度可变的二进制数据，,0-2^31-1B。

5.日期和时间数据类型


（1）Date。0001年1月1日至9999年12月31日格式的数据，存储空间为3B。
（2）Time。小时：分钟：秒.[n]格式的时间数据，n是0~7位数据，范围为0~9999999，表示秒的小数部分，存储空间为3~5B。
（3）Datetime。1753年1月1日至9999年12月31日格式的数据，精确到最近的
3.33ms，存储空间为8B。
（4）Datetime2（n）。0001年1月1日至9999年12月31日格式的数据，n的取值范围为0~7，指定小数秒的位数，存储空间为6~8B。
（5）Smalldate Time。1900年1月1日至2079年6月6日，精确到1min的数据，存储空间为4B。

T-SQL创建表

CREATE TABLE语句

语法格式

CREATE TABLE 表名
(
    列名1  数据类型  是否允许为空[not null|null]
    列名2  数据类型  是否允许为空[not null|null]
    ....
)

示例

CREATE TABLE 课程表
(
    ID char(6) NOT NULL,
    Name varchar(20) NULL,
    grade tinyint ,
    Period smallint
)

数据完整性

数据完整性是指数据的正确性，有效性和相容性。

        正确性是指数据的合法性；

        有效性是指数据是否属于所定义的有效范围；

        相容性是指描述统一实体的数据应当一致。

数据完整性一般分为三类：实体完整性，参照完整性与域完整性。

实现实体完整性

        实体完整性要求将行定义为特定表的唯一实体，要求表中的每一行必须是唯一的，即要求表中的所有行都有一个唯一的标识符。

实现实体完整性的方法有PRIMARY KEY约束和UNIQUE约束。

PRIMARYKEY约束


        PRIMARYKEY约束也称为主键约束。主键是表中的一个或多个字段，它的值用于唯一地标识表中的某一条记录。表中主键在所有行上必须取值唯一且不能为空值。通过主键可强制表的实体完整性，一个表只能有一个主键。当创建或更改表时可通过定义PRIMARYKEY约束来创建主键。

CONSTRAINT  PRIMARY KEY 语句

语法

//添加，需要指明对象表
ADD CONSTRAINT 约束名称(自定义) PRIMARY KEY(指明主键约束) (被约束的列名 ASC|DESC)
//默认为ASC升序排列，DESC降序排列


//删除，需指明对象表
DROP CONSTRAINT 约束名称

示例

--创建时直接指定主键则无需使用CONSTRAINT
CREATE TABLE 课程表
(
    ID char(11) not null PRIMARY KEY
)


--创建时未指定主键,需要使用修改
CREATE TABLE 课程表
(
    ID char(11) not null 
)

//添加主键约束
ALTER TABLE 课程表
ADD CONSTRAINT ID的主键约束 PRIMARY KEY(ID)



//删除主键约束
ALTER TABLE 课程表
DROP CONSTRAINT ID的主键约束

 UNIQUE约束

        UNIQUE约束也称为唯一约束，确保表中指定列中不出现重复值，即表中任意两行在该列上的值都不运行相同。与PRIMARY KEY的区别如下:

（1）一个表只能有一个PRIMARY KEY约束；一个表中可以由多个UNIQUE约束。

（2）PRIMARY KEY 约束所定义的列不允许有空值；UNIQUE约束所定义的列可以有空值。

语法与PRIMARY KEY约束相同，只需改变为UNIQUE约束即可

//添加，需要指明对象表
ADD CONSTRAINT 约束名称(自定义) UNIQUE(指明UNIQUE约束) (被约束的列名 ASC|DESC)
//默认为ASC升序排列，DESC降序排列


//删除，需指明对象表
DROP CONSTRAINT 约束名称

示例

//添加UNIQUE约束
ALTER TABLE 课程表
ADD CONSTRAINT ID的UNIQUE约束 UNIQUE(ID)



//删除UNIQUE约束
ALTER TABLE 课程表
DROP CONSTRAINT ID的UNIQUE约束

实现参照完整性

        参照完整性是指在数据库内表与表之间数据的一致性。参照完整性基于主键（被参照表）与外键（参照表）之间或唯一键与外键的关系。通过外键将参照表和被参照表关联起来。可以通过FOREIGN KEY约束实现参照完整性。

FOREIGN KEY

语法

//添加，需要指明对象表
ALTER TABLE 参照表名
ADD CONSTRAINT 约束名称(自定义) FOREIGN KEY (被约束的列名)
REFERENCES 被参照表名(列名)

示例

//添加FOREIGN KEY约束
ALTER TABLE 课程表
ADD CONSTRAINT ID的FOREIGN约束 FOREIGN KEY(ID) REFERENCES 学生成绩表(ID)
//注意学生成绩表里的ID列需要设置为主键

  实现域完整性

        域完整性是指列数据的完整性和有效性，即要求对指定列有效的一组值并决定是否允许有空值。可以通过约束，规则和默认值的方法实现。

强制域有效的方法可通过CHECK约束、DEFAULT约束、限制数据类型、格式或取值范围实现。

CHECK约束

语法

CREATE TABLE 表名
(
    列名  数据类型  是否允许为空 CONSTRAINT 约束名 CHECK(表达式)
)


ALTER TABLE 表名
ADD CONSTRAINT 约束名 CHECK(表达式)

示例

ALTER TABLE 学生情况
ADD CONSTRAINT ck_学生性别 CHECK(性别 in ('男','女'))

DEFAULT约束

DEFAULT约束也成为默认值约束。

CREATE TABLE 表名
(
    列名  数据类型  是否允许为空 CONSTRAINT 约束名 DEFAULT(表达式)
)


ALTER TABLE 表名
ADD CONSTRAINT 约束名 DEFAULT(表达式)
ALTER TABLE 学生情况
ADD CONSTRAINT ck_学生性别 DEFAULT '男' for 性别

默认值对象

默认也称为默认值，是数据库对象的一种，它的作用是为列设置一个默认值，而这个默认值可以被一个数据库中的多个表中的多个列绑定使用。使用默认对象的过程是先创建默认，再绑定到列。删除默认时，要先解除绑定，然后再删除。

创建默认

CREATE DEFAULT 默认值名
AS 表达式
CREATE DEFAULT default_subsidy as 300

 绑定默认

EXEC sp_bindefault(系统自带函数) 'default_subsidy','表名.列名'

 解除绑定

EXEC sp_unbindefault(系统自带) '表名.列名'

 删除默认

DROP DEFAULT 默认值名

规则

        规则是一种数据库对象，它的作用类似与CHECK约束，也是对列的数据进行约束,不满足其约束条件的数值是不能输入到相应列中去的。

创建规则

CREATE RULE 规则名 as 表达式

 绑定规则

exec sp_bindrule '规则名','表名.列名'

 删除规则

drop rule 规则名
