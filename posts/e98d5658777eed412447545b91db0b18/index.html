三、有线侧的管理协议---CAPWAP

  1. AC于AP之间使用有线的方式进行连接，在连接的链路上运行CAPWAP协议，构建CAPWAP隧道。

  1. CAPWAP隧道中负责传送2种流量。

2.1、控制流量【AC用来统一管理瘦AP的控制信令流量】

2.2、业务流量【移动终端用户通过连接AP所发送的数据流量】

四、AP于AC的组网方式：

  1. AP与AC之间用二层交换机连接，通过二层交换机的多端口，使多个AP接入交换机，实现和AC的配置下发，集中管理，所有设备都属于同一个广播域当中，适用于小型网络。

  1. AP与AC之间使用三层交换机互联，通过三层交换机的多VLNA，多网段的功能，实现AC对不同网络之间的AP的控制，适用于大型网络的互联。

五、AC在组网中的连接方式：

  1. 直连式：AC于AC直连，对AP进行统一的直接管理，AC也作为流量的出口设备，作为内网的转发网管。缺点：容易成为流量转发的瓶颈，AC的转发速率成为全网的最高速率。

  1. 旁挂式：AC不直接于AP相连，而是通过其他网络于其进行互联。优势在于不会成为网络的瓶颈，实现最大带宽的透传。缺点在于无法对多个AP的数据进行统一的集中管理。

六、无线电磁波频段：

极低频(3Hz-30Hz):潜艇通讯或直接转换成声音。

超低频(30Hz-300Hz):直接转换成声音或交流输电系统(50-60赫兹)。特低频(300Hz-3KHz):矿场通讯,或直接转换成声音。

甚低频 (3KHz-30KHz): 直接转换成声音、超声、地球物理学研究。低频(30KHz-300KHz):国际广播。

中频(300KHz-3MHz):调幅(AM)广播、海事及航空通讯。高频(3MHz-30MHz):短波、民用电台。

其高频(30MHz-300MHz):调频(FM)广播、电视广播、航空通讯。

特高频(300MHz-3GHz):电视广播、无线电话通讯、无线网络、微波炉。超高频(3GHz-30GHz):无线网络、雷达、人造卫星接收。

极高频(30GHz-300GHz):射电天文学、遥感、人体扫描安检仪。300GHz以上:红外线、可见光、紫外线、射线等。

八、无线侧面网络组网的基本概念：

  1. BSS【基本服务集】：一个AP所发射的无线覆盖范围

范围：通过改AP连接的用户都可以互访

  1. SSID【服务集标识符】：一个无线信号的名称

  1. BSSID【基本服务集标符】:无线AP的MAC地址

4.VAP【虚拟接入点】：一个AP可以发射多个BSS,每个BSS就是一个VAP，每个BSS都有一个单独的SSID信号。

  1. ESS【扩展服务集】：由多个相同SSID的AP所组成的一个大的无线通讯范围。

九、WLAN的工作流程

【一】AP上线

  1. 获取地址

a、静态分配地址

b、让AP向AC获取IP地址

c、让AP向网络中的DHCPSERVER获取IP地址

  1. 让AC之间建立CAPWAP隧道

发现隧道隧道：有两种

a、静态建立隧道：管理员在AP内配置其所需要寻找的AC【在该方式下，AP所发送的DiscoveryRequest 是以单播方式发送】

b、动态建立隧道：令AP以广播的形式发送Discovery Request 消息，在全网范围内寻找AC

建立隧道：

3. 在AC接收到所发送的DiscoveryRequest消息后，将会回应Discovery Reponse消息与AP 建立CAPWAP隧道。

4.CAPWAP隧道中包含两个子隧道

a、控制隧道：传递的是AC于AP的控制信令

b、数据隧道：传递的是STA设备发送的实际业务数据

5、AP接入控制

在AP通过Discovery Request 消息找到AC后，下一步AP将会给AC发送joinRequest 消息，请求于AC建立连接。

6、AC认证AP的方式有三个：

a、mac地址验证

b、SN序列号验证

c、不验证【丧失了安全性，但是非常轻松，强烈不推荐】

7、AP的版本升级【可选项】

AP可以向AC发送ImageData Request ，向AC请求更新的操作系统版本。

若AC上拥有对应的版本系统，则向AP 传递Image Data Response 消息

8、CAPWAP隧道的维护

在AP取得IP地址后，也成功的和AC取得了联系，建立了CAPWAP隧道，而且上线成功，则正式进入CAPWAP的隧道维护。

隧道维护分为两种：因为有两个CAPWAP子隧道

a、控制隧道维护：依靠Echo Request消息来维护【UDP 5246】

b、数据隧道维护：依靠发送Keepalive消息来维护【UDP 5247】

十：WLAN业务配置下发

a、在AP完成正式上线后，AP主动向AC发送Configuration Status Request 消息，请求AC将所有的配置信息进行下发，AC收到请求消息后，通过Configuration Status Request 消息先进性回复。

b、AC将配置下发给AP设备，AP接收到了配置信息后，回复一个确认消息。

AC向AP传送的配置信息包含了：

  1. 域管理模板：包含国家代码

  1. 射频模板：包含AP所使用的是2.4G或5G

  1. VAP模板：主要包含AP所发射的SSID，密钥，加密认证方式

  1. 其他模板：包含AP的版本信息参数

  1. 射频参数模板：包含所使用的信道、工作频率、最大功率等

其中VAP模板包含：

a、SSID模板：使用的SSID号为多少

b、安全模板啊：加密所使用的加密方式及密钥

c、数据转发方式：数据报文转发的模式（旁挂式或者是直连式）

配置业务VLAN：用户发送数据所属的VLAN ID

3、连接 STA【无线设备】设备<

STA 设备接入 AP 共分为 6 个阶段:<

3.1、扫描阶段一

3.1.1、有明确指向性的扫描SSID

STA设备以单播的方式发送Probe Request.寻找明确的SSID←

3.1.2、没有明确目的的广播扫描SSID一

STA 设备以广播的方式发送Probe Request，发送当前网络

中的所有开启了广播功能的 SSID

3.2、STA 的链路认证

当 STA 设备发现了 SSID 之后，需要与 SSID 建立连接时，AP

将向用户发送密钥请求

STA 设备主动向 AP 发送Authentication Request，请求进行

认证，AP 向 STA 设备回传 Authentication Response，包含

Challenge挑逗消息】.之后 STA 设备向 AP 发送Encryption

消息进行认证.最后 AP 向 STA 设备返回认证结果 success、

failed

STA 设备与AP 之间的认证分为4类:<

3.2.1、WEP:不认证，所有用户均可连接一

3.2.2、WEP:使用对称密钥，发送密钥时不加密<

3.2.3、WPA:同样使用对称密钥。发送密钥匙加密←

3.2.4、WPA2:同样使用对称密钥，发送密钥匙加密一

注:WPA 与WPA2 的加密长度不一样，WPA2 的加密长度更

长

3.3、关联消息

当 STA 设备与AP 之间的验证通过后，STA 设备主动向AP发送Association Request 消息，向AP 告知自身可以接收的例如:速率信息、双工模式、带宽、信道、QoS 等参数

AP 接收到该消息后，将该消息上传至AC，AC 向 AP 回传 Association Response 消息，AP 再向 STA 设备回传心

3.4、接入认证阶段←

STA 设备与 AP 之间协商数据在发送时所使用的加密的方式

3.5、DHCP 阶段

在 STA 设备与 AP 之间协商完所有参数后，STA 设备此时需

要获取 IP 地址，完成通讯前的最后准备工作

在瘦 AP 中,AP 不会向 STA 设备分配 IP 地址,AP 将该DHCP

Discovery Request 消息直接透传给网络中负责分配 IP 地址

服务器，由该服务器向 STA 设备分配IP 地址。

3.6、用户认证阶段

为防止来自企业内网的攻击，在 STA 设备已经获取了 IP 地址后，在其想与内网主机通讯时，需要最后在企业的门户网站进行 Web 验证【包含:用户名、手机号码、工号、个人 ID号等信息】e

4、WLAN 的数据业务转发一

根据数据业务所使用的网络路径,数据业务转发可分为2类:一

4.1、隧道转发方式:优点:AC 可对整体网络的数据进行监控;缺点:AC有可能成为整个网络数据转发的瓶颈<

4.2、直接转发方式:优点:不会产生瓶颈问题，数据可通过交换机直接透传至上游网络;缺点:AC 无法整体对网络进行统监管。
