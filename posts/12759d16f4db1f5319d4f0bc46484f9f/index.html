一、问题描述

代码写好了，我要提交的 git 库。

在 git gui 界面对比代码时，我发现有不需要改动的代码被改动了，如下图。

这时，我想快速找到这个文件中的这个内容，我该怎么办呢？

下面是我的解决方案，供大家参考。

二、解决方案

（一）方案一：【双击shift】+【ctrl+F】

第一，【双击shift】搜索查找指定文件。

第二，在该文件中，【ctrl+F】查找指定内容。

第三，搞定啦，已快速找到了我想要的内容了。

1、【双击shift】搜索查找指定文件

【双击shift】出现弹跳框如下图。

首先，在搜索框输入文件名，会自动搜索。（也就是紫色长方形的地方）

然后，在搜索的结果中，找到目标文件。

再是，双击此目标文件，会立即打开该目标文件，并关闭搜索弹跳框。

2、在该文件中，【ctrl+F】查找指定内容

在打开的目标文件中，执行【ctrl+F】，出现紫色框内容，如下图。

在紫色框中，输入要查找的内容，点上下的箭头。

这样，就找到了想要的内容，如红色框。

（二）方案二：【ctrl+shift+F】

【ctrl+shift+F】出现弹跳框如下图。

首先，在搜索框输入目标内容，会自动搜索。（也就是紫色长方形的地方）

然后，在搜索的结果中，上下翻找，找到目标文件目标内容。（也就是红色长方形的地方）

再是，双击此目标文件内容，会立即打开该目标文件，并跳转到目标内容处，同时关闭搜索弹跳框。
