ORA-01109: database not open

一般是oracle12c 及以上会出现这种问题.

装的数据库是CDB和PDB一起的，但是网上说的startup启动数据库的时候其实只是默认只启动了数据库容器就是CDB，这个时候如果你要连接PDB数据库的时候就会提示ORA-01109: database not open。

1、sqlplus "/as sysdba" 进入数据库
2、select name,open_mode from v$pdbs查看数据库状态，这里的ORCLPDB1是我要连接的数据库，目前是MOUNT状态
3、使用命令alter pluggable database ORCLPDB1 open;把他打开就行了。

====服务器停止异常：?watchdog没有启动，resin没法正常链接watchdog

[root@190-235-cwwb-ecology bin]# ./stopresin.sh
doing monitorhook!
Jun 03, 2022 9:50:32 PM com.caucho.monitor.MonitorHook checkMonitorAsync
INFO: checkMonitorAsync disabled: null…
Resin/4.0.58 can’t stop -server ‘app-0’ for watchdog at 127.0.0.1:6600.
com.caucho.bam.RemoteConnectionFailedException: BamError[type=cancel,group=remote-connection-failed,text=Cannot connect to http://127.0.0.1:6600/hmtp
java.net.ConnectException: Connection refused (Connection refused)]

　描述：为了稳定和安全，Resin使用一个独立的watchdog进程来启动和监视Resin服务器。
watchdog连续检测Resin服务器的状 态，若是其没有反应或者迟钝，将会重启Resin服务器进程。
大多数状况下，watchdog从resin.xml中读取配置信息，而不须要其它的附加配 置。某些ISP服务商可能使用特别的配置，
为了安全他们会配置watchdog来隔离JVM(Java虚拟主机)。

which java：查看java安装路径
在resin目录下：
/usr/java/jdk1.8.0_201-amd64/bin/java -jar lib/resin.jar  start：启动resin服务监听器

vim ifcfg-ens33
 TYPE="Ethernet" #⽹卡类型（通常是Ethemet以太⽹）
PROXY_METHOD="none" #代理⽅式：为关闭状态
BROWSER_ONLY="no"  #只是浏览器：否

DEFROUTE="yes" #默认路由
IPV4_FAILURE_FATAL="no"  #是否开启IPV4致命错误检测
IPV6INIT="yes" #IPV6是否⾃动初始化：是（现在还未⽤到IPV6，不会有任何影响）
IPV6_AUTOCONF="yes" #IPV6是否⾃动配置：是（现在还未⽤到IPV6，不会有任何影响）
IPV6_DEFROUTE="yes" #IPV6是否可以为默认路由：是（现在还未⽤到IPV6，不会有任何影响）
IPV6_FAILURE_FATAL="no"  #是否开启IPV6致命错误检测
IPV6_ADDR_GEN_MODE="stable-privacy"  #IPV6地址⽣成模型
NAME="ens33"  #⽹卡物理设备名称
UUID="dd4e5cec-7e01-4648-91f1-2f9815d0da1e"
DEVICE="ens33" #指定当前配置的是 ens33 网卡
ONBOOT="yes" #表示该网卡在启动时自动激活
IPADDR=192.168.2.111 #指定 IP 地址为 192.168.2.111
DNS1=223.5.5.5  #指定首选 DNS 服务器的 IP 地址为 223.5.5.5
DNS2=114.114.114.114  #指定首选 DNS 服务器的 IP 地址为 114.114.114.114
GATEWAY=192.168.150.2  #指定默认网关为 192.168.150.2
NETMASK=225.225.225.0  #指定子网掩码为 255.255.255.0
