MySQL基础使用方法

一、 MySQL基础

（1）什么是MySQL

数据库是一个以某种有组织的方式存储的数据集合。而SQL全称为Structured Query Language，翻译过来就是结构化查询语言，是一种专门用来与数据库通信的语言。MySQL则是一种数据库管理系统(DBMS)，是一种软件系统。

（2）数据库结构

数据库：保存有组织的数据的容器
表：某种特定类型数据的结构化清单
模式：关于数据库和表的布局以及特性的信息，也即是定义了数据在表中如何存储的。
列：每一列就是表中的一个字段。
行：每行作为表中的一个记录
主键：一列(或者一组列)，其值能够唯一区分表中的每个行。(主键不允许NULL值)
外键：为某个表的某一列，包含了另一个表的主键值，定义了两个表之间的关系

（3）数据库引擎

MySQL常用的引擎有两种，分别是InnoDB和MyISAM。

InnoDB

InnoDB是MySQL默认的事务型引擎，也是最重要、最广泛的存储引擎。它的设计是用来处理大量短期事务，短期事务大部分是正常提交的，很少回滚。InnoDB的性能和自动崩溃恢复特性，使得它在非事务型存储的需求中，也很流行。除了非常特别的原因需要使用其他引擎，InnoDB也是非常好值得花时间研究的对象。
InnoDB的数据存储在表空间中，表空间是由InnoDB管理的黑盒文件系统，由一系列系统文件组成。InnoDB可以将每个表的数据和索引存放在单独的文件中。InnoDB也可以使用裸设备作为表空间存储介质。
InnoDB通过间隙锁（next-key locking）防止幻读的出现。InnoDB是基于聚簇索引建立，与其他存储引擎有很大的区别，聚簇索引对主键查询有很高的性能，不过它的二级索引（secondary index，非主键索引）必须包含主键列。所以如果主键列很大的话，索引会很大。

MyISAM

在5.1之前，MyISAM是默认的引擎，MyISAM有大量的特心态，包括全文索引、压缩、空间函数。但是MyISAM不支持事务和行级锁，而且在崩溃后无法安全恢复。即使后续版本中MyISAM支持了事务，但是很多人的概念中依然是不支持事务的引擎。

MyISAM并不是无所事处。对于一些只读数据，或者表空间较小，可以忍受恢复操作，可以使用MyISAM。MyISAM会将表存储在两个文件中：数据文件、索引文件。分别是.MYD、.MYI扩展名。MyISAM表可以包含动态或者静态行。MySQL会根据表定义选择那种行格式。MyISAM表的行记录数，取决于磁盘空间和操作系统中的单个文件最大尺寸。

在MySQL中，默认配置只能存储256TB的数据。因为指向数据记录的指针长度是6字节。需要修改可以修改表的MAX_ROWS和AVG_ROW_LENGTH选项。两个相乘是最大的大小。会导致重建索引。
MyISAM是对整个表加锁，而不是行锁，读取的时候对表加共享锁，写入的时候加排他锁。但是在表有读取查询的同时，也可以往表内写入记录。

对于MyISAM，即使是Blob，Text等等长字段，也可以基于前500字符创建索引，MyISAM支持全文索引，这是一个基于分词创建的索引，也可以支持复杂的查询。

MyISAM可以选择延迟更新索引键，在创建表的时候指定delay_key_write选项，在每次修改执行完成时，不会立刻将修改的索引数据写入磁盘，而是写到缓存区，只有在清理缓存区或者关闭表的时候才会将索引写入磁盘。这可以极大的提升写入性能，但是在主机崩溃时会造成索引损坏，需要执行修复操作。

MyISAM另一个特性是支持压缩表。如果数据在写入后不会修改，那么这个表适合MyISAM压缩表。可以使用myisampack对MyISAM表进行打包，压缩表是不可以修改数据的。压缩表可以极大的减少磁盘占用，因此可以减少磁盘IO，提升性能，压缩表也支持索引，但是索引也是只读的。
整体来说MyISAM并没有那么不堪，但是由于没有行锁机制，所以在海量写入的时候，会导致所有查询处于Locked状态。

（4）MySQL使用

USE语句：USE databasename 输出：Database changed，mysql命令行程序数据库选择成功后显示。

SHOW语句：显示相关信息，支持的语句有：
1)SHOW DATABASES，返回可用数据库的一个列表;
2)SHOW TABLES，返回一个数据库内的表的列表;
3)SHOW COLUMNS FROM table1，返回table1表中各列的信息;
4)SHOW STATUS，用于显示广泛的服务器状态信息;
5)SHOW CREATE DATABASE 或 SHOW CREATE TABLE，分别用来显示创建特定数据库或表的MySQL语句;
6)SHOW GRANTS，用于显示授予用户的安全权限;7)SHOW ERRORS 和 SHOW WARNINGS用来显示服务器错误或警告信息。

SQL语句大小写：SQL语句不区分大小写，习惯上将关键字大写，表名、列名等用小写，便于区分和阅读。

二、 数据检索

（1）SELECT

SELECT colname
FROM table1;

返回：表table1中名为colname的列。

SELECT colname1,colname2,colname3
FROM table1;

返回：表table1中名为colname1、colname2、colname3的列。

SELECT *
FROM table1;

返回：表table1中所有列，*为通配符。

（2）简单限制搜索

SELECT DISTINCT colname1
FROM table1;

返回：表table1中colname1列中具有不重复的值的行。DISTINCT关键字指示MySQL只返回不同值。

SELECT colname1
FROM table1
LIMIT 5;

返回：表table1中colname1列中符合条件的前5行。

SELECT colname1
FROM table1
LIMIT 10,5;

返回：表table1中colname1列中符合条件的从第10行开始的5行。

（3）数据排序检索

SELECT colname1
FROM table1
ORDER BY colname1;

返回：表table1中colname1列，并按照colname1中值的顺序进行排序。ORDER BY默认升序排序(也可以指定ASC关键字);

SELECT colname1
FROM table1
ORDER BY colname1 DESC;

返回：表table1中colname1列，并按照colname1中值的顺序进行降序排序输出;若多列排序时希望以降序排序，则需要对每个列指定DESC关键字。

SELECT colname1,colname2,colname3
FROM table1
ORDER BY colname1,colname2;

返回：表table1中colname1、colname2、colname3列，先根据colname1值再根据colname2中值的顺序进行排序。

（2）组合过滤

AND操作符

SELECT colname1,colname2
FROM table1
WHERE colname1  = 100 AND colname2 > 10;

返回：表table1中colname1和colname2两列中，满足colname1 = 100且colname2 > 10的所有行。(多个过滤条件需要添加相应的AND)
OR操作符

SELECT colname1,colname2
FROM table1
WHERE colname1  = 100 OR  colname2 > 10;

返回：表table1中colname1和colname2两列中，满足colname1 = 100或者colname2 > 10的所有行。
优先级：AND优先级高于OR，必要时需要使用括号组合操作符
IN操作符

SELECT colname1,colname2
FROM table1
WHERE colname1  IN (10,20);

返回：表table1中colname1和colname2两列中， colname1值位于10-20范围的所有行。注：可通过OR等价，只是写法繁杂了许多
NOT操作符

SELECT colname1,colname2
FROM table1
WHERE colname1  NOT IN (10,20);

返回：表table1中colname1和colname2两列中， colname1值不位于10-20范围的所有行。注：MySQL支持使用NOT对IN、BETWEEN、EXISTS取反，这与其他DBMS存在差异。

（3）使用通配符

通配符含义
%表示任何字符出现的任意次数

使用LIKE操作符与通配符组合，LIKE匹配整个列。

SELECT colname1

（4）使用通配符

基本字符匹配

SELECT colname1
FROM table1
WHERE colname1  REGEXP '1000';

返回：表table1中colname1值包含1000的所有行。
注：若使用LIKE，即“WHERE colname1 LIKE ‘1000’;”，将会进行全文匹配，匹配colname1值为1000的所有行，colname1 = abc1000的行将不会被返回，除非使用通配符。

SELECT colname1
FROM table1
WHERE colname1  REGEXP '1000|2000';

返回：表table1中colname1值包含1000或2000的所有行。
多个字符之一匹配

FROM table1

范围匹配

SELECT colname1
FROM table1
WHERE colname1  REGEXP '[a-c] def';

返回：表table1中colname1值包含adef、bdef、cdef的所有行。范围可选[0-9]或[a-z]

SELECT colname1
FROM table1
WHERE colname1  REGEXP '\\.';

返回：表table1中colname1值包含.的所有行。

（7）数据分组

GROUP BY： 将数据进行分组，可对每个分组执行聚集函数

SELECT id, COUNT(*) AS num
FROM table1
GROUP BY id;

返回：返回值将对每个不同的id进行一次COUNT()计数
HAVING： 针对分组进行过滤（WHERE针对行进行过滤）

SELECT id, COUNT(*) AS num
FROM table1
GROUP BY id
HAVING COUNT(*) >=2;

返回：对每个不同的id进行一次COUNT()计数，并返回其中大于2的部分。

（8）子查询

指嵌套在其他查询中的查询。

SELECT id
FROM table1
WHERE num IN(SELECT num
			 From table2
			 WHERE colname = 'abc');

返回：首先进行括号内的查询，查询table2中colname列值为’abc’的行，返回对应行的num值；然后进行括号外的查询，返回table1中num值为第一次查询的结果的行，返回对应的id值。

SELECT name,
	   state,
	   (SELECT COUNT(*)
		FROM orders
		WHERE orders.id = customers.id) AS orders
FROM customers
ORDER BY name;

子查询的结果也可以作为计算字段使用。
部分子查询会涉及外部查询，如’WHERE orders.id = customers.id’

四、 多表搜索与组合查询

将数据分解为多个表，可以实现更有效的存储。以供应商-产品信息举例，一个供应商可能对应多种产品，主表vendors中维护供应商的信息，每行对应一个唯一的供应商；在products表中储存供应商对应的产品信息，并包含vendors主键值。这样做的好处是，供应商信息不重复，节省储存资源；供应商信息变更时，只需更新vendors中的单行数据。

（1）联结

SELECT vend_name, prod_name, prod_price
FROM vendors, products
WHERE vendors.vend_id = products.vend_id
ORDER BY vend_name, prod_name;

返回：表vendors与products通过WHERE子句完成联结，指示查询范围为vendors表与products表中vend_id一致的数据。

（2）内部联结

SELECT vend_name, prod_name, prod_price
FROM vendors INNER JOIN products
ON vendors.vend_id = products.vend_id;

返回：表vendors与products通过FROM子句中INNER JOIN 完成联结，过滤条件由ON指定，其结果与(1)一致。

（3）多表联结

SELECT vend_name, prod_name, prod_price, quantity
FROM orderitems, products, vendors,
WHERE products.vend_id = vendors.vend_id
AND orderitems.prod_id = products.prod_id
AND  order_num = 20005;

返回：表orderitems储存订单信息，本次查询查找订单号为20005订单中的物品，根据表中产品ID查询products表中信息，再根据供应商id在vendors表中查询供应商信息。

（4）自联结

SELECT p1.prod_id, p2.prod_name
FROM products AS p1, products AS p2
WHERE p1.vend_id = p2.vend_id
AND p2.prod_id = 'ABCD';

返回：通过使用别名，可以在同一条SELECT语句中重复引用同一个表，此次查询返回与’ABCD’具有相同供应商id的所有行。

SELECT prod_id, prod_name
FROM products 
WHERE vend_id = (SELECT vend_id
				 FROM products
				 WHERE prod_id = 'ABCD');

返回：通过子查询得到与自联结相同的结果。

（5）自然联结

在标准联结的例子中，至少有一个列出现在不止一个表中，自然联结通过对某个表使用通配符(*)，其余表列使用具体的子集来保证输出的列唯一。

SELECT c.*,o.order_num,o.order_date,oi.prod_id, oi.quantity, oi.item_price
FROM customs AS c,orders AS o, orderitems AS oi
WHERE c.cust_id = o.cust_id
AND oi.order_num = o.order_num
AND prod_id = 'FB';

（6）外部联结

SELECT customs.cust_id, orders.order_num
FROM customs LEFT OUTER JOIN orders
ON customs.cust_id = orders.cust_id;

返回：外部联结包含了在相关表中没有关联行的行。使用OUTER JOIN时，必须使用LEFT或RIGHT指定包含所有行的表。

（7）组合查询

SELECT colname1,colname2,colname3
FROM table1
WHERE colname1 <10
UNION 
SELECT colname1,colname2,colname3
FROM table2
WHERE colname2 IN(100,100);

返回：由UNION指定组合查询，将两次的查询结果组合成为一个结果。每两个SELECT之间以UNION分隔；UNION中每个查询包含相同的列、表达式、聚集函数；UNION会自动去除重复行，若需保留，使用UNION ALL；在最后一个SELECT后添加ORDER BY即可对所有结果排序。

五、 数据表创建、操作、数据增改删

（1）创建数据表

CREATE TABLE customers(
	cust_id		int		NOT NULL AUTO_INCREMENT,
	cust_name   char(50)	NOT NULL,
	cust_addr	char(50) NULL,
	PRIMARY KEY (cust_id)
)ENGINE = InnoDB

结果：创建表customers，表中含有cust_id,cust_name,cust_addr三列，并指定cust_id作为主键;
NOT NULL/NULL标识该列值是否允许为NULL；
AUTO_INCREMENT则说明该列在每次INSERT时，自动对该列增量，赋予下一个可用值。每张表只允许一个AUTO_INCREMENT列，并且该列必须被索引。通过last_insert_id()函数可以获得最后一个AUTO_INCREMENT值

（2）更新数据表

ALTER TABLE vendors
ADD   colname_new	char(50)
DROP colname_old
FOREIGN KEY (order_num) REFERENCES orders(order_num);

结果：使用ALTER TABLE 关键字更新表；第二行，新增colname_new 列表；第三行，删除colname_old列；第四行，指定外键order_num。

（3）重命名数据表

RENAME TABLE table_new1 TO table_old1
			 table_new2 TO table_old2;

结果：对多个表重命名。

（1）视图

**视图：**是一种虚拟的表，只包含使用时动态检索数据的查询

SELECT cus_name, cus_contact

WHERE customers.cust_id = order.cust_id
AND orderitems.order_num = order.order_num
AND pro_id = 'TNT2';

结果：一个对三个表进行搜索的查询，可以将整个查询包装成一个productcustomers的虚拟表，即视图，下面的查询即与上面的等价

SELECT cus_name, cus_contact
FROM productcustomers
WHERE pro_id = 'TNT2';

结果：创建后即可使用上面的例子利用视图进行查询。

CREATE VIEW formatted_output AS
SELECT Concat(RTrim(colname1),’(‘,RTrim(country),’)’) AS result_new
FROM table1

结果：创建一个提供格式化输出的视图。

CREATE VIEW formatted_output AS
SELECT Concat(RTrim(colname1),’(‘,RTrim(country),’)’) AS result_new
FROM table1
WHERE colname1 IS NOT NULL;

结果：创建一个筛选colname1非空并提供格式化输出的视图。
视图的优点： 代码复用；简化操作；使用表的组成部分；可以给用户以部分表的权限，以保护数据；提供原始表的格式化输出。
视图的使用规则： 命名唯一；数目无限；创建视图需要一定权限；视图允许嵌套；从视图中检索数据的ORDER BY命令会覆盖视图内部的ORDER BY；视图不能索引。
视图的更新： 允许使用INSERT、UPDATE、DELETE(这会更新原始表)；但若视图中使用了分组、联结、子查询、并、聚集函数、DISTINCT、导出操作，则视图不允许更新。

（2）存储过程

存储过程： 是为以后的使用而保存的MySQL语句集合。

CREATE PROCEDURE productpricing()
BEGIN 
	SELECT  AVG(prod_price) AS priceaverage
	FROM products
END;

结果：创建一个存储过程

CALL productpricing();

结果：调用上面的存储过程

DROP PROCEDURE productpricing;

结果：删除存储过程

CREATE PROCEDURE productpricing(
OUT pl DECIMAL(8,2),
OUT ph DECIMAL(8,2),
OUT pa DECIMAL(8,2))
BEGIN 
	SELECT  MIN(prod_price) 
	INTO pl
	FROM products;
	SELECT  MAX(prod_price) 
	INTO ph
	FROM products;
	SELECT  Avg(prod_price) 
	INTO pa
	FROM products;
END;

结果：创建一个含3个变量的存储过程，并将prod_price的最小值、最大值、平均值分别存储在变量pl、ph、pa中。MySQL支持IN、OUT、INOUT三种类型参数，分别为传入、传出、传入传出。

CALL productpricing(@pricelow,
					@pricehigh,
					@priceaverage);

结果：调用上面的存储过程，该调用不返回任何结果，会将调用结果存储在相应的变量中。MySQL的变量需要以@起始。查看调用结果需要输入下面的语句：

SELECT @pricelow;

一个例子：

--Name:ordertotal
--Parameters: onumber = order number
--taxable = 0 if not taxable,1 if taxable
--ototal = order total variable
CREATE PROCEDURE ordertotal(
		IN onumber  INT,
		IN taxable     BOOLEAN,
		OUT ototal DECIMAL(8,2)
)COMMENT ‘Obtain order total, optionally adding tax’
BEGIN
		--Declare variables
		DECLARE total DECIMAL(8,2)；
		DECLARE taxrate INT DEFAULT 6;

		--Get the order total
		SELECT SUM(item_price*quantity)
		FROM orderitems
		WHERE order_num = onumber
		INTO total;

		--Is this taxable?
		IF taxable THEN
			SELECT total+(total/100*taxrate) INTO total
		END IF;

		SELECT total INTO ototal;
END

结果：创建一个具有以下功能的存储过程：
计算订单总和；
对于需要计税的订单，计算税值并添加到总和；
返回合计。
’–’开头的内容是注释；
DECLARE语句定义局部变量；
COMMENT的内容将在输入SHOW PROCEDURE STATUS时显示，该命令获得包括何时、由谁创建等详细信息的存储过程列表；
SHOW CREATE PROCEDURE ordertotal 则显示创建ordertotal的CREATE语句。

（3）游标

游标： 是一个储存在MySQL服务器上的数据库查询检索出来的结果集，存储了游标之后，应用程序可以根据需要滚到或浏览其中的数据，通常应用于交互式应用。在使用前必须声明，显式地打开和关闭游标。

CREATE PROCEDURE processorders()
BEGIN 
	DECLARE ordernumbers CURSOR
	FOR
	SELECT order_num FROM orders
END;

结果：创建一个存储过程，其中使用DECLARE语句定义了一个游标(CURSOR)，名为ordernumbers。

OPEN ordernumbers;

结果：打开游标ordernumbers.在处理OPEN语句时进行查询，存储检索出来的数据以供浏览和滚动。

CLOSE ordernumbers;

结果：关闭游标ordernumbers ，释放游标所使用的内存和资源。

CREATE PROCEDURE processorders()
	BEGIN
		--Declare local variables
		DECLARE done BOOLEAN DEFAULT 0;
		DECLARE o INT;
		DECLARE t DECIMAL(8,2);

		--Declare the Cursor
		DECLARE ordernumbers CURSOR
		FOR
		SELECT order_num FROM orders;

		--Declare continue handler
		DECLARE CONTINUE HANDLER FOR SQLSTATE '02000' SET done = 1;
				
		--Create a table to store the results
		CREATE TABLE IF NOT EXISTS ordertotals
		(order_num INT, total DECIMAL(8,2));

		--Open the cursor
		OPEN ordernumbers;

		--Loop through all rows
		REPEAT
	
			--Get order number
			FETCH ordernumbers INTO o;

			--Get the total for this order
			CALL ordertotal(o,1,t);

			--Insert order and total into ordertotals
			INSERT INTO ordertotals(order_num,total)
			VALUES(o,t);
		--End of loop
		UNTIL done END REPEAT;

		--Close the cursor
		CLOSE ordernumbers;
	END;

输出：创建了存储过程，在过程中使用REPEAT来进行循环，直到done为真（即SQLSTATE ‘02000’，这是一个未找到条件）；
循环中使用FETCH来检索当前行的ordernumbers列 到一个名为o的局部变量中，定义了一个局部变量t，存储订单总计；
还创建了一个新表ordertotals，用CALL调用另一个存储过程计算含税总计存储到t，最后保存在新表中。

（4）触发器

触发器： 是MySQL响应(DELETE, INSERT,UPDATE)自动执行的一条语句。

CREATE TRIGGER newproduct AFTER INSERT ON products
FOR EACH ROW SELECT 'Product added';

结果：创建一个名为newproduct的新触发器，AFTER 指示触发器将在INSERT操作执行之后在执行(BEFORE则在操作执行之前)。

DROP TRIGGER newproduct;

结果：删除触发器
INSERT触发器： 触发器代码内可以引用一个名为NEW的虚拟表，访问被插入的行；
NEW中的值可以被更新；
对于AUTO_INCREMENT列，NEW在INSERT执行前包含0，执行后包含自动生成值。
DELETE触发器： 触发器代码内可以引用一个名为OLD的虚拟表，访问被删除的行。
UPDATE触发器： 触发器代码内可以引用一个名为OLD的虚拟表，访问被更新前的行，OLD表全部是只读的，不能更新；
引用一个名为NEW的虚拟表，访问被更新后的行，NEW的值也可以被更新。

七、 事务处理

MyISAM引擎不支持明确的事务处理，InnoDB则支持。事务处理是一种管理必须成批执行的MySQL操作，保证数据库不包含不完整的操作结果，即操作作为整体成功执行，或者完全不执行。

（1）ROLLBACK

用于回退MySQL语句(INSERT, UPDATE, DELETE,但SELECT不能回退)

SELECT * FROM ordertotals;
START TRANSACTION;
DELETE FROM ordertotals;
SELECT * FROM ordertotals;
ROLLBACK;
SELECT * FROM ordertotals;

结果：START TRANSACTION指示事务处理开始；
删除ordertotals中所有行，然后使用SELECT显示表为空；
用ROLLBACK语句回退START TRANSACTION所有语句。

（2）COMMIT

一般的MySQL语句直接针对数据库表执行和编写，这就是所谓的隐含提交。在事务处理块中，需要使用COMMIT显式地提交。

START TRANSACTION;
DELETE FROM orderitems WHERE order_num = 20010;
DELETE FROM orders WHERE order_num = 20010;
COMMIT;

结果：完全删除20010订单，在两个DELETE成功执行后显示提交结果。

（2）保留点

使用保留点进行部分提交或部分回退

SAVEPOINT delete1;

结果：创建保留点 delete1。

ROLLBACK TO delete1;

结果：回退到保留点delete1。
