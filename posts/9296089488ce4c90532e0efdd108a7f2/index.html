1. Ubuntu的基本配置

网络连接

在上方菜单栏里的找到虚拟机里，点击设置，然后选择网络适配器，在网络连接里选桥接模式，勾选复制物理网络连接状态，我们安装的Ubuntu系统就会有网了。

Ubuntu网络图标突然消失的问题的解决方案

基本上网络图标不见了都是Ubuntu的NetworkManager有问题

  1. 进入root

sudo su

  1. 更新NetworkManager的配置

vim /etc/NetworkManager/NetworkManager.conf

需要修改的配置文件是这样的，将flase改为True就可以了，如果你本身就是True的话，那就不用修改了，改好如下图；

  1. 删除原本的配置，并重启服务

service NetworkManager stop
rm /var/lib/NetworkManager/NetworkManager.state
service NetworkManager start

在完成上述操作后，原本消失的Ubuntu网络图标就会又突然出现了！

最后，建议重启一下Ubuntu，这能解决大多数问题，别问为什么！

当然不一定所有人的解决方法都是这样，如果这个方法解决不了你的问题，可以参考一下其它方法。

换源

我们找到软件与更新（Software Updater）--设置（Settings）--Ubuntu软件（Ubuntu Software）--下载自（Download form）。

然后选择：其它（Other...）往上划，选择中国（china）打开，在这里选择自己想要的源，我选择的是阿里云的，

选好后点击Choose Server，然后会让你输入密码，最后点击Close，至此完成换源。

安装Vmware Tools

方法1：在线安装

虚拟机有网络的情况下，安装Vmware Tools很简单，直接在线安装即可。

  1. 更新软件源列表

sudo apt-get update

  1. 更新软件

sudo apt-get upgrade

  1. 下载安装 open-vm-tools-desktop

sudo apt-get install open-vm-tools-desktop -y

（ -y 表示在需要确认的场景中回应 yes，无需再手动确认）

方法2：手动安装

当没有网络的情况下，采用手动安装是最好的选择

当我们安装好Ubuntu系统开机后，发现窗口并不是全屏，此时就需要安装VMware Tools工具了 ，

VMware附带一个映像ISO文件，该映像包含所有受支持的客户机操作系统的VMware Tools安装程序。 该文件位于主机上，可以从VMware GUI菜单在虚拟机系统上安装。 挂载后，即可开始在客户机系统上安装VMware Tools。

点击我们安装的Ubuntu系统，右键，查看VMware-Tool按钮状态，

如果像我的一样，VMware-Tools是显示灰色的，原因就是你的VMware并没有识别到你的VMware Tools的安装程序，那么就先配置CD/DVD（SATA）、CD/DVD2（SATA）、软盘为自动检测，尝试让VMware自己去识别VMware Tools的安装程序，

操作如下：

  1. 先关机；

  2. 关机以后调出如下图，然后点击设置；

然后依次点击检查下面的三项，如果这三项右侧不是“自动检测”，那么需要将这三项都设置成“自动检测”，改好的效果如下。

如果上面操作完毕，，我们再右键点击虚拟机--可移动设备--CD/DVD(SATA) --连接！！！，这一步必不可少，否则就会找不到VMware-Tools安装程序的压缩包，连接好的效果如下：

然后再点击 CD/DVD（SATA）右侧的 [浏览] 按钮，在文件目录中打开 VMware Workstation 的安装目录，然后选择 inux.iso 镜像文件，打开，

设置好点击确定以后就开机，开机后可能会出现类似于以下的弹窗，不用管，直接选是确定。

再查看VMware-Tools时，已经可以正常显示了，点击安装即可。

点击安装以后，我们开机后会发现桌面多了一个光盘

如果有这个光盘，说明上面点击的按钮生效了，那么我们右击这个光盘，然后点击 Open in Terminal

点击以后会进入一个命令行界面。我们使用ls命令查看一下当前目录中的文件，

其中红色的那个压缩包正是我们所需要的，然后我们将它复制到/usr/local/bin/目录下，

执行如下命令：

sudo cp ./VMwareTools-10.3.23-16594550.tar.gz /usr/local/bin/

注意：

可能由于版本问题，你的红色的那个压缩包的名字可能不同，因此命令中的压缩包名字要改成你自己的。

然后再进入到/usr/local/bin/目录，ls一下查看是否复制成功，

cd /usr/local/bin/
ls

出现下图中的红色箭头所指的内容，说明成功复制。

将压缩包解压，注意将命令中的压缩包名字要改成你自己的，

执行如下命令：

sudo tar -zxvf VMwareTools-10.3.23-16594550.tar.gz

注意：

解压过程终端会产生大量文件，不用管它们。

解压以后，我们ls一下，显示会多了一个vmware-tools-distrib文件夹

进入vmware-tools-distrib文件夹并ls查看

cd vmware-tools-distrib
ls

可以看到有10个文件（夹），

我们的目标是vmware-install.pl，它是一个安装程序

执行这个安装程序

sudo ./vmware-install.pl

出现如下图[no]，都输入y，Enter回车，如果出现[yes]，直接Enter回车，会有很多Enter回车需要按，

等待一会，出现下图红色框着的，然后原来的小屏幕自动地转变成了全屏，说明安装成功。如果安装成功以后屏幕还是不能放大，也不能使用复制黏贴功能，重启虚拟机或重新安装即可

2. Ubuntu常用命令

cd

功能：切换工作目录

cd 目标目录 ：切换到目标目录

cd .. ：切换回到上一级目录

注意：

如果cd 后面没有跟任何参数，则默认切换到当前登录用户HOME目录

pwd

功能：显示当前工作目录

mkdir

功能：创建一个新文件夹

mkdir 新文件夹名

ls

功能：显示指定目录下的所有文件

ls 指定目录

注意：

如果ls 后面没有跟任何参数，则默认显示当前工作目录下的所有文件

touch

功能：创建一个新文件

touch 新文件名

mv

功能：移动文件、文件夹

mv 参数1 参数2

参数1：被移动的目标

参数2：目标要被移动去的地方，参数2如果是另一个文件或 文件夹名，则会对参数1改名参数2

cp

功能：复制文件、文件夹

cp 参数1 参数2

参数1：被复制的目标文件

参数2：要被复制到的目标目录

cp -r 参数1 参数2

参数1：被复制的目标文件夹

参数2：要被复制到的目标目录

注意：

工作目录要在被复制的目标文件或文件的目录下才能复制成功，否则会找不到要复制的目标

rm

功能：删除一个或多个文件、文件夹

rm 参数1 参数2 ... 参数n

参数1 参数2 ... 参数n：被删除的目标文件

rm -r 参数1 参数2 ... 参数n

参数1 参数2 ... 参数n：被删除的目标文件夹

注意：

  • 删除一个或多个目标文件夹，会将其目录下的子目录及其文件均删除，删除文件夹请慎重；

  • 对于链接文件，只是删除了链接，原有文件均保持不变

su

功能：切换用户，

普通用户的权限，一般在其HOME目录内是不受限的，一旦出了HOME目录，大多数地方，普通用户仅有只读和执行权限，无修改权限。

su [-] [用户名]

- ：符号是可选的，表示是否在切换用户后加载环境变量，

建议带上；

用户名：表示要切换的用户，用户名也可以省略，省略表示

默认切换到root

注意：

  • 切换用户需要输入密码；

  • 切换用户后，可以通过exit命令退回上一个用户，也可以使用快捷键：ctrl + d

sudo

功能：让一条普通命令临时带有root权限

sudo 其它命令

示例：

更新软件源列表 sudo apt-get update
