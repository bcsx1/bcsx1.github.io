一、下载 MySQL

这里，我们直接选择 MySQL Community Server，进入之后我们根据需要选择想要安装的版本，这里我选择 5.7 的版本，同时需要根据自己的 Linux 系统版本选择相应的操作系统选项：

二、卸载旧的 MySQL

1、为什么要卸载旧的 MySQL

下载完成之后，先不要急着安装。由于某些原因，在本次安装 MySQL 之前，可能系统中已经有安装过 MySQL 或这还残余一些没有卸载成功的 MySQL 相关文件，这可能会给后面 MySQL 的安装带来各种问题，比如：如密码不能初始化、MySQL 执行失败等。

因此首先需要先卸载清除系统中与 MySQL 有关的文件。并且，如果想要重新安装 MySQL 的话也是需要先卸载删除 MySQL 再安装的。

2、卸载流程

2-1、检查电脑里是否安装了 MySQL

sudo yum list installed | grep mysql

或者

rpm -qa | grep -i mysql

如果命令的输出什么也没有，那就无需卸载。如果有，则需要进行卸载操作。

2-2、卸载 MySQL

sudo yum list installed | grep mysql | awk '{print $1}' | xargs sudo yum erase -y

注意： 此操作不可逆，请谨慎操作！

2-3、清除残余文件

卸载通常不是万能的，还需要手动删除与 MySQL 有关的文件，输入以下命令来查找相关文件：

sudo find / -name 'mysql*'

这些文件大致位于目录 /var/、/usr/、/etc/ 中，需要根据自己的实际情况来决定。如果在上面发现其它位置下需要删除的文件，也可以酌情删除。

输入以下命令来删除上面找到的那些残留文件，以目录 /var/、/usr/、/etc/ 为例（此操作不可逆，请谨慎操作）：

sudo find /var /usr /etc -name "mysql*" -exec rm -r {} \;

删除完成之后，再使用上面的查找命令，看看还能不能找到这些文件。


三、安装 MySQL

进入下载好的 MySQL 压缩包所在的目录，然后进行解压缩：

tar -xvf mysql-5.7.44-1.el7.x86_64.rpm-bundle.tar

解压之后可以得到一系列的 rpm 安装包：

mysql-community-libs-5.7.44-1.el7.x86_64.rpm
mysql-community-client-5.7.44-1.el7.x86_64.rpm
mysql-community-libs-compat-5.7.44-1.el7.x86_64.rpm
mysql-community-common-5.7.44-1.el7.x86_64.rpm
mysql-community-server-5.7.44-1.el7.x86_64.rpm
mysql-community-devel-5.7.44-1.el7.x86_64.rpm
mysql-community-test-5.7.44-1.el7.x86_64.rpm
mysql-community-embedded-5.7.44-1.el7.x86_64.rpm
mysql-community-embedded-compat-5.7.44-1.el7.x86_64.rpm
mysql-community-embedded-devel-5.7.44-1.el7.x86_64.rpm

核心的安装包是 mysql-community-server，在正常的情况下，使用命令 sudo yum localinstall mysql-community-server-5.7.44-1.el7.x86_64.rpm 即可。但并不能这样做，因为 MySQL 的各个组成安装包之间有复杂的依赖关系，而 mysql-community-server 并不是依赖的起点。

但我们可以通过以下命令可以自动解析本目录下各安装包的依赖关系，并自动按顺序安装：

sudo yum -y localinstall *.rpm

注意：

  • 虽然这是离线安装包，但安装时并非完全不需要网络。在初次安装时，操作系统可能会首先对软件仓库进行更新，此过程需要联网；
  • 千万不要自己手动去安装各个依赖包，因为依赖包数目众多，依赖关系复杂，手动安装将会遭遇各种问题。

安装完成后查看是否安装成功：

$ mysql --version
mysql  Ver 14.14 Distrib 5.7.44, for Linux (x86_64) using  EditLine wrapper

四、配置 MySQL

1、运行 MySQL

查看 MySQL 运行状态：

systemctl status mysqld

如果 MySQL 服务没有运行，则输入以下命令启动 MySQL：

systemctl start mysqld

如果想要开机自动启动的话，则输入以下命令：

systemctl enable mysqld

2、设置数据库密码

安装完成之后，需要设置数据库密码。在第一次启动的时候，MySQL 会自动生成一个随机密码。生成的密码在文件 /var/log/mysqld.log 中，输入以下命令可快速查看：

$ grep 'temporary password' /var/log/mysqld.log
2020-06-20T12:53:03.077653Z 1 [Note] A temporary password is generated for root@localhost: #drisVEg4ekW

可以看出，初始临时密码是：#drisVEg4ekW

注意：MySQL 关于此文件 /var/log/mysqld.log 的初始密码的只会生成一次。要尽快备份此文件直到修改密码之前。不然一旦忘记密码，就只能先彻底删除 MySQL，然后再重新安装它。只是简单地卸载 MySQL 并不会重置 MySQL 的密码！

有了密码之后，我们登录 MySQL：

$ mysql -u root -p
Enter password: #drisVEg4ekW

首次登录后，需要修改密码才能使用 MySQL 的其它命令，修改密码命令如下：

mysql> ALTER USER 'root'@'localhost' IDENTIFIED BY '123456';

注意：MySQL 默认要求设置的密码对安全系数要求较高，密码设置时必须包含大小写字母、特殊符号、数字，并且长度大于8位，如不满足以上条件则会报密码策略问题异常信息：

ERROR 1819 (HY000): Your password does not satisfy the current policy requirements

如果想要设置简单的密码的话，就需要修改密码策略，查看 MySQL 初始的密码策略：

mysql> show variables like 'validate_password%';
+--------------------------------------+--------+
| Variable_name                        | Value  |
+--------------------------------------+--------+
| validate_password_check_user_name    | OFF    |
| validate_password_dictionary_file    |        |
| validate_password_length             | 8      |
| validate_password_mixed_case_count   | 1      |
| validate_password_number_count       | 1      |
| validate_password_policy             | MEDIUM |
| validate_password_special_char_count | 1      |
+--------------------------------------+--------+
7 rows in set (0.01 sec)

关于 mysql 密码策略相关参数；

  • validate_password_length ：固定密码的总长度；
  • validate_password_dictionary_file：指定密码验证的文件路径；
  • validate_password_mixed_case_count： 整个密码中至少要包含大/小写字母的总个数；
  • validate_password_number_count ：整个密码中至少要包含阿拉伯数字的个数；
  • validate_password_policy：指定密码的强度验证等级，默认为 MEDIUM；
  • validate_password_special_char_count：整个密码中至少要包含特殊字符的个数；

关于 validate_password_policy 的取值：

  • LOW：只验证长度；
  • MEDIUM：验证长度、数字、大小写、特殊字符；
  • STRONG：验证长度、数字、大小写、特殊字符、字典文件；

首先需要设置密码的验证强度等级，设置 validate_password_policy 的全局参数为 LOW 即可:

mysql> set global validate_password_policy=LOW;
Query OK, 0 rows affected (0.00 sec)

当前密码长度为 8 ，如果不介意的话就不用修改了，按照通用的来讲，设置为 6 位的密码，设置 validate_password_length 的全局参数为 6 即可:

mysql> set global validate_password_length=6; 
Query OK, 0 rows affected (0.00 sec)

现在可以为 mysql 设置简单密码了，只要满足六位的长度即可:

mysql> ALTER USER 'root'@'localhost' IDENTIFIED BY '123456';
Query OK, 0 rows affected (0.00 sec)

或者

mysql> SET password=password('123456');
Query OK, 0 rows affected, 1 warning (0.00 sec)

这里建议不要将密码策略修改太低，尽量设置个复合默认要求的密码。

3、修改 MySQL 相关配置

MySQL 的配置文件在 /etc/my.cnf ，修改前尽量先备份一下配置文件：

cp /etc/my.cnf /etc/my_back.cnf

我们可以编辑配置文件 /etc/my.cnf 里面的的字符集编码：

[client]
# 设置mysql客户端默认字符集
default-character-set=utf8

[mysqld]
# 设置服务端字符集
character-set-server=utf8
# 设置端口为3306
port=3306
# 设置允许的最大连接数
max_connections=20
# 设置创建新表时的默认存储引擎
default-storage-engine=INNODB

其他默认配置文件路径：

  • 配置文件：/etc/my.cnf
  • 日志文件：/var/log/mysqld.log
  • 服务启动脚本：/usr/lib/systemd/system/mysqld.service
  • socket文件：/var/run/mysqld/mysqld.pid
  • 数据库存放路径：/var/lib/mysql/
