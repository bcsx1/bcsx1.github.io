DQL

DQL-介绍

Data Query Language(数据查询语言),数据查询语言,用来查询数据库中表的记录。

关键字：select

主要介绍单表查询

select 
	字段列表
from
	表名列表
where
	条件列表
group by
	分组字段列表
having
	分组后条件列表
order by
	排序字段列表
limit
	分页参数

由浅入深，从基本查询到条件查询，再到聚合，再到分组

基本查询

查询多个字段

select 字段1，字段2，字段3，... from 表名;
#查询全部
select * from 表名;

设置别名

select 字段1 [as 别名1]，字段2[as 别名2] ... from 表名;

去除重复记录

select distinct 字段列表 from 表名;

分组查询
select 字段列表 from 表名 [where 条件] group by 分组字段名 [Having 分组后过滤条件];
  • where分组之前过滤，不满足where条件不参与分组，而having是分组之后进行过滤
  • 判断条件不同，where不能对聚合函数进行判断，而having可以

练习：

根据性别分组，统计男性员工和女性员工的数量

多表查询

表之间主要分为三种关系：

  • 一对多 部门对多个员工
  • 多对多 课程对学生 需要建立中间表来包含两个外键，分别关联两方主键
  • 一对一 用户和用户详情 一张表放用户基础数据 另一张放详情字段 在任意一方加入外键关联另外一方的主键，并且设置外键为唯一的(unique)

多对多演示

create table student(
	id int auto_increment primary key comment '主键ID',
	name varchar(10) comment '姓名',
	no varchar(10) comment '学号'
)comment '学生表';
数据自行插入，懒得写了
create table course(
	id int auto_increment primary key comment '主键ID',
	name varchar(10) comment '课程名称'
)comment '课程表';

create table student_course(
	id int auto_increment comment '主键' primary key,
	studentid int not null comment '学生ID',
	courseid int not null comment '课程ID',
	constraint fk_courseid foreign key (courseid) references course (id),

开始练习多表查询

select * from emp,dept; #此时不加任何条件 会出现笛卡尔积 即a有两条数据 b有四条数据 a*b = 8条数据 需要加入条件消除笛卡尔积
select * from emp,dept where emp.dept_id = dept.id;
多表查询分类
  1. 连接查询：
    内连接：相当于查询ab交集部分数据
    外连接：左外连接：查询左表所有数据，以及两张表交集部分数据 ；右外连接：查询右表所有数据，以及两张表交集部分数据；
    自连接：当前表与自身的连接查询，自连接必须使用表别名
  2. 子查询
连接查询内连接

隐式内连接

select 字段名 from a,b where 条件 ...;

显式内连接

select 字段名 from a [inner] join b on 连接条件 ...;

内连接查询的是两张表交集部分

练习：

查询每个员工的姓名及关联部门的名称（隐式内连接）

select emp.name,dept.name from emp,dept where emp.dept_id = dept.id;

如果起了别名 后面不能使用原表名来指向表

显示内连接

select e.name,d.name from emp e inner join dept d on e.dept_id = d.id;

左外连接

select 字段列表 from 表1 left [outer] join 表2 on 条件...;

完全包含左表的数据和包含表一表二交集数据

右外连接

select 字段列表 from 表1 right [outer] join 表2 on 条件...;

练习

select * from emp e left join dept d on e.dept_id = d.id; 

select e.*,d.name from emp e right join dept d on e.dept_id = d.id;

事务

事务 是一组操作的集合，它是一个不可分割的工作单位，事务会把所有的操作作为一个整体一起向系统提交或撤销操作请求，即这些操作要么同时成功，要么同时失败。

例如银行取钱，张三取了一千块钱，转给李四一千块钱，如果转的时候失败，李四余额没有加一千，出于事务的机制，张三也不会减少他的一千块钱。事务回滚回了一开始的样子

模拟转账操作

#查询余额
select * from account where name ='张三';
#余额减一千
update account set money = money-1000 where name ='张三';
#李四余额加一千
update account set money = money + 1000 where name = ' 李四';

set @@autocommit = 0;

事务的四大特性ACID
  • 原子性 事务是不可分割的最小操作单元，要么全部成功，要么全部失败
  • 一致性 事务完成时，必须使所有的数据保持一致状态
  • 隔离性 数据库系统提供的隔离机制，保证事务在不受外部并发操作影响的独立环境下运行
  • 持久性 事务一旦提交或者回滚，它对数据的改变是永久的
