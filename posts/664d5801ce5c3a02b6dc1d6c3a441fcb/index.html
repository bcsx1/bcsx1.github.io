生产者
/**
 * 用于测试 queueProducer
 */
@Test
public void testQueueProducer() throws JMSException {
    // 1.创建工厂，指定url，用户名，密码
    ActiveMQConnectionFactory activeMQConnectionFactory = new ActiveMQConnectionFactory("admin", "admin", "tcp://localhost:61616");
    // 2.通过工厂获取连接，并开启连接
    Connection connection = activeMQConnectionFactory.createConnection();
    connection.start();

    // 3.创建session，两个参数，第一个是事务，第二个是签收
    Session session = connection.createSession(false, Session.AUTO_ACKNOWLEDGE);
    // 4.创建目的地（队列/主题）
    Queue queue = session.createQueue("heartNum");

    // 5.创建生产者
    MessageProducer producer = session.createProducer(queue);

    // 6. 通过messageProducer生产消息到MQ队列里面
    // 7.创建消息
    TextMessage textMessage = session.createTextMessage("msg-333");


    // 8通过sessageProducer发送给mq
    producer.send(textMessage);


    // 9.关闭资源，后开先关
    producer.close();
    session.close();
    connection.close();

}

五、Queue和Topic对比
1.工作模式
Topic

如果没有订阅者，消息将被丢弃，如果存在多个消费者，所有订阅的消费者都会收到消息。

Queue

如果没有消费者，消息暂存，不会丢弃。

如果有多个消费者，那么一条消息只会发送一个消费者，并且要求消费者签收ack。

多个消费者的接收以负载均衡默认轮询实现。

2.有无状态
Topic

无状态

Queue

默认在MQ服务器以文件保存，一般存储在MQ_HOME\datalkr-storeldata，也可以配置为DB

3.传递完整性
Topic

没有订阅者，消息会被丢弃

Queue

消息不会丢弃

4.处理效率
Topic

消息要按照订阅者的数量进行复制，性能处理会随着订阅者的增加而明显降低，并且要结合不同消息协议自身的性能差异

Queue

一条消息只发送给一个消费者，消费者再多性能也不会有明显降低。

五、JMS message

JMS message：一个JMS消息由消息头、消息属性、消息体三个结构组成

1.消息头
messageProducer提供的方法

  • JMSDestination() 设置消息的目的地是队列还是主题

  • JMSDeliveryMode() 设置消息的持久性

  • JMSExpiratio() 设置消息的过期时间，不设置默认永不过期

  • JMSPriorit() 消息的优先级 ， 0-9十个级别，0-4是普通消息，5-9是加急消息，默认4级，加急的消息会先于普通消息到达

  • JMSMessageID() 唯一识别每个消息的标识有MQ产生，每个消息都会带有一个ID ， 用来查看消息是否重复消费

2.消息体

  • TextMessage-----普通字符串消息，包含一个string

  • MapMessage--------一个Map类型的消息，key为string类型，而值为Java的基本类型 （set设置值）

  • BytesMessage------二进制数组消息，包含一个byte[]

  • StreamMessage--------Java数据流消息，用标准流操作来顺序的填充和读取。

  • ObjectMessage----------对象消息，包含一个可序列化的Java对象

3.消息属性

增强消息体，更加细粒度的设置消息

如果需要除消息头字段以外的值，那么可以使用消息属性

识别/去重/重点标注等操作非常有用的方法

 // 以前的正常设置
TextMessage textMessage = session.createTextMessage("TOPIC 消息 -->" + i );
// 设置一条消息的属性

Topic持久化

首先启动消费者，再启动生产者，表示以订阅生产者，此时，消费者下线了，没有持久化的topic消费者，再次上线是收不到不在线时生产者发布的消息的，（分布的消息没有订阅者接收到，成为了废消息），而持久化的topic消费者再次上线是可以收到消息的，比如微信，就算下线了，再次上线，也不会丢失信息

2.事务

生产者开启了事务，并commit提交了，就认为自动签收了，就算开启手动提交，没有确认签收，也不会重复消费。

生产者开启了事务，没有commit提交，开启手动提交，就算确认签收，也会造成重复消费

简单来说就是事务优先级更大

Session session = connection.createSession(false, Session.AUTO_ACKNOWLEDGE);

  • false —> 只要执行send，就进入到队列中

  • 关闭事务，那第2个签收参数的设置需要有效

  • true—> 先执行send再执行commit，消息才被真正的提交到队列中

  • 消息需要批量发送，需要缓冲区处理

  • 开启事务的原因是，批处理时保证同时成功/同时失败，如果出现错误，可以回滚( session.rollback() ) .

3.签收
三种状态

1.自动签收（默认） Session.AUTO_ACKNOWLEDGE

2.手动签收

​ Session.CLIENT_ACKNOWLEDGE

​ 客户端调用Acknowledge方法手动签收

3.允许重复消息

​ Session.DUPS_OK_ACKNOWLEDGE

签收编码
/**
     * 用于测试 签收
     */
@Test
public void topicTxCheck() throws JMSException, IOException {
    // 1.创建工厂，指定url，用户名，密码
    ActiveMQConnectionFactory activeMQConnectionFactory = new ActiveMQConnectionFactory("admin", "admin", "tcp://localhost:61616");
    // 2.通过工厂获取连接，并开启连接
    Connection connection = activeMQConnectionFactory.createConnection();

    connection.start();

    // 3.创建session，两个参数，第一个是事务，第二个是签收
    Session session = connection.createSession(true, Session.CLIENT_ACKNOWLEDGE);

    // 4.创建目的地（队列/主题）
    Queue queue = session.createQueue("tx-queue");

    MessageConsumer consumer = session.createConsumer(queue);
    Message receive = consumer.receive();
    int i = 0;
    while(receive != null){
        i++;
        TextMessage textMessage = (TextMessage) receive;
        // 部分签收
        if(i >2){
            receive.acknowledge();
        }
        System.out.println("textMessage = " + textMessage.getText());
        receive = consumer.receive(4000L);


    }

    //        事务不提交可能导致重复消费 （批量消费时）
    //  事务范围大于签收
    session.commit();

    // 9.关闭资源，后开先关
    consumer.close();
    session.close();
    connection.close();

}

九、三大投递

异步、延时、定时

慢消费者，快生产者，可以使用异步投递

异步发送:
它可以最大化produer端的发送效率。我们通常在发送消息量比较密集的情况下使用异步发送，它可以很大的提升Producer性能;不过这也带来了额外的问题，
就是需要消耗较多的Client端内存同时也会导致broker端性能消耗增加;
此外它不能有效的确保消息的发送成功。在useAsyncSend=true的情况下客户端需要容忍消息丢失的可能。

编码
/**
     * 用于测试 延时投递 重复投递 定时投递
     */
@Test
public void testQueueProducerDelayPeriodRepeat() throws JMSException, IOException {
    // 1.创建工厂，指定url，用户名，密码
    ActiveMQConnectionFactory activeMQConnectionFactory = new ActiveMQConnectionFactory("admin", "admin", "tcp://localhost:61616");
    // 2.通过工厂获取连接，并开启连接
    Connection connection = activeMQConnectionFactory.createConnection();
    connection.start();

    // 3.创建session，两个参数，第一个是事务，第二个是签收
    Session session = connection.createSession(false, Session.AUTO_ACKNOWLEDGE);
    // 4.创建目的地（队列/主题）
    Queue queue = session.createQueue("heartNumRepeat");

    // 5.创建生产者
    MessageProducer producer = session.createProducer(queue);

    // 6. 通过messageProducer生产消息到MQ队列里面

    // 延时
    long delay = 3 * 1000;
    // 周期间隔
    long period = 4 * 1000;
    // 重复次数
    int repeat = 5 ;

    // 7.创建消息
    TextMessage textMessage = session.createTextMessage("msg-333");
    System.out.println("textMessage = " + textMessage.getText());

    textMessage.setLongProperty(ScheduledMessage.AMQ_SCHEDULED_DELAY,delay);
    textMessage.setLongProperty(ScheduledMessage.AMQ_SCHEDULED_PERIOD,period);
    textMessage.setIntProperty(ScheduledMessage.AMQ_SCHEDULED_REPEAT,repeat);

    // 8通过sessageProducer发送给mq
    producer.send(textMessage);


    System.in.read();
    // 9.关闭资源，后开先关
    producer.close();
    session.close();
    connection.close();

}

异步发送可能会消息丢失，想要知道哪些消息被成功发送哪些丢失了，所以

为了确保消息被成功发送，需要更细粒度的创建消息的生产，生产后发送，需要回调消息（类似手机信息的已读回执）

延时投递、定时投递需要更改配置文件 activemq.xml

十、消息重试机制

具体哪些情况会引起消息重发
1 : Client用了transactions且在session中调用了rollback()
2 : Client用了transactions且在调用commit()之前关闭或者没有commit
3 : Client在CLIENT_ACKNOWLEDGE的传递模式下，在session中调用了recover()

一个消息被redelivedred超过默认的最大重发次数（默认6次）时，

标记”poison ack”表示这个消息有毒

broker会把这个消息放到DLQ（死信队列）。

编码测试
/**
     * 用于测试 消息重试机制,手动调用6次，发现消息重试机制为6次，时间间隔为1秒，如果都无法消费成功，则标记有毒，放入死信队列
     */
@Test
public void queueConsumerRepeat() throws JMSException, IOException {
    // 1.创建工厂，指定url，用户名，密码
    ActiveMQConnectionFactory activeMQConnectionFactory = new ActiveMQConnectionFactory("admin", "admin", "tcp://localhost:61616");
    // 2.通过工厂获取连接，并开启连接
    Connection connection = activeMQConnectionFactory.createConnection();
    connection.start();

    // 3.创建session，两个参数，第一个是事务，第二个是签收
    Session session = connection.createSession(true, Session.AUTO_ACKNOWLEDGE);
    // 4.创建目的地（队列/主题）
    Queue queue = session.createQueue("heartNumRepeat");

    // 5.创建消费者
    MessageConsumer consumer = session.createConsumer(queue);


    consumer.setMessageListener(message -> {

        try {
            if(message != null){
                TextMessage textMessage =  (TextMessage) message;
                String text = textMessage.getText();
                System.out.println("text = " + text);
            }
        } catch (JMSException e ){
            e.printStackTrace();
        }

    });


    System.in.read();
    // 9.关闭资源，后开先关
    consumer.close();
    session.close();
    connection.close();

}
