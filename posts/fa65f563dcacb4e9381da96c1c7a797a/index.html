互斥量

互斥量又叫相互排斥的信号量，是一种特殊的二值信号量。互斥量类似于只有一个车位的停车场：当有一辆车进入的时候，将停车场大门锁住，其他车辆在外面等候。当里面的车出来时，将停车场大门打开，下一辆车才可以进入。

一、互斥量工作机制

互斥量的状态只有两种，开锁或闭锁。用互斥量处理不同线程对临界资源的同步访问时，获取互斥量才可以对资源进行访问，互斥量的所有权只能由一个线程所掌握，其他线程无法干涉。当线程1持有时，互斥量的状态为闭锁状态，其他线程禁止访问；线程1释放互斥量时，互斥量的状态为开锁状态，线程1失去所有权，所有线程要使用资源，按照优先级获取互斥量。

1、优先级反转问题

优先级反转问题：当使用信号量时，高优先级的线程尝试通过信号量访问资源时，如果当信号量已经被低优先级访问，低优先级运行程序时被中优先级线程卡住，无法运行完释放信号量，导致高优先级无法从低优先级手中拿取信号量，违背了线程按照优先级大小使用资源的初衷。
如果低优先级被更高优先级卡住，无法运行完，这不算优先级反转

2.优先级继承机制

优先级继承算法：暂时提高某个占有某种资源的低优先级线程的优先级，使之与在所有等待该资源的线程中优先级最高那个线程的优先级相等，而当这个低优先级线程执行完毕释放该资源时，优先级重新回到初始设定值。因此，继承优先级的线程避免了系统资源被任何中间优先级的线程抢占。

1.优先级反转

优先级反转（A线程表示这个大哥当不了）
见图分析


系统中有A、B、C三个线程，优先级依次降低

  1. A、B线程等待事件触发，C线程被事件触发，C获取共享资源M
  2. A线程被事件触发，插入就绪链表，因为最高优先级，A线程运行尝试获取M资源，因为M被C线程占据，A线程挂起等待C线程释放M
  3. C线程运行一段时间后，B线程就绪，优先级比C线程高并且不需要使用M资源，C线程让出处理器，B线程运行
  4. B线程运行完，A线程启动无M资源挂起（相当于A线程没运行），C线程继续运行知道释放M资源
  5. A线程启动获得M资源，继续运行
    总结A线程因为M资源的原因，无法运行，而占据M资源的C线程，因为优先级低，需要让出处理器给不需要M资源的B线程。A线程表示我等C线程就算了，毕竟M资源（人质）在他手上，B线程你算什么，要老子等你

2.优先级继承

优先级继承（C线程的狐假虎威）
见图分析


系统中有A、B、C三个线程，优先级依次降低

  1. A、B线程等待事件触发，C线程被事件触发，C获取共享资源M
  2. A线程被事件触发，插入就绪链表，因为最高优先级，A线程运行尝试获取M资源，因为M被C线程占据，A线程挂起等待C线程释放M
  3. 将C线程的优先级存在互斥量控制块中的original_priority，A线程给C线程相同的优先级
  4. B线程就绪，C优先级=A优先级>B优先级，B无法抢占，C线程继续运行
  5. C线程运行完释放M资源，并且取回自己优先级，按照优先级链表排序，A线程运行，A获取M资源
  6. A线程运行完，才轮到B线程
    总结A线程因为M资源在C线程手中，忍痛给C线程相同优先级，C线程狐假虎威让B线程不敢妄动，B线程表示看在A线程面子上，让你一次

struct rt_mutex
{
    struct rt_ipc_object parent;     //继承自 ipc_object 类

    rt_uint16_t          value;       //互斥量的值

    rt_uint8_t           original_priority;    //持有线程的原始优先级
    rt_uint8_t           hold;                //持有线程的持有次数

    struct rt_thread    *owner;               //当前拥有互斥量的线程
};
typedef struct rt_mutex *rt_mutex_t;

3初始化和脱离

初始化
静态互斥量对象的内存是在系统编译时由编译器分配的，一般放于读写数据段或未初始化数据段中。在使用这类静态互斥量对象前，需要先进行初始化

rt_err_t rt_mutex_init(rt_mutex_t mutex, const char *name, rt_uint8_t flag)
{
    /* parameter check */
    RT_ASSERT(mutex != RT_NULL);//参数核查是否为互斥量

    /* init object */
    rt_object_init(&(mutex->parent.parent), RT_Object_Class_Mutex, name);//初始化互斥量对象

    /* init ipc object */
    rt_ipc_object_init(&(mutex->parent));//初始化ipc对象

    mutex->value = 1;                      //互斥量值为1
    mutex->owner = RT_NULL;                //持有互斥量线程为空
    mutex->original_priority = 0xFF;       //持有线程原始优先级为0xff
    mutex->hold  = 0;                      //持有数为0

    /* set flag */
    mutex->parent.parent.flag = flag;  //设置标志位

    return RT_EOK;
}
RTM_EXPORT(rt_mutex_init);

脱离（不删内存）从对象容器中脱离

rt_err_t rt_mutex_detach(rt_mutex_t mutex)
{
    /* parameter check */
    RT_ASSERT(mutex != RT_NULL);//参数核查是否为互斥量
    RT_ASSERT(rt_object_get_type(&mutex->parent.parent) == RT_Object_Class_Mutex);//判断对象是否为互斥量
    RT_ASSERT(rt_object_is_systemobject(&mutex->parent.parent));//判断对象是否为系统对象（静态对象）

    /* wakeup all suspend threads */
    rt_ipc_list_resume_all(&(mutex->parent.suspend_thread));//唤醒所有等待此互斥量的线程

    /* detach semaphore object */
    rt_object_detach(&(mutex->parent.parent));//脱离信号量对象

    return RT_EOK;
}
RTM_EXPORT(rt_mutex_detach);

4创建和删除

创建
自己从内存中分配一个互斥锁对象（初始化是系统分配好了内存）

rt_mutex_t rt_mutex_create(const char *name, rt_uint8_t flag)
{
    struct rt_mutex *mutex;

    RT_DEBUG_NOT_IN_INTERRUPT;//确保此函数不是运行在中断例程内

    /* allocate object */
    mutex = (rt_mutex_t)rt_object_allocate(RT_Object_Class_Mutex, name);//分配内存
    if (mutex == RT_NULL)//判断分配是否成功
        return mutex;

    /* init ipc object */
    rt_ipc_object_init(&(mutex->parent)); //初始化互斥量ipc对象

    mutex->value = 1;        //设置互斥量值为1
    mutex->owner = RT_NULL; //设置互斥量拥有线程为空
    mutex->original_priority = 0xFF;  //设置拥有线程原优先级值为0xff
    mutex->hold  = 0;        //设置持有互斥锁的线程数为0
    
   /* set flag */
    mutex->parent.parent.flag = flag;//设置互斥量标志位

    return mutex;
}
RTM_EXPORT(rt_mutex_create);

删除

rt_err_t rt_mutex_delete(rt_mutex_t mutex)
{
    RT_DEBUG_NOT_IN_INTERRUPT;//确保此函数不是运行在中断例程内

    /* parameter check */
    RT_ASSERT(mutex != RT_NULL);//参数核查是否为互斥量
    RT_ASSERT(rt_object_get_type(&mutex->parent.parent) == RT_Object_Class_Mutex);//判断对象是否为互斥量
    RT_ASSERT(rt_object_is_systemobject(&mutex->parent.parent) == RT_FALSE);//对象不是系统对象（静态对象）

    /* wakeup all suspend threads */
    rt_ipc_list_resume_all(&(mutex->parent.suspend_thread));//唤醒所有等待此互斥量的线程

    /* delete semaphore object */
    rt_object_delete(&(mutex->parent.parent));//删除互斥量对象，释放内存

    return RT_EOK;
}
RTM_EXPORT(rt_mutex_delete);

5 获取互斥量

  1. 线程设置了等待时间，等待时间到了，没有资源，返回超时
  2. 线程获取函数设置time为0，即时要求信号量，未果，返回超时
rt_err_t rt_mutex_take(rt_mutex_t mutex, rt_int32_t time)
{
    register rt_base_t temp;//中断标志位
    struct rt_thread *thread;//线程指针

    /* this function must not be used in interrupt even if time = 0 */
    RT_DEBUG_IN_THREAD_CONTEXT;//此函数在关中断中使用

    /* parameter check */
    RT_ASSERT(mutex != RT_NULL);//参数核查是否为互斥量
    RT_ASSERT(rt_object_get_type(&mutex->parent.parent) == RT_Object_Class_Mutex);//判断对象是否为互斥量

    /* get current thread */
    thread = rt_thread_self();获取当前线程

    /* disable interrupt */
    temp = rt_hw_interrupt_disable();//关中断

    RT_OBJECT_HOOK_CALL(rt_object_trytake_hook, (&(mutex->parent.parent)));//调用回调函数

    RT_DEBUG_LOG(RT_DEBUG_IPC,
                 ("mutex_take: current thread %s, mutex value: %d, hold: %d\n",
                  thread->name, mutex->value, mutex->hold));//打印日志

    /* reset thread error */
    thread->error = RT_EOK;//将线程错误码设置为RT_EOK

    if (mutex->owner == thread)//当前线程持有互斥量
    {
        /* it's the same thread */
        mutex->hold ++;//持有线程的持有次数加一
    }
    else
    {
__again:
        /* The value of mutex is 1 in initial status. Therefore, if the
         * value is great than 0, it indicates the mutex is avaible.
         */
        if (mutex->value > 0)//互斥量值大于0
        {
            /* mutex is available */
            mutex->value --;//互斥量值减一

            /* set mutex owner and original priority */
            mutex->owner             = thread;//互斥量拥有线程为当前线程
            mutex->original_priority = thread->current_priority;//线程存储自己的优先级
            mutex->hold ++;//持有线程的持有次数加一
        }
        else
        {
            /* no waiting, return with timeout */
            if (time == 0)//不等了，返回超时
            {
                /* set error as timeout */
                thread->error = -RT_ETIMEOUT;

                /* enable interrupt */
                rt_hw_interrupt_enable(temp);//开中断

                return -RT_ETIMEOUT;
            }
            else
            {
                /* mutex is unavailable, push to suspend list */
                RT_DEBUG_LOG(RT_DEBUG_IPC, ("mutex_take: suspend thread: %s\n",
                                            thread->name));//打日志

                /* change the owner thread priority of mutex */
                if (thread->current_priority < mutex->owner->current_priority)
                //当前线程优先级比持有互斥量优先级高，执行优先级继承
                {
                    /* change the owner thread priority *///持有互斥量线程优先级（低）变成当前线程优先级（高）
                    rt_thread_control(mutex->owner,
                                      RT_THREAD_CTRL_CHANGE_PRIORITY,
                                      &thread->current_priority);
                }

                /* suspend current thread *///当前线程挂起，插入此互斥量的挂起链表
                rt_ipc_list_suspend(&(mutex->parent.suspend_thread),
                                    thread,
                                    mutex->parent.parent.flag);

                /* has waiting time, start thread timer */
                if (time > 0)//互斥量没有，可以等待time
                {
                    RT_DEBUG_LOG(RT_DEBUG_IPC,
                                 ("mutex_take: start the timer of thread:%s\n",
                                  thread->name));

                    /* reset the timeout of thread timer and start it */
                    rt_timer_control(&(thread->thread_timer),
                                     RT_TIMER_CTRL_SET_TIME,
                                     &time);//线程定时器设置时间
                    rt_timer_start(&(thread->thread_timer));//启动定时器
                }

                /* enable interrupt */
                rt_hw_interrupt_enable(temp);//开中断

                /* do schedule */
                rt_schedule();//启动调度器

                if (thread->error != RT_EOK)//线程错误码不是RT_EOK，说明此线程运行出问题
                {
                	/* interrupt by signal, try it again */
                	if (thread->error == -RT_EINTR) goto __again;//线程获取互斥量因为中断出问题，重新获取，跳到__again处

                    /* return error */
                    return thread->error;
                }
                else
                {
                    /* the mutex is taken successfully. */
                    /* disable interrupt */
                    temp = rt_hw_interrupt_disable();//开中断
                }
            }
        }
    }

    /* enable interrupt */
    rt_hw_interrupt_enable(temp);//开中断

    RT_OBJECT_HOOK_CALL(rt_object_take_hook, (&(mutex->parent.parent)));//设立回调函数

    return RT_EOK;
}
RTM_EXPORT(rt_mutex_take);

6释放互斥量

rt_err_t rt_mutex_release(rt_mutex_t mutex)
{
    register rt_base_t temp;//中断标志位
    struct rt_thread *thread;//线程指针
    rt_bool_t need_schedule;//调度锁

    /* parameter check */
    RT_ASSERT(mutex != RT_NULL);//参数核查是否为互斥量
    RT_ASSERT(rt_object_get_type(&mutex->parent.parent) == RT_Object_Class_Mutex);
    //初始化互斥量对象

    need_schedule = RT_FALSE;//调度锁为关

    /* only thread could release mutex because we need test the ownership */
    RT_DEBUG_IN_THREAD_CONTEXT;//此函数在关中断中使用

    /* get current thread */
    thread = rt_thread_self();//获得当前线程

    /* disable interrupt */
    temp = rt_hw_interrupt_disable();//关中断

    RT_DEBUG_LOG(RT_DEBUG_IPC,
                 ("mutex_release:current thread %s, mutex value: %d, hold: %d\n",
                  thread->name, mutex->value, mutex->hold));//打印日志

    RT_OBJECT_HOOK_CALL(rt_object_put_hook, (&(mutex->parent.parent)));//回调函数

    /* mutex only can be released by owner */
    if (thread != mutex->owner)//当前线程不是互斥量拥有者，返回错误
    {
        thread->error = -RT_ERROR;

        /* enable interrupt */
        rt_hw_interrupt_enable(temp);

        return -RT_ERROR;
    }

    /* decrease hold */
    mutex->hold --;//持有互斥量的线程持有数减一
    /* if no hold */
    if (mutex->hold == 0)//持有线程不再持有时
    {
        /* change the owner thread to original priority */
        if (mutex->original_priority != mutex->owner->current_priority)//优先级继承的优先级改回去
        {
            rt_thread_control(mutex->owner,
                              RT_THREAD_CTRL_CHANGE_PRIORITY,
                              &(mutex->original_priority));
        }

        /* wakeup suspended thread */
        if (!rt_list_isempty(&mutex->parent.suspend_thread))//互斥量释放了，轮到下一个等待互斥量的挂起线程
        {
            /* get suspended thread */
            thread = rt_list_entry(mutex->parent.suspend_thread.next,
                                   struct rt_thread,
                                   tlist);//获得挂起线程

            RT_DEBUG_LOG(RT_DEBUG_IPC, ("mutex_release: resume thread: %s\n",
                                        thread->name));

            /* set new owner and priority *///更改互斥量参数
            mutex->owner             = thread;
            mutex->original_priority = thread->current_priority;
            mutex->hold ++;

            /* resume thread */
            rt_ipc_list_resume(&(mutex->parent.suspend_thread));//唤醒等待互斥量的线程

            need_schedule = RT_TRUE;//调度锁开启
        }
        else   //没有线程在等待互斥量
        {
            /* increase value */
            mutex->value ++;

            /* clear owner */
            mutex->owner             = RT_NULL;
            mutex->original_priority = 0xff;
        }
    }

    /* enable interrupt */
    rt_hw_interrupt_enable(temp);

    /* perform a schedule */
    if (need_schedule == RT_TRUE)//没有更换互斥量持有线程，就不用调度了
        rt_schedule();

    return RT_EOK;
}
RTM_EXPORT(rt_mutex_release);
