boot - 覆盖第三方jar包中的一个类

场景

在引用第三方jar包时，有时候需要改动第三方jar中的一个类的方法，一种方式是通过继承的方式可以解决，但是，有些时候可能这个方法是 private 或者是 final 类型的，这时候就很尴尬

解决方案

现在工程 boot-mvc, 我想重写第三方 jar 包中的 SentinelWebInterceptor 类，只需要建立和SentinelWebInterceptor 所在 jar的一样的包目录即

├── pom.xml
├── src
│   ├── main
│   │   ├── java
│   │   │   └── com
│   │   │       ├── alibaba
│   │   │       │   └── csp
│   │   │       │       └── sentinel
│   │   │       │           └── adapter
│   │   │       │               └── spring
│   │   │       │                   └── webmvc
│   │   │       │                       └── SentinelWebInterceptor.java
│   │   │       └── x
│   │   │           └── z
│   │   │               └── bootmvc
│   │   │                   ├── BootMvcApplication.java
│   │   └── resources
│   │       ├── application.properties

为什么建立覆盖类所在 jar 一样的目录的 类就可以实现类覆盖呢？

  1. 这个要从 spring boot 打包成 jar 后运行加载类的机制说起，简单的来讲，spring boot 启动时，先加载当前工程的编译的class ,然后再加载第三方jar包的类，可以参考 spring boot jar 启动过程
  2. 那加载了当前目录的 SentinelWebInterceptor 类后， 第三方 jar 包会再加载 自己包下的 SentinelWebInterceptor 类吗？这个就要看 ClassLoader 的加载机制了。首先要知道， 当前 classpath 类下的所有包都由 AppClassLoader 加载的，除非一些类打破了双亲委派的机制，如使用线程上下文加载器。具体看下 ClassLoader 加载源码
class ClassLoader{
    protected Class<?> loadClass(String name, boolean resolve)
        throws ClassNotFoundException
    {
        synchronized (getClassLoadingLock(name)) {
            // First, check if the class has already been loaded
            Class<?> c = findLoadedClass(name);
            if (c == null) {
                long t0 = System.nanoTime();
                try {
                    if (parent != null) {
                        c = parent.loadClass(name, false);
                    } else {
                        c = findBootstrapClassOrNull(name);
                    }
                } catch (ClassNotFoundException e) {
                    // ClassNotFoundException thrown if class not found
                    // from the non-null parent class loader
                }

                if (c == null) {
                    // If still not found, then invoke findClass in order
                    // to find the class.
                    long t1 = System.nanoTime();
                    c = findClass(name);

                    // this is the defining class loader; record the stats
                    PerfCounter.getParentDelegationTime().addTime(t1 - t0);
                    PerfCounter.getFindClassTime().addElapsedTimeFrom(t1);
                    PerfCounter.getFindClasses().increment();
                }
            }
            if (resolve) {
                resolveClass(c);
            }
            return c;
        }
    }

}

主要看下 findLoadedClass 方法:

    /**
     * Returns the class with the given <a href="#binary-name">binary name</a> if this
     * loader has been recorded by the Java virtual machine as an initiating
     * loader of a class with that <a href="#binary-name">binary name</a>.  Otherwise
     * {@code null} is returned.
     * 如果 java虚拟机已经记录了这个类，这直接返回这个类的名称，不然返回 null  ！！！
     * @param  name
     *         The <a href="#binary-name">binary name</a> of the class
     *
     * @return  The {@code Class} object, or {@code null} if the class has
     *          not been loaded
     *
     * @since  1.1
     */
    protected final Class<?> findLoadedClass(String name) {
        if (!checkName(name))
            return null;
        return findLoadedClass0(name);
    }

大概的意思就是，如果 java虚拟机已经记录了这个类，这直接返回这个类的名称，不然返回 null， 也就是说，如果已经加载了 class ,就不会再加载！！这样就结合spring boot 加载 class 顺序，就可以解释为什么可以覆盖类实现重写了。当加载了当前 工程覆盖的 SentinelWebInterceptor 后，加载到内存， 因为包名和类名完全相同，ClassLoader在加载第三方 jar 包中的 SentinelWebInterceptor 时， 查看到内存中已经加载了这个类，就会会跳过第三方此类加载

  1. java 加载 classpath 每个jar 包的顺序可以参考文章 Setting the Class Path
