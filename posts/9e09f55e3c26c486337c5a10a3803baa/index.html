SQL Server 2000 SP4补丁打不上的解决办法

这两天在做数据库的作业，用java连sql server 2000，可是总也成功不了，mysql都可以，最后发现是sql 2000的1433端口打不开，必须得装sp4补丁，匆匆找来补丁，却还是装不上，忙活了整整一个晚上，还是没用，放弃了。今天觉得不甘心，又google了一下，没想到，意外的发现了这篇文章，竟然只需给电脑里加个1，问题全都解决，简直不可思议，看来还是不要轻言放弃，虽然有时候确实搞的人很烦，软件这东西，……，但是，换个关键字再搜索一下，多看看，也许，你也能像我一样，柳暗花明又一村！

 

另注：我的系统是XP Sp3，至于Sql Server是什么版本我也忘记了。同时，我也遇到文中所提到的“区域与语言选项”选项打之后是空的，不能添加语言的问题，通过下面的方法也修改好了。谢谢原创作者！

 

SQL server 2000 SP4补丁打不上的解决办法

这段时间在处理SQL server 2000 SP4补丁打不上的问题上花了不少时间，回头想想应该总结一下：

系统说明：dell1800服务器，操作系统：win2003sp2企业版

故障：前期，在安装sql2000企业版后，sql的sp4补丁打不上！这样局域网中的其它电脑就不能通过sql连接到这台服务器上的sql数据库；通过telnet 1433端口测试不通；

处理过程： 在网上找了若干资料，没有一个管用；无论如何打补丁时都提示sa密码验证错误；而事实上没有错！

后来找到了sql server黄皮书；打开一看果然有关于打不上补丁的帮助；说是sql sp3的一个bug 并且提供了解决办法！一时间如获至宝！但是，系统在我清理注册表中的sql垃圾时已经损坏（主要是因为朋友告诉我控制面板打不开，而且OFFICE2003安装时提示系统不支持该语言版本；因从未遇到过这样的情况，所以误以为是我删除注册表内容时损坏了系统；不过系统确实已经损坏了，因为sql已经装不上了，而且用sfc /scannow命令扫描一遍之后完全没有作用）!重装系统在所难免！为保险起见我决定格式化整个硬盘！

重装系统完成后发现依旧装不上OFFICE2003 我才开始重视这个问题，并在网上查到了答案；结果发现“区域与语言选项”选项打之后是空的，不能添加语言！后来终于在一个博客中找到了关于“区域与语言选项”及控制面板打不开的解决办法！

一试果然管用！而同时OFFICE和SQL补丁的问题同时解决了！呵呵

总结：当发现一个很难解决的问题时，要看看系统是否还有其它不正常的小毛病？如果有把这些小毛病解决掉；很可能惊喜的发现那个大难题也同时解决了：）

解决的办法：修改注册表：

[HKEY_LOCAL_MACHINE]→SYSTEM→CurrentControlSet→Control→Nls→Locale ；令：

"(Default)"="00000409" ；"00000804"="1"；问题解决！
