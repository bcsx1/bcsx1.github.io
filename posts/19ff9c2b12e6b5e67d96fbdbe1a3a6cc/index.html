现在写一个服务类：
注意:此处的@Service为dubbo包下的注解而非Spring的注解

代码如下：

package cn.redocloud.service;

import cn.redocloud.api.PayService;
import lombok.extern.slf4j.Slf4j;
import org.apache.dubbo.config.annotation.Service;

import java.util.Map;

/**
 * @Author: ChenTaoTao
 * @Date: 2021/12/29 19:24
 * @Describe:
 */
@Service
@Slf4j
public class PayServiceImpl implements PayService {
    @Override
    public void pay(Map map) {
        log.info("数据为{}",map);
    }
}

编写一个启动类

import org.apache.dubbo.config.spring.context.annotation.EnableDubbo;
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

/**
 * @Author: ChenTaoTao
 * @Date: 2021/12/29 19:42
 * @Describe:
 */
@SpringBootApplication
//scanBasePackages 这个就是业务类所在的包路径 启动dubbo
@EnableDubbo(scanBasePackages = "cn.redocloud.service")
public class PayApplication {
    public static void main(String[] args) {
        SpringApplication.run(PayApplication.class,args);
    }
}

注意事项：
以上内容使用了nacos的配置中心，所以需要你在nacos中添加部分配置文件如我的：

其中data id的命名必须与你工程中的命名对应，如：
application.yml

spring:
  profiles:
    active: dev
  application:
    name: userservice

bootstrap.yml:

spring:
  cloud:
    nacos:
      config:
        namespace: 9eeffe66-d561-4b00-a42c-38d13fd5cd09
        group: dev
        server-addr: 127.0.0.1:8848
        file-extension: yaml

那么你的配置文件的名字应该为：
userservice-dev.yaml （也就是application.name+profiles+file-extension的内容）并且需要严格区分namespace 和 group，file-extension为文件后缀名。
其他服务的配置以此类推。

dubbo的熔断机制：
在调用者这里实现一个熔断类（和FeignClient差不多）

@Slf4j
public class PayServiceFallback implements PayService {
    @Override
    public void pay(Map map) {
        log.info("服务请求超时");
    }
}

在@Refrence注解中设置：

    @Reference(mock = "cn.redocloud.fallback;",cluster = "failfast")
    PayService payService;
