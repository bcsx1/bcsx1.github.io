uni-app - 图片加载失败的各种场景

在uni-app开发过程中，图片加载失败是一个常见的问题。这可能是由于网络问题、图片资源不存在或者其他未知原因。在这篇文章中，我们将探讨如何处理uni-app中的图片加载失败问题，包括单个图片、v-for循环加载图片，以及图片加载失败后高度失效等等问题。

正常图片显示如下，当加载失败显示空，用户体验不好

image标签

1. 单个图片加载失败

情况1： 图片地址为空

当从后台获取到的图片地址我们不确定是否为空，可以考虑使用methods方法来生成图片地址。

<template>
	<view style="height: 100px; background-color: #fff;">
		<image :src="getImage()" class="image-style"></image>
	</view>
</template>

<script>
	export default {
		data() {

			return {
				imgUrl: ""

			}
		},
		methods: {


			getImage() {
				if (this.imgUrl) {
					return this.imgUrl
					console.log("../../static/images/1.jpg")

				}
				return "../../static/images/1.jpg"

			},

		}
	};
</script>

<style>
	.image-style {
		border: 1px solid #000;
		width: 80px;
		height: 80px;
	}
</style>

情况2 ：地址不为空，但无法加载

在uni-app中，我们可以使用@error事件来处理图片加载失败的问题。当图片加载失败时，@error事件会被触发。我们可以在这个事件中设置一个备用图片。

下面的代码，我imgUrl的初始值是错误的地址，图片无法加载，此时会触发@error事件，我们在@error事件给imgUrl一个备用的图片地址。

<template>
	<view style="height: 100px; background-color: #fff;">
		<image :src="imgUrl" @error="imgError" class="image-style"></image>
	</view>
</template>

<script>
	export default {
		data() {

			return {
				imgUrl: "../../static/images/"
			}
		},
		methods: {



			imgError(e) {
				console.log('加载失败', e)
				this.imgUrl = '../../static/images/noImage.png'; // 设置备用图片

			}
		}
	};
</script>

<style>
	.image-style {
		border: 1px solid #000;
		width: 80px;
		height: 80px;
	}
</style>

2. v-for循环加载图片

在v-for循环中加载图片时，我们需要为每个图片设置一个独立的@error事件处理器。

情况1 ：地址不为空，但无法加载

例如下面id为5的图片的地址是错误的地址，可以

<template>

	<view>
		<view v-for="(item,index) in imageList" :key="index">

			<image :src="item.url" @error="imgError($event, index)" class="image-style"></image>

		</view>

	</view>


</template>

<script>
	export default {
		data() {

			return {
				imgUrl: "../../static/images/1.jpg",

				imageList: [{
					id: 1,
					url: "../../static/images/1.jpg"
				}, {
					id: 2,
					url: "../../static/images/1.jpg"
				}, {
					id: 3,
					url: "../../static/images/1.jpg"
				}, {
					id: 4,
					url: ""
				}, {
					id: 5,
					url: "../../static/images/"
				}]

			}
		},
		methods: {



			imgError(e,i) {
				console.log('加载失败', e,i)
				this.imageList[i].url = '../../static/images/noImage.png'
			}
		}
	};
</script>

<style>
	.image-style {
		border: 1px solid #000;
		width: 80px;
		height: 80px;
	}
</style>

情况2： 图片地址为空

而对于id为4的图片，会显示空白，可以考虑使用方法来获取图片的地址。

<template>

	<view>
		<view v-for="(item,index) in imageList" :key="index">

			<image :src="getUrl(item)" @error="imgError($event, index)" class="image-style"></image>

		</view>

	</view>


</template>

<script>
	export default {
		data() {

			return {
				imgUrl: "../../static/images/1.jpg",

				imageList: [{
					id: 1,
					url: "../../static/images/1.jpg"
				}, {
					id: 2,
					url: "../../static/images/1.jpg"
				}, {
					id: 3,
					url: "../../static/images/1.jpg"
				}, {
					id: 4,
					url: ""
				}, {
					id: 5,
					url: "../../static/images/"
				}]

			}
		},
		methods: {


			getUrl(item) {
				if (item.url) {
					return item.url;
				} else {
					return '../../static/images/noImage.png'

				}
			},


			imgError(e, i) {
				console.log('加载失败', e, i)
				this.imageList[i].url = '../../static/images/noImage.png'
			}
		}
	};
</script>

<style>
	.image-style {
		border: 1px solid #000;
		width: 80px;
		height: 80px;
	}
</style>

3. widthFix模式下未设置高度，加载失败高度变为240px

在官方的说明文档中，image标签未设置宽高，加载失败的时候高度会变成240px。

解决方法

正常情况下：

<view style="height: 100px; background-color: #fff;" >
			<image :src="imgUrl"   class="image-style" mode="widthFix"></image>
	</view>

当图片加载失败，即使我们处理了加载失败，高度仍然会变为240px；

<template>
	<view style="height: 100px; background-color: #fff;">
		<image :src="imgUrl" @error="imgError" class="image-style"></image>
	</view>
</template>

<script>
	export default {
		data() {

			return {
				imgUrl: "../../static/images/"
			}
		},
		methods: {
			imgError(e) {
				console.log('加载失败', e)
				this.imgUrl = '../../static/images/noImage.png'; // 设置备用图片

			}
		}
	};
</script>

<style>
	.image-style {
		border: 1px solid #000;
		width: 80px; 
	}
</style>

想要解决这个问题，可以考虑把高度设置为auto。这样的效果仍然和widthFix一致。

在CSS中，将image的height属性设置为auto意味着图片的高度将自动调整以保持其原始宽高比。这样，无论图片的宽度如何变化，其高度都会相应地进行比例调整，以保持图片的原始形状不变。这对于响应式设计非常有用，可以确保在不同设备和屏幕尺寸上，图片都能以适当的比例显示。

.image-style {
    border: 1px solid #000;
    width: 80px;
    height: auto; 
}

4.使用@无法加载图片

src 仅支持相对路径、绝对路径

// imgUrl: '@/static/xxx.png',	// 不生效
// imgUrl: '~@/static/xxx.png',	// 不生效
imgUrl: '/static/xxx.png'		// 有效

如果非要使用@,可以考虑使用require

<image :src="require('@/static/images/1.jpg')"  @error="imgError" class="image-style"></image>

或者
<image :src="imgUrl"  @error="imgError" class="image-style"></image>
imgUrl: require("@/static/images/1.jpg")

5. css中无法加载图片

background 属性中，得使用下面写法引入本地图片：

background: url("~@/static/images/noImage.png");

否则图片无法被正确加载。
