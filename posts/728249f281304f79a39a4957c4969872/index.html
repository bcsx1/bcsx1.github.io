cron使用中遇到的问题

 

  项目中难免遇到一些服务组件挂掉,或者不工作的情况.所以想到写一些维护脚本来定期监控组件状态.所以想到了使用cron.

 

环境为 ubuntu系统.

安装cron组件 

apt-get install cron

启动cron 

service cron start

查看状态

service cron status

显示如下图就是启动成功了

ubuntu系统中,cron系统日志默然是没有开启的.如果想看需要配置

修改文件

vim /etc/rsyslog.d/50-default.conf

将里面有一行

#cron.*              /var/log/cron.log
修改成
cron.*              /var/log/cron.log

 然后重启两个服务即可在/var/log/cron.log中查看日志了

service rsyslog  restart
service cron restart

 

第一次使用cron来执行脚本会遇到一些问题,主要的集中在两个上面,一个是环境变量,还有一个是路径问题.

cron不会使用当前系统环境变量来执行脚本,需要执行时指定.并且路径也是一样的,cron默认路径是从root开始找的,所以使用cron来执行任务,路径最好都使用绝对路径.避免找不到的问题

例如:

 我使用cron检测kafka状态,挂掉后自动启动kafka

shell脚本为:check_kafka_alive.sh

#!/bin/sh
pid_count=$(netstat -tnlp | grep 9092 | wc -l) #检查kafka进程是否存活
date_current=$(date +%Y%m%d)
echo "current_date:${date_current},kafka pid count:${pid_count}" >> ${date_current}.log
if [ $pid_count -eq 0 ]
        then
        echo 'kafka server is died'
        /usr/local/kafka_2.11-1.0.1/bin/kafka-server-start.sh /usr/local/kafka_2.11-1.0.1/config/server.properties & #启动kafka
        pid=$(netstat -tnlp | grep 9092 | wc -l)
        echo "${pid}"      
fi

然后cron里面配置定时任务

*/1 * * * * . /etc/profile;/usr/local/kafka_2.11-1.0.1/bin/check_kafka_alive.sh >>/usr/local/kafka_2.11-1.0.1/bin/check_kafka.log 2>&1

*/1 * * * * cron表达式  表示没分钟执行一次

. /etc/profile; 这里就是在告诉cron 需要去读取环境变量

/usr/local/kafka_2.11-1.0.1/bin/check_kafka_alive.sh >>/usr/local/kafka_2.11-1.0.1/bin/check_kafka.log 表示执行shell脚本,并到日志输出到 指定文件中.

2>&1 最后的这句话,是cron在报错时,会把错误日志通过邮件系统发送出去.但如果系统中没有安装邮件系统,变通处理就可以这样写.
