WPF动画的技巧与注意点

动画结束后想保存结束时UI状态不变

FillBehavior
在动画对象中将FillBehavior 设置为Stop；还有一种可选值HoldEnd，即在控件Active状态下，会一直持有动画的结束状态；设置为Stop，只要动画完成，动画序列(可借助Blend查看)就结束，恢复到原始状态

想要保持结果UI状态，设置为HoldEnd即可，但此时无法再用外部方法修改动画绑定的属性，要想更改，把FillBehavior改为Stop即可，或者清空动画即可

动画后，手动更改属性无效

具体表现为：值确实改变了，但是UI效果没有变化

解决办法：

  1. FillBehavior
  2. 移除动画对象
  3. 清空指定属性动画对象
//Completed是完成后的事件
        <Storyboard>
          <ColorAnimation
            Storyboard.TargetName="Button2BackgroundBrush"
            Storyboard.TargetProperty="Color"
            From="Red" To="Yellow" Duration="0:0:5"
            FillBehavior="Stop"
            Completed="setButton2BackgroundBrushColor" />
        </Storyboard>

注意
但是若你在Animation的Completed事件中追加动画Begin的话，则FillBehavior的效果可能不尽人意，这是由于WPF的执行顺序问题，他会先执行完动画+Completed事件再进行FillBehavior，他会在上一个动画结束的基础上进行下一个动画
具体可见链接

移除动画对象
MyStoryboard是动画对象，Button3是作用的对象所属控件名称

 MyStoryboard.Remove(Button3);

清空指定属性动画对象
Button4BackgroundBrush是动画作用的对象名称

    Button4BackgroundBrush.BeginAnimation(SolidColorBrush.ColorProperty, null);

动画对象的清空

若要从特定属性中移除动画，请使用 BeginAnimation (DependencyProperty，AnimationTimeline)方法。指定被动画化的属性作为第一个参数，null作为第二个参数。这将从属性中删除所有动画时钟。
在通过导航定位到另一个窗口或页面时，最好清空不必要的动画对象，否则会即使切换了窗口或画面，那些循环的动画依然在进行计算，会浪费不必要的资源

动画的起点终点规划，From/To/By

From和To用于明确的起点和终点，如果只是想基于当前位置进行一定的位移的话，使用By，并且只写By
如果只是想基于当前位置移动到固定位置的话，只写To即可。
唯独From不能单独存在，否则没有作用

// Demonstrates the use of the By property.

// Create a NameScope for this page so that
// Storyboards can be used.
NameScope.SetNameScope(this, new NameScope());

Rectangle myRectangle = new Rectangle();

// Assign the Rectangle a name so that
// it can be targeted by a Storyboard.
this.RegisterName(
    "byAnimatedRectangle", myRectangle);
myRectangle.Height = 10;
myRectangle.Width = 100;
myRectangle.HorizontalAlignment = HorizontalAlignment.Left;
myRectangle.Fill = Brushes.RoyalBlue;

// Demonstrates the By property used by itself.
// Increments the Rectangle's Width property by 300 over 10 seconds.
// As a result, the Width property is animated from its base value
// (100) to 400 (100 + 300) over 10 seconds.
DoubleAnimation myDoubleAnimation = new DoubleAnimation();
myDoubleAnimation.By = 300;
myDoubleAnimation.Duration =
    new Duration(TimeSpan.FromSeconds(10));

Storyboard.SetTargetName(myDoubleAnimation, "byAnimatedRectangle");
Storyboard.SetTargetProperty(myDoubleAnimation,
    new PropertyPath(Rectangle.WidthProperty));
Storyboard myStoryboard = new Storyboard();
myStoryboard.Children.Add(myDoubleAnimation);

// Use an anonymous event handler to begin the animation
// when the rectangle is clicked.
myRectangle.MouseLeftButtonDown += delegate(object sender, MouseButtonEventArgs args)
    {
        myStoryboard.Begin(myRectangle);
    };

ControlTemplate中动画From/To/By的Binding

这种情况只会出现在ControlTemplate中，因为ControlTemplate中不支持DataBinding；
当你想用binding的方式绑定From或者To或者By的值时，运行时就会报错，而且是一打开就报错
要如何解决这个问题？比如我们想让label的宽度变成跟容器一样大，通过动画的形式
下面这样写会报错

  •   InnerException	{"无法冻结此 Storyboard 时间线树供跨线程使用。"}	System.Exception {System.InvalidOperationException}

<DoubleAnimation
    Storyboard.TargetName="label"
    Storyboard.TargetProperty="Width"
    To="{Binding ElementName=border,Path=ActualWidth}"
    Duration="0:0:0.5" />

**解决办法：**可以通过将目标控件的宽度绑定到要更改控件的Tag属性上，然后再将要改变控件的宽度绑定到自己的Tag上，通过MultiValueConverter来进行计算，相当于传入目标宽度和变化比例，得出需要变化的宽度，然后让Tag从0~1动画变化，就可以达到宽度动画变化到目标宽度

<Setter Property="Template">
            <Setter.Value>
                <ControlTemplate TargetType="Button">
                    <Border
                        x:Name="border">
                        <Grid>
                            <Rectangle
                                x:Name="rectangle">
                                <!--  初始值，否则报错  -->
                                <Rectangle.Tag>
                                    <sys:Double>0.0</sys:Double>
                                </Rectangle.Tag>
                                <Rectangle.Width>
                                    <MultiBinding Converter="{StaticResource multiplyConverter}">
                                        <!--  目标值  -->
                                        <Binding ElementName="border" Path="ActualWidth" />
                                        <!--  0~1的变化比例  -->
                                        <Binding Path="Tag" RelativeSource="{RelativeSource Self}" />
                                    </MultiBinding>
                                </Rectangle.Width>
                            </Rectangle>
                            <ContentPresenter  />
                        </Grid>
                    </Border>
                    <ControlTemplate.Triggers>
                        <Trigger Property="IsMouseOver" Value="True">
                             <Storyboard>
                                        <DoubleAnimation
                                            Storyboard.TargetName="rectangle"
                                            Storyboard.TargetProperty="Tag"
                                            To="1"
                                            Duration="0:0:1" />
                                    </Storyboard>
                                </BeginStoryboard>
                            </Trigger.EnterActions>
                            <Trigger.ExitActions>
                                <BeginStoryboard>
                                    <Storyboard>
                                        <DoubleAnimation
                                            Storyboard.TargetName="rectangle"
                                            Storyboard.TargetProperty="Tag"
                                            To="0"
                                            Duration="0:0:0.5" />
                                    </Storyboard>
                                </BeginStoryboard>
                            </Trigger.ExitActions>
                        </Trigger>

                    </ControlTemplate.Triggers>

                </ControlTemplate>
            </Setter.Value>
        </Setter>

Converter代码

public class MultiplyConverter : IMultiValueConverter
{
    public object Convert(object[] values, Type targetType, object parameter, CultureInfo culture)
    {
        double result = 1.0;
        for (int i = 0; i < values.Length; i++)
        {
            if (values[i] is double)
                result *= (double)values[i];
        }

        return result;
    }

    public object[] ConvertBack(
        object value,
        Type[] targetTypes,
        object parameter,
        CultureInfo culture
    )
    {
        throw new Exception("Not implemented");
    }
}

Storyboard完全自定义段落循环执行

要控制动画循环执行在xaml中有2种方式，一种是设置Storyboard的RepeatBehavior，还有一种设置Animation的RepearBehavior；
这两种的区别是Animation是针对单个动画对象，Storyboard是针对整个剧本的动画对象；
假设有如下动作，

  1. 动画调整FontSize
  2. 动画调整背景板Rectangle宽度递增
  3. 动画调整背景板Retangle宽度递减

其中，第一步对FontSize的动画只需要执行一次，第二步和第三步需要循环顺序执行(2-3-2-3-2-3)；显然单纯设置Storyboard或者Animation很难达成效果；
可以通过嵌套Storyboard的方式来实现，然后分别设置RepeatBehavior，嵌套Storyboard这个用法可能比较冷门

<Storyboard>
    <Storyboard>
        <DoubleAnimation
            Storyboard.TargetProperty="FontSize"
            To="32"
            Duration="0:0:0.2" />
    </Storyboard>
    <Storyboard RepeatBehavior="9999">
        <DoubleAnimation
            BeginTime="0:0:0"
            Storyboard.TargetName="rectangle"
            Storyboard.TargetProperty="Tag"
            To="1"
            Duration="0:0:1" />
        <ThicknessAnimation
            BeginTime="0:0:1.2"
            FillBehavior="Stop"
            Storyboard.TargetName="rectangle"
            Storyboard.TargetProperty="Margin"
            To="240,0,0,0"
            Duration="0:0:1" />
    </Storyboard>
</Storyboard>

单靠这个可以做出类似这种效果

Converter代码，其他类型或者其他逻辑的Converter原理一样；跟上面的ControlTemplate中绑定控件宽度原理一致

public class Compare2DoubleValueConverter : IMultiValueConverter
{
    public object Convert(object[] values, Type targetType, object parameter, CultureInfo culture)
    {
        double a = double.Parse(values[0].ToString()!);
        double b = double.Parse(values[1].ToString()!);

        var res = a == b;

        return res;
    }

    public object[] ConvertBack(
        object value,
        Type[] targetTypes,
        object parameter,
        CultureInfo culture
    )
    {
        throw new NotImplementedException();
    }
}

通用枚举绑定ComboBox，替代ObjectDataProvider

扩展类

public static class EnumHelper
{
    public static string Description(this Enum e)
    {
        return (
                e.GetType()
                    .GetField(e.ToString())
                    .GetCustomAttributes(typeof(DescriptionAttribute), false)
                    .FirstOrDefault() as DescriptionAttribute
            )?.Description ?? e.ToString();
    }
}

[ValueConversion(typeof(Enum), typeof(IEnumerable<(Enum Value, string Desc)>))]
public class EnumToCollectionConverter : MarkupExtension, IValueConverter
{
    public object Convert(object value, Type targetType, object parameter, CultureInfo culture)
    {
        var res = Enum.GetValues(value.GetType())
            .Cast<Enum>()
            .Select(e => new { Value = e, Desc = e.Description() })
            .ToList();
        return res;
    }

    public object ConvertBack(object value, Type targetType, object parameter, CultureInfo culture)
    {
        return null;
    }

    public override object ProvideValue(IServiceProvider serviceProvider)
    {
        return this;
    }
}

IOC

public partial class App : Application
{
    public new static App Current => (App)Application.Current;

    public IServiceProvider Services { get; private set; }

    private void Application_Startup(object sender, StartupEventArgs e)
        {
            Services = ConfigureServices();
            var mainWindow = Services.GetService<MainWindow>();
            mainWindow!.Show();
        }

    private static IServiceProvider ConfigureServices()
        {
            var services = new ServiceCollection();

            services.AddTransient<MainWindow>();

            return services.BuildServiceProvider();
        }

    public App() { }
}
