List的实现类

Java 的 List 是非常常用的数据类型。List 是有序的 Collection。Java List 一共三个实现类：分别是 ArrayList、Vector 和 LinkedList。 ArrayList（数组） ArrayList是线程不安全的，是最常用的 List 实现类，内部是通过数组实现的，它允许对元素进行快速随机访问。因此，它适合随机查找和遍历，数组的缺点是每个元素之间不能有间隔，当数组大小不满足时需要增加存储能力，就要将原数组的数据复制到新的存储空间中。当从 ArrayList 的中间位置插入或者删除元素时，需要对数组进行复制、移动，代价比较高。因此，不适合插入和删除。 LinkedList（链表） LinkedList是线程不安全的， 是用链表结构存储数据的，很适合数据的动态插入和删除，随机访问和遍历速度比较慢。另外，他还提供了 List 接口中没有定义的方法，专门用于操作表头和表尾元素，可以当作堆栈、队列和双向队列使用。 Vector（数组实现、线程同步） Vector 是线程安全的，与 ArrayList 一样，也是通过数组实现的，不同的是它支持线程的同步，即某一时刻只有一 个线程能够写 Vector，避免多线程同时写而引起的不一致性，但实现同步需要很高的花费，因此，访问它比访问 ArrayList 慢。

NullPointerException(空指针异常)

NullPointerException(空指针异常)对象为Null还拿来使用，就会出现此异常。 多的不说少的不聊，直接上代码！！！ public class Main { public static void main(String[] args) { String str1 = ""; String str2 = null; System.out.println(str1.length()); //0 System.out.println(str2.length()); //Exception in thread "main" java.lang.NullPointerException at Main.main(Main.java:7) } } 首先创一个长度为0的空字符串，然后让String类型的引用str1指向该对象。 然后用引用str1调用String类的length()方法。得到str1引用所指向的字符串对象的长度为0。 接下来让String类型的引用str2指向null。 然后用引用str2调用String类的length()方法。这里并不会如str1那样得到str2引用所指向的对象空间的长度也为0。反而会抛出NullPointerException(空指针异常)对象为Null还拿来使用，就会出现此异常。该异常继承自RuntimeException，是运行时异常。 当引用str2调用String类中的length()方法时，控制台会显示下述异常。 Exception in thread "main" java.lang.NullPointerException at Main.main(Main.java:7) 总结： 不管是那种引用数据类型，只要引用类型的引用指向null对象且调用引用类型的方法， 就会抛出NullPointerException(空指针异常)对象为Null还拿来使用，就会出现此异常。
